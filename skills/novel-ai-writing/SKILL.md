# AI 协作写作

## 功能定位

把 novel-skills 的所有写作模板**转译成可执行的 AI prompt**。

之前的 skill 都是"人看"的——告诉作者"应该怎么写"。这个 skill 是"AI 看"的——告诉 AI 怎么帮作者写。

**核心理念**：作者是导演，AI 是演员/编剧/编辑。**作者要的是 AI 能直接产出"可用的稿子"或"有用的辅助"**。

## 三阶段 prompt 设计

| 阶段 | 解决什么 | 典型 prompt |
|------|---------|----------|
| **写前** | 让 AI 帮你做节拍卡、章纲、设定检查、爽点设计 | "根据这本书的卷纲和节拍卡，请补全 chapter-005 的完整 beat" |
| **写中** | 让 AI 帮你写对话、战斗、感情戏、章尾、描写 | "请用对话写作技法的 5 种刺点，写一段主角和苏晚的告别戏" |
| **写后** | 让 AI 帮你做体检、改写、连贯性检查、去 AI 味 | "请用章节体检模板检查这一章，按 P0/P1/P2 列出问题清单" |

## 特殊场景 prompt

| 场景 | 解决什么 | 典型 prompt |
|------|---------|----------|
| **角色扮演** | 让 AI 扮演某角色和你对戏 | "你现在是苏晚。根据你的人物卡和当前状态，回复主角刚才对你说的话" |
| **批量生成** | 让 AI 一次性生成多章或多场景 | "请按节拍卡生成 chapter-001 到 chapter-005 的完整初稿" |
| **风格学习** | 让 AI 学习你的文风 | "请分析我最近 10 章的文风，然后模仿这种风格重写这一段" |
| **灵感生成** | 让 AI 给你情节建议 | "主角被困禁地，请给 5 种可能的脱困方式" |

## 与其他 skill 的关系

- `novel-studio` 路由到本 skill 时，会调用对应阶段的 prompt
- `novel-writing` 写正文时，可以引用本 skill 的 prompt
- `novel-quality` 体检时，可以使用本 skill 的写后 prompt
- `novel-worldbuilding` 设设定时，可以使用本 skill 的写前 prompt

## 加载规则

> 使用本 skill 时，**先告诉 AI 你的项目结构**，再调用具体 prompt。

### 标准开场

```
我正在写一本{{题材}}小说，名为《{{书名}}》。
当前写到第{{卷号}}卷第{{章节号}}章。

我的项目目录结构：
- 00-书核/：核心书核、读者承诺
- 10-设定/：人物、世界、势力、地点
- 20-大纲/：卷纲、节拍、伏笔、回收
- 30-正文/：已经写完的章节
- 40-修订/：体检报告、修稿报告
- 90-运行/：当前进度、决策记录、连载驾驶舱

接下来我要做的事是：{{你的具体任务}}
```

## 使用方法

### 方式 1：直接调用 prompt

读 `references/写前-prompt.md`，找到对应模板，填入具体信息，直接贴给 AI。

### 方式 2：通过 skill 触发

```
用 $novel-ai-writing 帮我做 chapter-005 的节拍卡。
```

skill 会自动加载本目录的所有 prompt 模板，帮你拼接最合适的 prompt。

## 不在这里完成的事

- 不替你做"作者决策"（写什么、怎么写、为什么这样写）
- 不替你做"价值判断"（这个情节是否好、这个角色是否立体）
- AI 的产出需要你**人工审校**——它只是工具，不是作者
- 不要把所有事都交给 AI——AI 擅长"填空"和"补完"，不擅长"原创构思"

## 配合使用的最佳实践

1. **重要决策自己做**：选什么结构、什么走向、什么结局——这些由你定
2. **重复工作给 AI**：写固定模板的内容、做机械检查
3. **AI 产出后必改**：AI 写出来的东西一定有"AI 味"，必须人工润色
4. **多版本对比**：让 AI 给 2-3 个版本，你挑最好的
5. **保持作者 voice**：用 `references/特殊场景-prompt.md` 的"风格学习"让 AI 学习你的文风
