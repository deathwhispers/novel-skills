# 写前阶段 AI Prompt 模板库

> **使用说明**：写正文之前的准备工作——节拍卡、章纲、设定检查、爽点设计等。
> 每个 prompt 都可以直接复制贴给 AI，配合具体项目信息使用。

---

## 一、标准开场（每次必用）

> 让 AI 知道你的项目背景。

```
我正在写一本 {{题材}} 小说，名为《{{书名}}》。
当前写到第 {{卷号}} 卷第 {{章节号}} 章。

我的项目目录结构：
- 00-书核/：核心书核、读者承诺
- 10-设定/：人物、世界、势力、地点
- 20-大纲/：卷纲、节拍、伏笔、回收
- 30-正文/：已经写完的章节
- 40-修订/：体检报告、修稿报告
- 90-运行/：当前进度、决策记录、连载驾驶舱

接下来我要做的事是：{{你的具体任务}}
```

## 二、节拍卡生成

### Prompt 1：从卷纲生成节拍卡

```
我正在写《{{书名}}》第 {{卷号}} 卷。

请按以下信息，生成 chapter-{{章节号}} 的完整节拍卡。

【本卷卷纲】
{{粘贴卷通用模板填写后的内容}}

【本章理论任务】
{{这一章在全卷中扮演什么角色}}

【需要的 10 项输出】
1. 这一章在全卷中扮演什么角色
2. 这一章是谁的戏（视角人物）
3. 这章必须完成什么（核心戏剧任务）
4. 这一章是被前文什么结果逼出来的（因果锚点）
5. 真正卡住角色的阻力是什么
6. 这一章局面怎么变了
7. 情绪/关系怎么变了
8. 本章写完后必须记住什么（新增的信息债/关系债/物件债）
9. 章尾拿什么继续吊读者
10. 本章碰了哪几条埋线（旧线升级/新线埋下/信息回收/关系债）

要求：
- 每一项都要具体、可执行
- 不要用"会"、"可能"等模糊词
- 章尾必须从"新问题/不可逆决定/倒计时/权力反转/情绪余震/关系破口"中选
```

### Prompt 2：节拍卡检查

```
请检查以下 chapter-{{章节号}} 的节拍卡：

{{粘贴节拍卡内容}}

按以下标准检查：
1. 10 项是否都填写
2. 每一项是否具体（不是"主角变强了"这种空话）
3. 章尾是否明确（不能是"平静收束"）
4. 因果锚点是否清楚（不是凭空开始）
5. 阻力是否真实（不是"很难"这种抽象）

按 P0/P1/P2 列出问题清单。
```

【章纲要求】
- 字数：3000-4000 字
- 场景数：2-3 个
- 章内节奏：4 段式（铺垫-推进-高潮-余震）

1. 场景 1（章节前 1/3）
2. 场景 2（章节中段）
3. 场景 3（章节末段）
4. 每场景的核心戏剧任务
5. 章尾的具体设计（最后 1-2 段写什么）

## 四、设定检查

### Prompt 5：设定一致性检查

```
请帮我检查 chapter-{{章节号}} 的正文与项目设定的一致性。

【正文】
{{粘贴章节内容}}

【需要对照的设定文件】
- 10-设定/硬设定.md
- 10-设定/世界规则.md
- 10-设定/力量体系.md
- 10-设定/地点.md
- 10-设定/角色/主角.md 及相关角色卡
- 20-大纲/因果/场景因果图.md

请逐项检查：
1. 力量体系是否被突破（境界、代价）
2. 世界规则是否被违反
3. 角色人设是否前后一致
4. 地点描写是否与设定一致
5. 因果链是否合理

按 P0/P1/P2 列出问题清单。
```

### Prompt 6：硬设定冲突扫描

```
请扫描项目所有正文（{{章节数}} 章）与硬设定.md 的冲突。

【硬设定】
{{粘贴硬设定.md}}

【正文位置】
30-正文/章节/

【检查重点】
1. 不可违反的规则（如"修为不可越级"）
2. 不可改变的事实（如"主角是孤儿"）
3. 时间线冲突（如"三年前发生过 X"但正文中 Y 事件在 X 之后）
4. 地点冲突（如"青云宗在北"但正文中说"在东"）
5. 关系冲突（如"A 是 B 的师父"但正文中 B 不认识 A）

只报告 P0/P1 级问题。
```
