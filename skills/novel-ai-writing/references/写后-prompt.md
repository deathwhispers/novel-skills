# 写后阶段 AI Prompt 模板库

> **使用说明**：写完正文后的检查、修缮、去 AI 味等。
> 每个 prompt 都对应 novel-checkup / novel-revision / novel-deslop 的核心能力。

## 一、章节体检

### Prompt 1：完整章节体检

```
请按以下章节体检模板，检查 chapter-{{章节号}}。

【章节正文】
{{粘贴}}

【章节信息】
- 节拍卡：{{}}
- 上一章最后段落：{{}}
- 角色状态：{{}}

【检查项】
【novel-checkup/references/章节体检模板.md 的 7 项】
1. 本章是否站得住

### Prompt 2：单点深度检查

#### Prompt 2.1：连续性检查

```
请检查 chapter-{{章节号}} 的连续性。

【章节正文】
{{粘贴}}

【必须对照的文件】
- 上一章正文：chapter-{{章节号-1}}.md
- 90-运行/人物状态变迁日志.md
- 90-运行/当前进度.md
- 10-设定/角色/{{相关角色}}.md
- 10-设定/硬设定.md
- 10-设定/世界规则.md

【检查重点】
- 人物称呼是否稳定
- 视角人物知道的信息是否稳定
- 时间顺序是否说得通
- 地点移动是否有交代
- 物件是否凭空出现或消失
- 伤势/能力消耗/关系裂痕是否持续生效
- 设定规则是否被正文自己打破

【长篇额外】
- 支线人物是否无故失踪
- 旧线是否长时间失联又无提醒
- 本章新增的信息债是否记进后续跟踪表
- 近期最值钱的读点是否被连续几章写散

按 P0/P1/P2 列出问题。
```

#### Prompt 2.2：前后照应检查

```
请检查 chapter-{{章节号}} 的前后照应。

【章节正文】
{{粘贴}}

【检查重点】（中文长篇最常丢的 5 类）

2. 前章刚受的伤、刚知道的信息、刚破的关系，这章是否还在生效

- 丢的是哪一条

【中后段额外】
- 这条线如果继续不碰，会不会伤追读
- 它是该正式回收，还是至少要提醒一次还活着

按 P0/P1/P2 列出问题。
```

## 二、章尾强度检测

### Prompt 3：章尾强度

```
请为 chapter-{{章节号}} 的章尾打分（1-5）。

【章尾最后 200 字】
{{粘贴}}

【章尾的 6 种发动机】
1. 新问题
2. 不可逆决定
3. 倒计时
4. 权力反转
5. 情绪余震
6. 关系破口

【章尾强度评分】
- 5：极强，读者会立刻想翻下一章
- 4：强，有明显推动
- 3：中等，有一定推动
- 2：弱，可能让读者犹豫
- 1：几乎没有，提示章尾设计失败

【输出】
1. 强度评分
2. 发动机类型
3. 改进建议
```

```

4. 至少有一项推进（信息/关系/局面/角色）
