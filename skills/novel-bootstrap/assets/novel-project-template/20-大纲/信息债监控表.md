# 信息债监控表

> **核心问题**：这条信息债是什么时候埋的？已经多少章没碰了？会不会读者已经忘了？
> 信息债（伏笔/悬念/未揭示的信息）是追读的核心动力，但过期不碰的信息债会变成毒点。
> **规则**：每 10 章扫描一次本表，标出即将「过期」的信息债。

## 信息债总览

| 线头 | 类型 | 埋下章节 | 上次触碰 | 距上次触碰 | 计划回收章节 | 过期预警 |
|------|------|---------|---------|-----------|------------|---------|
| | | | | (章) | | 安全/注意/危险/过期 |
| | | | | | | |

> **过期预警说明**：
> - **安全**：10 章内碰过
> - **注意**：11-20 章没碰了，需要尽快刷存在感
> - **危险**：21-30 章没碰了，很大一部分读者已经遗忘
> - **过期**：30 章以上没碰，建议重新规划这条线的回收方式

## 不同类型信息债的合理保鲜期

| 类型 | 定义 | 最长不碰间隔 | 读者遗忘风险 |
|------|------|-------------|-------------|
| 线索 | 指向某个真相的碎片信息 | 20 章 | 中 |
| 关系债 | 一段关系中的未兑现承诺或未爆发的冲突 | 15 章 | 高 |
| 物件债 | 某个关键物品的来历/用途/后果 | 25 章 | 中 |
| 承诺债 | 角色之间的约定/威胁/誓言 | 20 章 | 中 |
| 情绪债 | 未消化的情绪冲突或误解 | 12 章 | 高 |
| 身份谜题 | 角色真实身份/来历的悬疑 | 15 章 | 极高 |

## 需要立即处理的信息债

> 标记为「危险」或「过期」的债，优先在 3 章内安排一次触碰（不一定要回收，提醒即可）。

| 线头 | 危险等级 | 上次触碰 | 最近一次提醒的方式 | 计划行动 |
|------|---------|---------|------------------|---------|
| | | | 角色提起/间接影响/类似事件暗示 | |
| | | | | |

## 信息债触碰方式

> 不一定非要在该回收的时候才碰。可以用以下方式「刷存在感」：

| 方式 | 效果 | 适用场景 |
|------|------|---------|
| 角色不经意提起 | 温和提醒，不增加紧迫感 | 中期线头 |
| 间接影响当前事件 | 让读者感觉到「这条线还在动」 | 长线伏笔 |
| 类似事件暗示 | 提醒读者「之前也有过类似的事」 | 对称结构 |
| 代价浮出水面 | 让读者想起「这条线如果爆了会很严重」 | 即将回收的线 |
| 误判或假回收 | 给读者一个假答案，再推翻 | 悬疑线 |

## 信息债回收节奏建议

| 全书进度 | 应该收多少 | 建议 |
|---------|-----------|------|
| 0-25% | 埋线期 | 只埋不收，但每条已埋线至少碰 1-2 次 |
| 25-50% | 小回收期 | 回收 20-30% 的短线，长线开始升级/变形 |
| 50-75% | 中回收期 | 回收 50% 左右的线，最长的那几条线开始加速 |
| 75-100% | 大回收期 | 回收 90%+ 的线，留少数做续集/余韵 |
