# 小说工作区规格

统一小说工程采用以下目录：

```text
00-书核/
10-设定/
20-大纲/
30-正文/
40-修订/
90-运行/
```

## 目录职责

- `00-书核/`: 创作简报、作品定位、系列承诺
- `10-设定/`: hard canon、世界规则、人物与势力
- `20-大纲/`: 主线结构、分卷计划、章节节拍卡、埋线与回收表
- `30-正文/`: 正文章节、场景和导入稿
- `40-修订/`: 体检报告、修订记录与章节报告
- `90-运行/`: 当前状态、决策、交接和连续性问题

## 最小可用状态

一个“能继续写”的项目至少应当有：

- `00-书核/作品总表.md`
- `00-书核/立项单.md`
- `20-大纲/全书总纲.md`
- `90-运行/当前进度.md`

如果是中文长篇，建议再补：

- `20-大纲/分卷/volume-01.md`
- `20-大纲/节拍卡/chapter-001.md`
- `20-大纲/因果/scene-causality-map.md`
- `20-大纲/回收/payoff-ledger.md`
- `40-修订/体检报告/`

如果目标是“尽快进入长篇连载”，建议把下面这些当成快启核心：

- `00-书核/立项单.md`
- `00-书核/长线承诺.md`
- `20-大纲/分卷/volume-01.md`
- `20-大纲/节拍卡/chapter-001.md`
- `20-大纲/首卷发射台.md`
- `90-运行/当前进度.md`
- `90-运行/连载驾驶舱.md`

这套组合的目标不是一次把全书想完，而是先让作者能稳定写出前三章、第一卷和第一波追读点。
