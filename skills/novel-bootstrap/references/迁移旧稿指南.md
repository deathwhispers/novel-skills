# 迁移旧稿指南

如果你已经有正在写的小说项目（已有的稿件、设定、大纲），想迁移到这个模板中，按以下步骤操作。

## 迁移流程

### Step 1：初始化项目目录

```bash
python3 skills/novel-bootstrap/scripts/init_novel_project.py \
  --output /path/to/new-project \
  --title "你的书名" \
  --genre "题材" \
  --premise "一句话故事前提"
```

### Step 2：迁移设定

| 源文件 | 目标位置 | 处理方式 |
|-------|---------|---------|
| 角色卡/人设表 | 10-设定/角色/ | 逐角色迁移，确保角色名和身份一致 |
| 世界观文档 | 10-设定/世界规则.md | 合并入对应文件，删除冗余 |
| 力量体系/等级设定 | 10-设定/力量体系.md | 保持等级清晰，标注前后是否有矛盾 |
| 地图/地点列表 | 10-设定/地点.md | 按重要性排序 |
| 势力/门派设定 | 10-设定/势力.md | 理清势力间关系 |
| 时间线 | 10-设定/时间线.md | 标注关键事件节点 |

### Step 3：迁移大纲

| 源文件 | 目标位置 | 处理方式 |
|-------|---------|---------|
| 全书大纲 | 20-大纲/全书总纲.md | 提炼主线、副线、感情线 |
| 分卷规划 | 20-大纲/分卷/ | 一卷一文件 |
| 已有的章节规划 | 20-大纲/节拍卡/ | 一章一节拍卡 |
| 伏笔记录 | 20-大纲/回收/回收总账.md | 逐条登记，标注当前状态 |
| 因果笔记 | 20-大纲/因果/场景因果图.md | 把已发生和未发生的因果对应上 |

### Step 4：迁移正文

| 源文件 | 目标位置 | 处理方式 |
|-------|---------|---------|
| 已写章节 | 30-正文/第X卷-卷名/ | 按章节编号重命名（如第001章.md） |
| 待定场景 | 30-正文/场景/ | 命名：{场景关键字}-草稿.md |
| 导入源文件 | 30-正文/导入/源文件索引.md | 记录原始文件位置，方便校对 |

### Step 5：初始化运行数据

| 目标文件 | 填写内容 |
|---------|---------|
| 90-运行/当前进度.md | 当前写到第几卷第几章 |
| 90-运行/连载驾驶舱.md | 当前连载状态和近期任务 |
| 90-运行/决策记录.md | 本次迁移的决策 |
| 90-运行/连续性查询.md | 已知的设定连续性疑问 |

### Step 6：校验

```bash
python3 scripts/validate_novel_project.py /path/to/new-project
```

## 迁移原则

1. **先搬后整**：先把所有资料搬进正确的目录，再回头整理。不要边搬边改。
2. **不删原始**：迁移过程中不要删除源文件，直到验证项目结构完整。
3. **标记老坑**：旧稿中已知的设定冲突、断线、未回收伏笔，在回收总账里标记为"旧稿遗留"。
4. **一次一卷**：如果已经写了多卷，优先迁移当前正在写的那卷，其余归档为旧稿。
