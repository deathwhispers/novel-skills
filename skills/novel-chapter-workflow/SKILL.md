# 章节质量门禁

## 功能定位

这是章节写作的强制质量门禁。它不直接写正文，而是管理从草稿到成品的全流程，确保每个章节都经过完整的质量检查。这个 skill 会强制执行以下流程：

1. **写前准备** - 确保下笔前所有必要信息就位
2. **草稿执行** - 按标准流程写初稿
3. **章节体检** - 强制进行质量检查
4. **修订统稿** - 根据体检结果进行必要修订
5. **去AI味** - 确保文字自然流畅
6. **状态回写** - 更新所有相关文件

## 长篇章节数量要求

**百万字长篇必须满足**：
- 总章节数 ≥ 400章
- 分卷数 ≥ 10卷
- 每卷章节数 ≥ 30章
- 每卷字数 ≥ 6万字

**章节规划验证清单**：
- [ ] 全书总章节数 ≥ 400章
- [ ] 分卷数 ≥ 10卷
- [ ] 每卷章节数 ≥ 30章
- [ ] 每卷字数 ≥ 6万字
- [ ] 每卷都有明确的功能定位
- [ ] 冲突升级路径清晰
- [ ] 角色成长弧线完整

## 强制流程触发条件

以下任意条件满足时，自动触发本流程：

1. 用户请求写新章节
2. 用户请求续写正文
3. 用户请求写下一章
4. 用户请求按大纲继续写

## 完整流程步骤

### 第一步：写前准备（强制）

**必须完成以下检查才能开始写作：**

1. **加载最小上下文包**
   - `00-书核/作品总表.md`
   - `00-书核/立项单.md`
   - 当前章节 beat
   - 当前卷文件

2. **完成写前检查表**
   - 读取 `references/写前检查表.md`
   - 确保所有空项都已补全
   - 如果有三项以上说不清，先回到 `novel-outlining` 补齐

3. **锁定本章任务**
   - 明确本章要完成什么任务
   - 明确本章冲突是什么
   - 明确前后照应关系
   - 明确人物状态
   - 明确章尾推进器

4. **确认可以下笔**
   - 所有必要信息就位
   - 没有遗漏的关键设定
   - 可以开始写作

### 第二步：草稿执行（强制）

**必须按以下标准执行：**

1. **按起稿执行链写作**
   - 先锁本章最重要的一场戏
   - 先决定角色这次主动做什么
   - 先把冲突和阻力摆上桌
   - 先让局面发生变化
   - 先保章尾发动机

2. **完成场景级写作循环**
   - 每场至少完成：角色想做什么、被什么卡住、局面变成什么、留下什么债
   - 如果参考了拆文结果，额外自检因果闭合、接上前文、细节承载

3. **守住质量底线**
   - 不用整段总结代替戏
   - 不用作者解释代替人物行动
   - 不用抽象情绪词代替身体反应
   - 不让角色在没有代价的情况下突然改变
   - 章尾必须留下问题、压力、决定或余震

4. **完成写后自检**
   - 读取 `references/写后自检卡.md`
   - 检查本章是否达到 `references/章节质量标准.md` 的最低标准
   - 如果主要问题在前后打架、漏线、状态没续上，转 `novel-checkup`
   - 如果主要问题在段落空、转折站不住、章尾没力，转 `novel-revision`

### 第三步：章节体检（强制）

**必须完成以下体检：**

1. **使用 `novel-checkup` 进行体检**
   - 按六层检查顺序进行
   - 输出体检报告到 `40-修订/体检报告/`

2. **体检内容**
   - 本章自己是否成立
   - 和上一章、上一场是否接得上
   - 人物状态、关系状态、已知信息是否延续
   - 伏笔、信息债、物件债、关系债有没有丢
   - 章尾是否有效推动下一章
   - 如果是长篇：卷级节奏、长线推进和追读压力是否还在

3. **根据体检结果决定下一步**
   - 如果体检通过，进入去AI味阶段
   - 如果体检发现问题，进入修订统稿阶段
   - 如果问题根源在大纲，转回 `novel-outlining`
   - 如果问题根源在设定冲突，联动 `novel-worldbuilding`

### 第四步：修订统稿（根据体检结果）

**如果体检发现问题，必须进行修订：**

1. **选择主问题模板**
   - 因果断裂、动机悬浮 -> `references/逻辑修补模板.md`
   - 细节稀薄、情绪不落地 -> `references/细节修补模板.md`
   - 伏笔散、回收硬 -> `references/回收修补模板.md`

2. **按三层修订梯子修订**
   - 结构层：beat是否成立、章节目标是否清晰、转折是否发生、结尾钩子是否有效
   - 场景层：对抗是否足够、行动与因果是否顺、信息是否在正确时机揭示、对话是否有角色区分度
   - 句子层：是否过度总结、是否讲解味太重、是否对称平铺泛化、是否存在明显AI套话

3. **输出修订报告**
   - 修订后的正文
   - `40-修订/修稿报告/` 下对应的修订报告
   - 如涉及 canon 改动，在 `90-运行/决策记录.md` 记录

### 第五步：去AI味（强制）

**必须完成以下去味检查：**

1. **使用 `novel-deslop` 去AI味**
   - 检查四类AI味问题：解释过多、节奏过匀、情绪靠概括词、角色说话像同一个人
   - 按处理顺序：先删解释句和总结句，再拆平滑句式，再补动作、视角感知和人味停顿，最后检查是否改坏角色声音

2. **题材差异化去味**
   - 玄幻/仙侠：避免修炼过程过于详细
   - 都市/现实：避免场景描写面面俱到
   - 言情/感情流：避免角色情绪被直接说出来
   - 悬疑/推理：避免线索解释过于清楚

3. **输出去味结果**
   - 主要AI味来源（具体到哪些句子是哪种问题）
   - 修订后的正文（标记改动处）
   - 极短的修改说明

### 第六步：状态回写（强制）

**必须完成以下更新：**

1. **更新章节文件**
   - `30-正文/第X卷-卷名/第XXX章.md` 中对应章节

2. **更新进度文件**
   - `90-运行/当前进度.md` 中的当前进度、未决问题、下一步

3. **更新相关文件**
   - 如出现新 hard canon，补到 `10-设定/硬设定.md` 或记入 `90-运行/决策记录.md`
   - 如本章推进了伏笔、人物状态、因果债或关系债，联动更新 `20-大纲/因果/场景因果图.md` 或 `20-大纲/回收/回收总账.md`

## 强制判定机制

### 步骤完成判定

每个步骤完成后，必须通过以下判定才能进入下一步：

1. **写前准备完成判定**
   - 写前检查表所有空项已补全
   - 本章任务、冲突、前后照应已明确
   - 人物状态已确认
   - 章尾推进器已设计

2. **草稿执行完成判定**
   - 正文初稿已完成
   - 场景级写作循环已完成
   - 质量底线已守住
   - 写后自检已完成

3. **章节体检完成判定**
   - 体检报告已生成
   - 所有检查项已完成
   - 问题清单已输出
   - 下一步方向已明确

4. **修订统稿完成判定**（如果需要）
   - 主问题模板已选择
   - 三层修订已完成
   - 修订报告已输出
   - 相关文件已更新

5. **去AI味完成判定**
   - 四类AI味问题已检查
   - 去味处理已完成
   - 去味结果已输出

6. **状态回写完成判定**
   - 章节文件已更新
   - 进度文件已更新
   - 相关文件已更新

### 失败处理机制

如果某个步骤失败：

1. **写前准备失败**
   - 停止写作流程
   - 转回 `novel-outlining` 补齐信息
   - 补齐后重新开始

2. **草稿执行失败**
   - 停止继续写作
   - 转 `novel-checkup` 检查问题
   - 根据检查结果决定下一步

3. **章节体检失败**
   - 停止进入下一步
   - 转 `novel-revision` 进行修订
   - 修订后重新体检

4. **修订统稿失败**
   - 停止进入下一步
   - 转回 `novel-outlining` 重新规划
   - 重新开始写作流程

5. **去AI味失败**
   - 停止进入下一步
   - 重新检查AI味问题
   - 重新处理

## 产物

每个流程完成后，必须输出：

1. **写前准备产物**
   - 完成的写前检查表
   - 本章任务卡

2. **草稿执行产物**
   - 正文初稿
   - 写后自检结果

3. **章节体检产物**
   - `40-修订/体检报告/{chapter}-checkup.md`
   - 问题清单

4. **修订统稿产物**（如果需要）
   - 修订后的正文
   - `40-修订/修稿报告/` 下对应的修订报告

5. **去AI味产物**
   - 去味后的正文
   - 去味说明

6. **状态回写产物**
   - 更新后的章节文件
   - 更新后的进度文件
   - 更新后的相关文件
