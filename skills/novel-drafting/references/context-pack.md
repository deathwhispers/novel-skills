# Context Pack

## 默认读取

1. `00-story-core/project-meta.md`
2. `00-story-core/creative-brief.md`
3. 对应的 chapter beat
4. 对应的 volume 文件

## 条件读取

出现这些情况再追加：

- 角色冲突较复杂 -> 读角色文件
- 规则制约明显 -> 读 `world-rules.md` 或 `canon.md`
- 强连续续写 -> 读上一章
- 会话接续不稳定 -> 读 `90-ops/current-state.md`
- 想提高逻辑完整性 -> 读相关拆文分析中的 `Logic And Causality`
- 想提高细节有效性 -> 读相关拆文分析中的 `Detail Carriers`
- 长篇推进跨章因果或伏笔 -> 读 `20-outline/causality/scene-causality-map.md` 和 `20-outline/payoff-tracking/payoff-ledger.md`

上下文包选完后，最后再读一次 `references/prewrite-checklist.md`，把章节目标、阻力、因果、细节载体和结尾推进器锁死，再开始起草。

## 拆文反哺

如果项目已经做过对标拆文，优先把拆文结果当成“写法约束”而不是“抄写参考”：

- 从 `Logic And Causality` 继承因果闭合方式
- 从 `Detail Carriers` 继承细节承担信息/情绪的方法
- 从 `Adaptation Notes For My Project` 继承哪些可借、哪些必须变形

## 禁忌

- 不默认全量读取整个 `10-bible/`
- 不默认把所有历史章节都读一遍
- 不在写作前先做全面修订
- 不把拆文样例当成表层模仿稿
- 不在写前自检仍有大块空白时强行开写
