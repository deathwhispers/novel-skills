# 场景多样化检查表

> **使用说明**：百万级长篇写到中后期，作者不自觉地"复用"场景——密室/对峙/打斗/告白都用一样的模板，读者会审美疲劳。
> 本表提供 20+ 种场景类型 + 多样化检查清单。
> 写每章前扫一眼"最近 10 章用了什么场景"，避免重复。

## 一、20+ 种场景类型

> 长篇网文常见的场景类型——每种有不同的"必备元素"和"读者情绪"。

### 1. 对峙场景

- **描述**：两人或多方面对面，言语/气势上的对抗
- **必备元素**：双方立场/身体语言/关键对话
- **读者情绪**：紧张/期待
- **示例**：李昭对主角的当众嘲讽

### 2. 战斗/打斗场景

- **描述**：物理上的冲突
- **必备元素**：实力对比/过程/胜负/代价
- **读者情绪**：紧张/爽/痛
- **示例**：内门大比、外门追杀

### 3. 追逐/逃亡场景

- **描述**：一方跑一方追
- **必备元素**：紧迫感/障碍物/转折
- **读者情绪**：紧张/揪心
- **示例**：主角被反派追杀

### 4. 误会/和解场景

- **描述**：双方产生误会或解开误会
- **必备元素**：误会的"看似合理"/和解的契机
- **读者情绪**：揪心/释然
- **示例**：苏晚与主角的误会

### 5. 告白/表白场景

- **描述**：一方或双方表达感情
- **必备元素**：铺垫/触发/回应/影响
- **读者情绪**：期待/释然
- **示例**：主角向苏晚表白

### 6. 诀别/告别场景

- **描述**：两人即将分开
- **必备元素**：未说出的话/留下的物件/场景
- **读者情绪**：揪心/遗憾
- **示例**：主角进禁地前的告别

### 7. 重逢场景

- **描述**：分离后再次相遇
- **必备元素**：变化的双方/关系重塑
- **读者情绪**：惊喜/复杂
- **示例**：失散多年的师徒重逢

### 8. 牺牲/死亡场景

- **描述**：某个角色为另一个人/某件事牺牲
- **必备元素**：动机/过程/留下的东西
- **读者情绪**：虐/震撼
- **示例**：苏晚为救主角挡剑

### 9. 揭示/揭秘场景

- **描述**：某个秘密被揭开
- **必备元素**：铺垫/揭示的"震撼"/反应
- **读者情绪**：震惊/释然
- **示例**：主角真实身份揭晓

### 10. 宴席/聚会场景

- **描述**：多人参与的社交场景
- **必备元素**：暗流涌动/各自立场
- **读者情绪**：观察/期待
- **示例**：宗门年会上的暗流

### 11. 集市/市井场景

- **描述**：普通人的日常生活
- **必备元素**：氛围/信息/小事件
- **读者情绪**：代入/轻松
- **示例**：临渊城集市上的偶遇

### 12. 私下密谈场景

- **描述**：两人（或少数人）私下说重要的事
- **必备元素**：信任关系/重要信息
- **读者情绪**：紧张/期待
- **示例**：主角与苏晚的深夜对话

### 13. 公开演讲/宣告场景

- **描述**：某人公开宣告某事
- **必备元素**：观众反应/影响
- **读者情绪**：震撼/期待
- **示例**：赵长老公开指控主角

### 14. 卧底/潜伏场景

- **描述**：某人假装身份潜伏
- **必备元素**：伪装/风险/心理压力
- **读者情绪**：紧张/揪心
- **示例**：江鹤潜伏在散修联盟

### 15. 审判/对峙公堂场景

- **描述**：多个角色在某种"公堂"上对峙
- **必备元素**：证人/证据/反转
- **读者情绪**：紧张/期待
- **示例**：宗门审判主角"勾结魔道"

### 16. 逃亡/流亡场景

- **描述**：主角离开一个地方
- **必备元素**：告别/目标/不确定
- **读者情绪**：期待/揪心
- **示例**：主角逃离青云宗

### 17. 重逢后的"日常"场景

- **描述**：主角和某人重新在一起
- **必备元素**：日常的细节/关系修复
- **读者情绪**：温暖/释然
- **示例**：主角与苏晚重逢后的相处

### 18. 顿悟/突破场景

- **描述**：角色经历心理/能力的突破
- **必备元素**：触发/过程/结果
- **读者情绪**：震撼/满足
- **示例**：主角顿悟龙魂之力

### 19. 传承/接任场景

- **描述**：师父将某物传给弟子
- **必备元素**：信任/仪式/影响
- **读者情绪**：感动/期待
- **示例**：守护者将龙族传承交给主角

### 20. 群像/会议场景

- **描述**：多个重要角色一起讨论
- **必备元素**：各人立场/矛盾/结果
- **读者情绪**：观察/期待
- **示例**：长老会议讨论主角的处置

### 21. 梦境/幻境场景

- **描述**：角色进入梦境/幻境
- **必备元素**：象征/启示/醒来后的变化
- **读者情绪**：神秘/期待
- **示例**：主角在梦中看到龙族历史

### 22. 战斗前的"准备"场景

- **描述**：角色为即将到来的战斗做准备
- **必备元素**：决心/细节/隐藏的紧张
- **读者情绪**：紧张/期待
- **示例**：主角在大比前夜的准备

## 三、场景多样化的 5 条原则

### 原则 1：连续 3 章不重复主场景

> 连续 3 章都是"战斗"会让读者疲劳。
> **建议**：战斗-对话-日常-战斗-对话-日常 的循环。

### 原则 2：避免"密室三连"

> 长篇网文最容易出现"密室-密室-密室"——所有场景都在室内。
> **建议**：加入 1 个室外场景（集市/追逐/野外）。

### 原则 3：避免"对话三连"

> 连续 3 章都是"对话"会显得"水"。
> **建议**：对话-动作-对话-动作-对话-动作 的循环。

### 原则 4：避免"战斗三连"

> 连续 3 章都是"战斗"会"打累了"。
> **建议**：战斗后必须有 1 章"喘息"（日常/对话/情感）。

### 原则 5：每个卷必须有 1-2 个"特别场景"

> 长篇网文必须有"标志性场景"——只有这本书有的。
> **示例**：一场"雪地告白"、一场"闹市对峙"。

## 五、场景的"变体化"技法

> 同样的场景类型可以通过"变体"避免重复。

### 战斗场景的 6 种变体

| 变体 | 写法 | 情绪 |
|------|------|------|
| 实力悬殊碾压 | 一招制敌 | 爽 |
| 势均力敌 | 双方都有机会 | 紧张 |
| 以弱胜强 | 智取+底牌 | 震撼 |
| 群战 | 多人对多人 | 混乱 |
| 追逐战 | 边跑边打 | 紧张 |
| 破阵战 | 多人合作破阵 | 紧张+合作 |

### 对峙场景的 5 种变体

| 变体 | 写法 | 情绪 |
|------|------|------|
| 当众对峙 | 公开场合 | 紧张 |
| 私下对峙 | 密室/夜谈 | 紧张+期待 |
| 第三方对峙 | 通过中间人 | 期待 |
| 隔空对峙 | 不见面但有冲突 | 期待 |
| 内心对峙 | 角色自己内心的矛盾 | 深入 |

### 告白场景的 4 种变体

| 变体 | 写法 | 情绪 |
|------|------|------|
| 直接告白 | 一方说"我喜欢你" | 紧张/释然 |
| 暗示告白 | 用行动/礼物 | 期待 |
| 第三方催化 | 共同朋友撮合 | 期待 |
| 危机中告白 | 生死关头的表白 | 震撼 |

## 六、场景的"地域变化"

> 同样的场景在不同地点有完全不同的"质感"。

| 场景类型 | 在密室 | 在野外 | 在闹市 | 在高处 |
|---------|------|------|------|------|
| 对峙 | 压抑 | 开阔 | 干扰 | 紧张 |
| 战斗 | 贴身 | 远程 | 平民介入 | 高空坠落风险 |
| 告白 | 私密 | 浪漫 | 尴尬 | 壮美 |
| 揭示 | 严肃 | 震撼 | 嘈杂 | 命运感 |

> **建议**：写不同场景时切换地点，避免"都在一个地方"。

## 七、场景与"读者情绪"配合

> 场景类型和读者情绪有强对应。

| 场景类型 | 主要读者情绪 |
|---------|------------|
| 战斗 | 紧张/爽 |
| 误会 | 揪心/着急 |
| 告白 | 期待/释然 |
| 揭示 | 震惊 |
| 牺牲 | 虐/震撼 |
| 重逢 | 惊喜 |
| 日常 | 温暖/代入 |
| 梦境 | 神秘/期待 |
| 传承 | 感动/期待 |
| 审判 | 紧张/期待 |

> **核心规则**：单章内可以有多个情绪，但应该"有主有次"。

## 八、场景的"翻车"形态

| 翻车 | 表现 | 危害 |
|------|------|------|
| **密室三连** | 连续 3 章都在室内 | 读者闷 |
| **对话三连** | 连续 3 章都是对话 | 读者觉得"水" |
| **战斗三连** | 连续 3 章都在打 | 读者打累了 |
| **场景同质化** | 同样是"对峙"但写法一样 | 读者审美疲劳 |
| **场景无变化** | 5 章都在一个地方 | 读者分不清 |
| **场景无意义** | 写了一个场景但没推动任何线 | 浪费篇幅 |
| **场景太多** | 一章 5+ 个场景切换 | 读者跟不上 |

## 九、场景多样化自检

写完一章后，逐项检查：

- [ ] 本章的"主场景"是什么类型（22 种之一）
- [ ] 必备元素是否都设计了
- [ ] 是否有"变体"（不是同一种场景的标准写法）
- [ ] 地点是否变化（不是和上章同一个地方）
- [ ] 视角是否变化（如果是多视角）
- [ ] 是否有"标志性"细节（让读者记住这个场景）
- [ ] 本章的场景是否与前 3 章"主场景"不同
- [ ] 本章的场景是否推动了剧情
- [ ] 是否有 1-2 个"特别场景"（标志性）
- [ ] 是否避开了 7 种翻车形态

## 十、场景的"复用 vs 变体"判断

> 什么时候复用场景，什么时候必须变体？

| 场景 | 复用 OK | 必须变体 |
|------|------|------|
| 战斗 | 短小战斗可以复用 | 关键战斗必须变体 |
| 对峙 | 配角对峙可以复用 | 主角对峙必须变体 |
| 告白 | 简单情节可以复用 | 主角正式表白必须变体 |
| 揭示 | 普通揭示可以复用 | 关键揭示必须变体 |
| 日常 | 日常可以复用 | 关键日常必须变体 |
| 牺牲 | 一般不能复用 | 重要牺牲必须有"独特写法" |

> **核心规则**：**对剧情有重大影响的场景必须变体**——这是避免"读者疲劳"的关键。
