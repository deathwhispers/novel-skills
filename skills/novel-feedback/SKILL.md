# 读者反馈追踪与分析

## 功能定位

把读者反馈从"零散评论"变成"可指导写作的系统性数据资产"。这个 skill 负责：

1. **评论收集与分类**：建立结构化的评论数据库
2. **追读数据追踪**：记录和分析追读率/读完率/弃书率的变化趋势
3. **情感与趋势分析**：从评论和数据中提炼情绪信号和优化方向
4. **反馈驱动写作调整**：把反馈转化为下一章/下一卷的具体写作指令
5. **跨 skill 协作**：向 outline/writing/market 输出标准化调整建议
6. **作者学习提炼**：把重复出现且有正文证据的问题转成一个阶段性训练目标

## 在重构流程中的位置

本 skill **不是重构流程的直线路径**，而是**旁路触发+贯穿全程**的反馈闭环：

```
可裁剪的写作直路
     ↓
发布 N 章
     ↓
触发反馈监控（每章/每周/每卷）
     ↓
读者反馈旁路启动
     ↓
收集 → 追踪 → 分析 → 决策 → 输出指令
     ↓
回到对应 skill（outline/writing/market）
     ↓
继续写作
```

**触发条件**（满足任一即启动）：
- 单章追读 < 50%
- 连续 2 章追读跌
- 读者集中骂某类问题（节奏/角色/感情线/套路）
- 每卷结束后 3 天内
- 用户主动要求"看看读者反馈"

## 工作流程

### 第一步：收集（与用户互动）

**一次性确认以下信息**：

1. **反馈来源**：评论/追读数据/书评/段评/其他平台反馈？
2. **范围**：最近几章？还是特定章节？
3. **重点**：节奏/角色/感情线/设定/其他？

**执行收集**（按 `references/评论收集规范.md`）：

1. 记录评论（分类：夸/骂/建议/讨论/催更）
2. 记录数据（追读率/读完率/弃书率）
3. 记录关键词（高频出现的词/句）
4. 更新 `90-运行/读者反馈/` 目录

### 第二步：追踪（周期性执行）

**按 `references/历史反馈追踪表.md` 更新**：

1. 每章发布后：记录评论数、情感分数、关键词
2. 每周：汇总数据，生成趋势曲线
3. 每卷：分析卷级反馈，更新读者画像

**产出**：
- `90-运行/读者反馈/章节反馈日志.md`
- `90-运行/读者反馈/周度反馈汇总.md`
- `90-运行/读者反馈/卷级反馈分析.md`

### 第三步：分析（按需执行）

**按 `references/评论情感分析.md` + `references/趋势识别指南.md`**：

1. 从评论提炼情绪（愤怒/失望/期待/喜爱/困惑）
2. 识别趋势模式（稳/升/降/波动/断崖）
3. 归因分析（这是单章问题/节奏型问题/卷方向问题？）

**产出**：
- 问题根因（不是症状）
- 趋势判断（上升/下降/拐点）
- 读者画像更新建议

### 第四步：决策（核心步骤）

**按 `references/优化方向决策.md`**：

1. 根据根因决定：改哪几章？改什么？怎么改？
2. 评估影响：改后追读预期提升多少？
3. 制定验证计划：改后多久看数据？

**产出**：
- 具体改写指令（章节/段落/技法）
- 预期效果
- 验证方式

### 第五步：输出指令（跨 skill）

**按 `references/跨-skill-协作接口.md`**：

把反馈转化为标准化指令，输出到对应 skill：

| 反馈显示…… | 输出到 | 输出内容 |
|-----------|--------|---------|
| 追读率断崖 | `novel-outline` | "第 X 卷节奏型需调整，建议增加小高潮" |
| 骂"节奏慢" | `novel-writing` | "ChXX 推进段需加冲突，具体：……" |
| 喜欢某角色 | `novel-worldbuilding` | "强化角色 A 弧线，增加独立目标" |
| 市场口味变化 | `novel-market` | "调整题材权重，增加 XX 元素" |
| 伏笔讨论多 | `novel-outline` | "提前回收或增加提醒" |
| 感情线争议 | `novel-outline` | "调整感情线节奏型" |

### 第六步：验证（执行后）

**按 `references/反馈驱动写作调整.md`**：

1. 改完后自检：是否引入新问题？
2. 发布后监控：追读率是否回升？评论是否改善？
3. 记录日志：`90-运行/决策记录.md`

### 第七步：提炼训练目标（每 5 章或卷末）

只有同一问题在多章正文和反馈中重复出现时，才写入 `90-运行/写作训练日志.md`。一次只保留一个训练目标，附修改前后证据；不要把读者偏好直接等同于作者能力缺陷。

## 产物要求

完成反馈闭环后，至少应生成：

- `90-运行/读者反馈/章节反馈日志.md`
- `90-运行/读者反馈/周度反馈汇总.md`
- `90-运行/读者反馈/卷级反馈分析.md`
- `90-运行/决策记录.md`（反馈改写决策条目）
- 向对应 skill 的输出指令

## 常见反馈错误

| 错误 | 表现 | 修复 |
|------|------|------|
| **凭感觉判断** | "读者好像不喜欢这段"——没有数据支撑 | 用追踪表记录具体评论数和情感分数 |
| **被极端评论带偏** | 1 个读者骂了就改 | 看数据：追读率是否真跌了？评论是否集中？ |
| **只骂不改** | "读者骂节奏慢"但不知道怎么改 | 用"优化方向决策"找到具体改写指令 |
| **改了不验证** | 改完后不跟踪数据变化 | 发布后必须监控追读率 |
| **过度迎合** | 读者骂什么改什么，失去作者立场 | 反馈是信号不是命令——先归因再决策 |
| **忽略正面反馈** | 只看骂的，不看夸的 | 记录"什么做对了"同样重要 |
| **不更新读者画像** | 早期读者假设一成不变 | 根据反馈定期更新读者画像 |

## 快速检查（5 分钟反馈体检）

回答这 5 个问题：

1. 最近 3 章的追读率趋势是什么？（升/降/平）
2. 最近 20 条评论中，夸和骂的比例是多少？
3. 被骂最集中的 1 个问题是什么？
4. 这个问题影响的章节范围是什么？
5. 如果改这个问题，下一章应该怎么写？

**评分**：
- 能回答 5 题：反馈追踪健康
- 能回答 3-4 题：需要补强追踪
- 能回答 <3 题：需要回到收集规范重新开始

## 输出要求

- 反馈产出结构化数据（不是"读者好像不喜欢"）
- 分析产出可执行的改写指令（不是"节奏慢，改快一点"）
- 指令必须指定：改哪章/改什么/怎么改/预期效果/验证方式
- 保持简洁，不展开与当前反馈无关的历史分析
