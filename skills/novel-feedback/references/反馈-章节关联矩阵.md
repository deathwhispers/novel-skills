# 反馈-章节关联矩阵

> **用途**：建立"哪条反馈影响哪章"的映射关系，避免反馈散乱无章。
> 适用于 novel-feedback 第二步-第四步。

---

## 一、矩阵结构

```markdown
# 反馈-章节关联矩阵

## 横向 = 章节
## 纵向 = 反馈类型
## 格内 = 影响强度（★=弱，★★=中，★★★=强）

| 反馈类型 \ 章节 | Ch40 | Ch41 | Ch42 | Ch43 | Ch44 | Ch45 |
|----------------|------|------|------|------|------|------|
| 节奏慢 | | | ★★ | ★★★ | ★★ | |
| 角色降智 | | ★ | | ★★ | | |
| 感情线拖 | ★★ | ★★ | | | | ★ |
| 设定崩 | | | | | ★ | |
| 章尾弱 | ★ | | ★ | | | ★★ |
```

---

## 二、矩阵用法

### 用法 1：定位问题源

看哪一列的 ★ 最多：
- Ch43 有 ★★★（节奏慢）+ ★★（角色降智）→ Ch43 是问题核心

### 用法 2：识别连锁影响

看哪一行跨多列：
- "节奏慢"影响 Ch42-Ch44 → 这不是单章问题，是节奏型问题

### 用法 3：验证修复效果

修复后，对应行的 ★ 应该减少：
- 改完 Ch43 后，"节奏慢"在 Ch43 的 ★★★ 变成 ★ → 修复有效

---

## 三、快速建立矩阵

**第一步：列出最近 10 章**

**第二步：收集反馈，按类型归类**

**第三步：填矩阵**
- 凭感觉判断每条反馈和每章的关联强度
- 不是精确科学，是帮助理清思路

**第四步：找最密集的行列交叉点**
- 最密集的列 = 问题最严重的章
- 最密集的行 = 最需要解决的问题类型

---

## 四、常见关联模式

| 模式 | 表现 | 含义 | 修复 |
|------|------|------|------|
| **单章集中** | 一列 ★ 很多 | 这章写法有问题 | 修复单章 |
| **单行扩散** | 一行跨多列 | 这类问题贯穿多章 | 调整节奏型/技法 |
| **对角线扩散** | 从 ChN 开始，逐章恶化 | 问题在恶化 | 立即止损 |
| **孤立点** | 只有一格有 ★ | 偶发问题 | 记录，观察 |

---

## 快速检查（3 分钟）

1. 最近哪一章的反馈最密集？
2. 哪类反馈影响最多章节？
3. 修复最密集的点后，矩阵会如何变化？

**答不上来→需要建立矩阵**
