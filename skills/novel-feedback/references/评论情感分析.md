# 评论情感分析

> **用途**：从海量评论中快速提炼核心情绪和诉求。
> 适用于 novel-feedback 第三步：分析。

---

## 一、情感分类

每条评论标注主情感：

| 情感 | 分数 | 标志词 | 含义 | 优先级 |
|------|------|--------|------|--------|
| **狂喜** | +5 | "封神""神作""吹爆" | 极度满意 | 记录但不用改 |
| **喜爱** | +3 | "好看""喜欢""期待" | 满意 | 维持 |
| **认可** | +1 | "还行""不错""可以" | 中性偏正 | 关注 |
| **困惑** | -1 | "看不懂""迷糊""乱" | 信息释放问题 | P2 |
| **失望** | -3 | "水""拖""没意思" | 节奏/期待问题 | P1 |
| **愤怒** | -5 | "毒""烂""弃了" | 严重问题 | P0 |

---

## 二、情感计算

### 单章情感均分

```
单章情感均分 = Σ(每条评论的情感分数) / 评论总数
```

| 均分范围 | 含义 | 行动 |
|---------|------|------|
| > +2 | 读者非常喜欢 | 维持当前写法 |
| 0 到 +2 | 读者基本满意 | 微调优化 |
| -2 到 0 | 读者不满 | 需要关注 |
| < -2 | 读者非常不满 | 必须立即分析原因 |

---

## 三、诉求提炼

从评论中提取读者的核心诉求：

| 诉求类型 | 标志词 | 含义 | 处理方式 |
|---------|--------|------|---------|
| **求爽** | "打脸""装逼""爽" | 读者需要即时满足 | 检查爽点密度 |
| **求推进** | "快点""别拖""水" | 读者觉得节奏慢 | 检查推进段效率 |
| **求揭秘** | "真相""谜底""为什么" | 读者想知道答案 | 评估是否提前释放 |
| **求感情** | "甜""虐""CP" | 读者需要情感满足 | 检查感情线进度 |
| **求角色** | "主角""配角""工具人" | 读者对角色有意见 | 检查角色弧线 |
| **求设定** | "世界观""规则""体系" | 读者对设定有期待 | 检查设定释放节奏 |

---

## 四、快速分析流程

**第一步：收集 20 条最近评论（5 分钟）**

**第二步：标注情感（5 分钟）**
- 给每条评论打 -5 到 +5 分
- 标记关键词

**第三步：计算均分（1 分钟）**
- 均分 < -2 → 立即启动问题诊断
- 均分 -2 到 0 → 关注，准备优化
- 均分 > 0 → 维持，记录正面反馈

**第四步：提炼诉求（5 分钟）**
- 出现频率最高的 3 个诉求是什么？
- 这 3 个诉求对应写作的哪个维度？

---

## 五、常见分析错误

| 错误 | 表现 | 修复 |
|------|------|------|
| **只看极端** | 只看 +5 和 -5，忽略中间 | 中间评论往往最真实 |
| **不量化** | "好像读者不喜欢" | 必须计算均分 |
| **混淆诉求** | "读者骂节奏慢"以为是求爽 | 区分"求推进"和"求爽" |
| **忽略正面** | 只看骂的，不看夸的 | 记录"什么做对了" |
| **不关联章节** | 不知道评论对应哪章 | 每条必须关联章节 |

---

## 快速检查（5 分钟）

1. 最近 20 条评论的情感均分是多少？
2. 出现频率最高的 3 个关键词是什么？
3. 这些关键词对应什么诉求？（求爽/推进/揭秘/感情/角色/设定）

**任一题答不上来→需要重新按流程分析**
