> **使用说明**：本文件解决"读者反馈分析完了，然后怎么改"这个问题。
> 配 `读者反馈机制.md` 使用——反馈机制负责收集+分析，本文件负责分析结果→改写决策→执行。
> 适用于长篇连载 20 章以上、追读数据开始有参考价值的阶段。

## 一、核心链路：反馈 → 改写决策 4 步

```
第1步：收集分类 ──→ 第2步：关联章节 ──→ 第3步：归因分析 ──→ 第4步：改写决策
```

| 步骤 | 做什么 | 产出 |
|------|--------|------|
| 1 收集分类 | 把本周评论按类型分类（夸/骂/建议/讨论），把追读数据做成表 | 分类评论表 + 追读率曲线 |
| 2 关联章节 | 把每条骂/建议 关联到具体章节（"骂节奏慢 → Ch18-Ch20"） | 问题章节清单 |
| 3 归因分析 | 用 AI 或人工判断：这是单章问题、卷方向问题、还是写法问题？ | 问题根因（不是症状） |
| 4 改写决策 | 根据根因决定：回哪一步？改什么？怎么改？ | 具体改写指令 + 责任人 |

## 二、反馈 → 改写决策对照表

### 2.1 追读数据类

| 追读信号 | 问题类型 | 改写决策 | 回到哪一步 |
|----------|----------|----------|------------|
| 单章追读突降 < 50% | 单章写法问题 | 体检这章 + 改写该段 | Step 4 重写 |
| 连续 3 章追读下降 | 节奏型问题 | 排查近 10 章，看哪类问题最集中 | Step 5 质量检测 |
| 卷末追读大跌 | 卷末钩子不够强 | 重设计卷末钩子 | Step 3 补节拍卡 |
| 开卷追读低（每卷第 1-3 章） | 开卷承诺立不住 | 重审开卷节奏 | Step 2 重审卷纲 |
| 整体趋势平稳但略有下滑 | 读者疲劳 | 加读点/加感情线 | Step 3 补节拍卡 |
| 追读率在某一卷突然跌后再未恢复 | 整卷方向问题 | 卷体检 + 必要时重写高潮 | Step 2 卷纲重审 |

### 2.2 评论内容类

| 评论类型 | 典型评论 | 问题根因 | 改写决策 | 回到哪一步 |
|----------|----------|----------|----------|------------|
| 骂"节奏慢"/"水" | "这章在干嘛" | 推进段无实质冲突 | 加实质冲突/压缩铺垫 | Step 4 重写 |
| 骂"看不懂" | "谁是谁来着" | 新角色出场太快/没建立 | 加角色出场仪式 | Step 4 补写 |
| 骂"太套路" | "又来这套" | 用了和上章同结构 | 换核心技法 | Step 3 改节拍卡 |
| 骂"感情线拖" | "他俩啥时候在一起" | 感情线停在暧昧太久 | 感情线推进 | Step 3 补节拍卡 |
| 骂"感情线太快" | "怎么突然就" | 铺垫不足直接跳 | 加感情线过渡 | Step 4 补写 |
| 骂"反派太蠢" | "这也能上当" | 反派降智了 | 重写反派决策逻辑 | Step 4 重写 |
| 骂"主角圣母" | "换我早杀了" | 代价没写够 | 加主角内心代价 | Step 4 补写 |
| 骂"太监了"/弃书 | "挖了不填" | 伏笔埋了超 30 章没回收 | 加速伏笔回收 | Step 3 改节拍卡 |
| 夸"这段太燃了" | "看哭了" | — | 强化同类写法 | 记录到文风指南 |
| 夸"感情线好甜" | "kswl" | — | 感情线维持节奏 | 同上 |

## 三、改写决策 AI Prompt（4 个）

### Prompt A：追读跌了 → 找根因

```
我最近 N 章的追读率如下：

{{追读率表}}

请分析：
1. 追读率明显下降的拐点是哪一（第）章？
2. 下降前后的章节在写法上有什么变化？（节奏/冲突密度/感情线/信息量）
3. 推测根因是什么（节奏变慢/新角色太多/感情线停了/某类套路重复/其他）？
4. 这个根因是"单章问题"还是"结构性问题"？

输出：
- 拐点章节列表
- 根因判断（1-2 句话）
- 问题类型：单章 / 节奏型 / 卷方向 / 套路化
```

### Prompt B：追读跌了 → 具体改哪几章

```
【来自 Prompt A 的根因判断】
{{根因}}

【近 10 章的节拍卡摘要】
{{粘贴近 10 章节拍卡}}

【追读率】
{{追读率}}

请给出：
1. 最需要改写的 1-3 章（按影响度排序）
2. 每章的改写优先级（P0/P1）
3. 每章的改写方向（不是具体改什么，是"这一章的问题是哪类"）
   - 推进密度不够 → 加冲突
   - 章尾无钩 → 重设章尾
   - 套路重复 → 换核心技法
   - 节奏太平 → 加小高潮

【决策规则】
- 改 1-2 章比改 5 章更有效
- 先修拐点章
- 改 1 章后等下一章追读数据再决定是否继续改
```

### Prompt C：角色被骂 → 改写方向

```
读者集中反馈：
{{读者骂角色的具体评论}}

被骂的角色卡：
{{角色.md 内容}}

请分析：
1. 读者骂这个角色的什么？（工具人/太蠢/太圣/没弧线/太脸谱化/其他）
2. 这个角色目前的弧线状态（起点→终点是否清晰）
3. 在未来 3-5 章内，给这个角色加什么能让读者改观？（加独立目标/加代价/加内心矛盾/其他）
4. 具体加在第几章？

输出：
- 问题诊断（1-2 句）
- 改写方向（不是具体台词，是"这一类动作要加"）
- 建议加入的章号
```

### Prompt D：节奏被骂 → 改写位置

```
读者反馈："节奏慢"/"中段无聊"/"推进不够"

被骂的章节范围：Ch{{X}} - Ch{{Y}}

【这些章节的节拍卡摘要】
{{粘贴}}

请分析：
1. 是哪几章的哪个段（铺垫/推进/高潮）拖慢了？
2. 是"没事发生"（真的空转）还是"戏不推进"（有冲突但不够强）？
3. 具体哪个冲突/哪段对话/哪个事件可以升级或删减？

输出：
- 问题段落定位（章号 + 段落 + 类型）
- 改写指令（升级冲突 / 删铺垫 / 换场景 / 加意外）
- 预计效果（改后追读预期提升多少）
```

## 四、改写决策后的执行链路

得出改写决策后，按以下链路执行：

| 改写决策类型 | 执行链路 | 回到 skill |
|-------------|----------|------------|
| 改单章某段 | Prompt B/C/D → 修订该段 → 追读验证 | novel-quality |
| 改节拍卡（某章的戏剧任务/章尾钩子） | 更新节拍卡 → 重写该章 | novel-outline → novel-writing |
| 改卷方向/加读点 | 卷体检 → 更新卷纲 → 补节拍卡 | novel-outline |
| 改全书套路化 | AI 提示"哪类写法重复" → 更新文风指南禁用列表 | novel-quality → novel-writing |
| 改主角弧线 | 更新角色卡弧线方向 → 在未来 5 章内补 | novel-worldbuilding → novel-outline |

## 五、决策执行后的验证

改写不是改完就结束。执行改写决策后，必须验证：

### 验证步骤

1. **改完后先自检**：
   - 这一改有没有引入新的问题（P0 新增）？
   - 有没有破坏前后章节的连续性？

2. **下一章发布后监控追读**：
   - 追读率回升 → 决策正确，继续
   - 追读率继续跌 → 根因判断错了，回 Prompt A 重新分析

3. **记录决策日志**：
   每次改写决策执行后，在 `90-运行/决策记录.md` 记录：
   ```
   ## [日期] 读者反馈改写决策
   
   触发：{{追读跌/读者骂什么}}
   根因：{{来自 Prompt A/B/C/D}}
   决策：{{回到哪步/改什么}}
   执行：{{已执行}}
   验证：{{追读率是否回升}}
   ```

## 六、触发条件与执行频率

| 触发条件 | 执行频率 | 做什么 |
|----------|----------|--------|
| 单章追读 < 50% | 立即（下一章发布前） | Prompt A + B |
| 连续 2 章追读跌 | 本周内 | Prompt A + B |
| 读者集中骂某类问题 | 3 天内 | Prompt C 或 D |
| 每卷结束后 | 卷末 3 天内 | 全卷反馈分析 + Prompt A + 决策 |
| 追读率持续下滑 10+ 章 | 月底 | 全面反馈+写法复盘 |

## 七、决策纪律

> **反馈是信号，不是命令。改写决策必须经过"归因"这一步。**

| 纪律 | 说明 |
|------|------|
| 不能读者骂什么改什么 | 先归因，找到根因再决定改哪里 |
| 不能一次改超过 3 章 | 改 1-2 章，等数据验证再决定下一轮 |
| 追读涨了要记为什么 | 同样重要——知道自己什么做对了 |
| 套路被夸不要立刻重复用 | 立刻重复 = 下一个套路化 |

## 九、与重构流程.md 的关系

读者反馈改写决策**不是重构流程的直线路径**，而是**旁路触发**：

```
可裁剪的写作直路
     ↓
发布 N 章
     ↓
触发追读监控 ──→ 读者反馈改写决策旁路
     ↓              ↓
追读率跌？       Prompt A 分析根因
     ↓              ↓
回到对应 Step   Prompt B/C/D 出改写指令
                 ↓
             执行链路（回 Step 3/4）
```

触发条件：
- 单章追读 < 50%
- 连续 2 章追读跌
- 读者集中骂某类问题

不触发条件：
- 追读平稳（不需要改）
- 早期章节（样本太少，数据不稳）

旁路完成后，**回到重构流程的当前步**，继续写下一章。
