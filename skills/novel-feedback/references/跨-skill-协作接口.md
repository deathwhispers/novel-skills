## 一、输出指令格式

所有向其他 skill 输出的指令必须使用以下格式：

```markdown
## 反馈指令 #FB-{{日期}}-{{序号}}

### 来源
- 触发：{{追读跌/读者骂/用户请求}}
- 数据支撑：{{具体数据}}
- 分析文件：{{引用本 skill 的分析文件}}

### 问题诊断
- 问题类型：{{节奏/角色/感情/设定/其他}}
- 影响范围：{{章节/卷/全书}}
- 严重程度：{{P0/P1/P2/P3}}
- 根因：{{一句话说明}}

### 输出指令
- 目标 skill：{{novel-outline/novel-writing/novel-worldbuilding/novel-market}}
- 指令内容：{{具体、可执行的写作/规划指令}}
- 预期效果：{{读者会有什么反应/数据会有什么变化}}
- 验证方式：{{发布后多久检查/检查什么指标}}

### 关联文件
- 反馈分析：{{novel-feedback/references/XX.md}}
- 需参考：{{其他 skill 的 references/XX.md}}
```

## 二、向各 skill 的输出示例

### 向 novel-outline 输出

```markdown
## 反馈指令 #FB-2026-0713-001

### 来源
- 触发：卷 3 追读率从 75% 跌至 55%
- 数据支撑：Ch80-Ch90 连续下降，评论"节奏慢"

### 问题诊断
- 问题类型：节奏
- 影响范围：卷 3 中段（Ch80-Ch90）
- 严重程度：P1
- 根因：推进段缺乏实质冲突，连续 10 章信息密度低

### 输出指令
- 目标 skill：novel-outline
- 指令内容：卷 4 必须从"标准型"切换为"波浪型"，在第 4 章和第 8 章各加一个小高潮
- 预期效果：追读率回升到 65%+
- 验证方式：卷 4 发布后看 Ch4/Ch8 的追读率

### 关联文件
- 反馈分析：novel-feedback/references/趋势识别指南.md
- 需参考：novel-outline/references/卷级节奏型模板.md
```

### 向 novel-writing 输出

```markdown
## 反馈指令 #FB-2026-0713-002

### 来源
- 触发：Ch95 评论集中骂"反派太蠢"

### 问题诊断
- 问题类型：角色
- 影响范围：Ch95
- 严重程度：P0
- 根因：反派决策没有自己的逻辑，纯为剧情服务

### 输出指令
- 目标 skill：novel-writing
- 指令内容：Ch95 重写反派段落，增加反派的"信息差"——他不知道主角的真实实力，他的决策基于错误信息
- 预期效果：读者理解反派行为，不再骂"蠢"
- 验证方式：Ch95 修改后看新评论

### 关联文件
- 反馈分析：novel-feedback/references/评论情感分析.md
- 需参考：novel-writing/references/写作技法库/对话写作技法.md
```

### 向 novel-worldbuilding 输出

```markdown
## 反馈指令 #FB-2026-0713-003

### 来源
- 触发：读者频繁讨论"这个设定怎么用"

### 问题诊断
- 问题类型：设定
- 影响范围：中期设定释放
- 严重程度：P2
- 根因：设定释放节奏不当，读者记不住规则

### 输出指令
- 目标 skill：novel-worldbuilding
- 指令内容：在 hard canon 中增加"设定使用场景"条目，每 5 章用 1 次已建立的设定
- 预期效果：读者熟悉设定，讨论从"怎么用"变成"接下来怎么发展"
- 验证方式：看后续评论中设定相关问题的比例

### 关联文件
- 反馈分析：novel-feedback/references/历史反馈追踪表.md
- 需参考：novel-worldbuilding/references/设定使用指南.md
```

### 向 novel-market 输出

```markdown
## 反馈指令 #FB-2026-0713-004

### 来源
- 触发：评论大量出现"和 XX 书好像"

### 问题诊断
- 问题类型：市场差异化
- 影响范围：全书定位
- 严重程度：P1
- 根因：和当前爆款同质化，缺乏独特卖点

### 输出指令
- 目标 skill：novel-market
- 指令内容：重新评估本书的差异化维度，至少改变 1 个维度（题材/类型/风格）
- 预期效果：读者评论从"和 XX 像"变成"和 XX 不一样"
- 验证方式：看后续评论中的比较类评论比例

### 关联文件
- 反馈分析：novel-feedback/references/评论情感分析.md
- 需参考：novel-market/references/形式矩阵.md
```

## 三、指令消费规范（其他 skill 必读）

收到 `novel-feedback` 的指令后：

1. **必须阅读**关联文件中的反馈分析
2. **必须评估**指令的可行性（是否和自己的规划冲突）
3. **必须反馈**执行结果（改了/没改/为什么）
4. **必须验证**效果（数据是否改善）

---

## 四、协作纪律

1. **novel-feedback 不说"怎么写"**——只说"写什么方向"
2. **其他 skill 不说"反馈不重要"**——必须评估反馈的合理性
3. **冲突时以数据为准**——追读率 > 个人判断
4. **所有指令必须闭环**——发出→执行→验证→记录

---

## 快速检查（2 分钟）

1. 反馈分析完后，应该向哪个 skill 输出指令？
2. 指令中是否包含：改什么/怎么改/预期效果/验证方式？
3. 目标 skill 收到指令后，是否知道该参考哪些文件？

**任一题"否"→指令不合格，需要重写**
