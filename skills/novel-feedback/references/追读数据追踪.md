# 追读数据追踪

> **用途**：标准化记录追读率、读完率、弃书率，建立数据趋势。
> 适用于 novel-feedback 第二步：追踪。

---

## 一、核心指标定义

| 指标 | 定义 | 计算方式 | 健康阈值 |
|------|------|---------|---------|
| **追读率** | 上一章读者中继续看下章的比例 | 下章阅读人数 / 上章阅读人数 | > 50% 合格，> 70% 优秀 |
| **读完率** | 看完本章全部内容的读者比例 | 完读人数 / 点击人数 | > 60% 合格 |
| **弃书率** | 本章后不再阅读的读者比例 | 弃书人数 / 上章阅读人数 | < 20% 合格，> 30% 危险 |
| **回流率** | 弃书后再次阅读的读者比例 | 回流人数 / 弃书人数 | 越高越好（表示内容吸引力） |

---

## 二、数据记录表

```markdown
## 追读数据追踪表

| 章节 | 发布日期 | 点击 | 完读 | 追读率 | 读完率 | 弃书率 | 周环比 | 趋势 |
|------|----------|------|------|--------|--------|--------|--------|------|
| Ch 1 | | | | | | | — | — |
| Ch 2 | | | | | | | ↑/↓/→ | |
| ... | | | | | | | | |

### 趋势标注
- ↑ 上升（比上周高 > 5%）
- ↓ 下降（比上周低 > 5%）
- → 平稳（±5% 内）
- ⚠️ 异常（波动 > 10%）
```

---

## 三、数据异常诊断

| 异常模式 | 可能原因 | 检查项 |
|---------|---------|--------|
| **单章追读突降** | 该章写法问题 | 章尾钩子/冲突密度/voice |
| **连续下降** | 节奏型问题 | 近 10 章推进密度/钩子分布 |
| **卷末大跌** | 卷末钩子弱 | 卷末承诺/兑现/新钩子 |
| **开卷就低** | 书名/简介/开篇问题 | 商业化检查 |
| **整体缓慢下滑** | 读者疲劳/套路重复 | 卷级节奏/读点分布 |
| **断崖式下跌** | 重大剧情问题/更新问题 | 对应章节内容/更新状态 |

---

## 四、数据与评论关联

把数据异常和评论关联：

```markdown
| 数据异常 | 对应评论 | 关联分析 |
|---------|---------|---------|
| Ch 45 追读从 70%→45% | "这几章水""节奏慢" | 推进段缺乏实质冲突 |
| Ch 50 追读回升到 75% | "终于爽了""打脸爽" | 小高潮兑现了期待 |
```
