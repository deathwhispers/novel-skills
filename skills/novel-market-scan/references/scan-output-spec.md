# Scan Output Spec

当执行扫榜、平台比较、选题验证时，优先输出成以下结构。

## A. `trend-notes.md`

```md
# Trend Notes

## Scope

- Platform:
- Time window:
- Query goal:

## Repeating Signals

- Signal 1:
- Signal 2:
- Signal 3:

## Opportunity Directions

### Direction 1
- Best for:
- Core promise:
- Why now:
- Risks:
- Better as: 长篇 / 短篇

### Direction 2
...

## What To Avoid

- Overheated pattern 1:
- Overheated pattern 2:

## Next Move

- Hand off to:
- Why:
```

## B. `benchmark-list.md`

```md
# Benchmark List

## Primary Benchmarks

| Work | Platform | Why it matters | Learn from |
|------|----------|----------------|------------|
| A | 起点 | | |
| B | 番茄 | | |

## Watchlist

- TBD
```

## 使用规则

- 如果是“全榜概览”，重点写 `Repeating Signals` 和 `Opportunity Directions`
- 如果是“验证一个选题”，重点写该方向是否成立、成立在哪个平台、应做哪些改造
- 如果是“平台对比”，要显式对比书名、简介、钩子、节奏和长短篇适配
