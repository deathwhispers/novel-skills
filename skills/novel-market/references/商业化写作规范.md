# 商业化写作规范

> **使用说明**：把作品包装成"读者愿意点击、愿意追读、愿意付费"的形态。
> 不是"骗人"——是"准确传达作品价值"。
> 适用于 novel-market 第三步：商业化包装。

## 商业化四件套

```
书名 → 简介 → 开篇 → 平台适配
  ↓      ↓       ↓         ↓
让读者点击 → 让读者期待 → 让读者追读 → 让读者付费
```

---

## 一、书名优化

### 书名的功能

| 功能 | 说明 | 比重 |
|------|------|------|
| 点击吸引 | 让读者在榜单中点击 | 40% |
| 题材信号 | 让读者知道这是什么类型 | 30% |
| 卖点暗示 | 让读者知道这书有什么 | 20% |
| 记忆传播 | 让读者能记住、能传播 | 10% |

### 书名结构公式

```
[核心信号] + [卖点暗示]
```

**核心信号**（题材/类型）：
- 修仙、玄幻、都市、重生、系统、种田、权谋、言情
- 用 1-2 个字点明

**卖点暗示**（独特价值）：
- 反差（废材→最强）
- 悬念（身份揭秘）
- 情绪（虐/甜/燃）
- 功能（签到/直播/模拟）

### 书名类型

| 类型 | 结构 | 示例 | 适用 |
|------|------|------|------|
| 反差型 | {{弱}}→{{强}} | 《废材逆袭：我是龙族太子》 | 凡人流、废材流 |
| 悬念型 | {{身份/秘密}}+{{暗示}} | 《我有一座冒险屋》 | 悬疑、无限流 |
| 功能型 | {{金手指}}+{{效果}} | 《开局签到荒古圣体》 | 系统流、签到流 |
| 情绪型 | {{情绪词}}+{{场景}} | 《万古神帝》 | 玄幻、燃向 |
| 直白型 | {{主角}}+{{做什么}} | 《大奉打更人》 | 历史、探案 |

### 书名自检

- [ ] 看到书名能猜到题材吗？
- [ ] 书名有"反差"或"悬念"吗？
- [ ] 书名在榜单中和同类书比有辨识度吗？
- [ ] 书名能缩写成 2-4 字让读者记住吗？
- [ ] 书名没有"过时感"（避免 5 年前爆款用过的结构）

### 书名备选清单

至少准备 3 个备选：

| 编号 | 书名 | 类型 | 核心信号 | 卖点暗示 | 自检通过 |
|------|------|------|----------|----------|----------|
| 1 | 《{{}}》 | {{}} | {{}} | {{}} | [ ] |
| 2 | 《{{}}》 | {{}} | {{}} | {{}} | [ ] |
| 3 | 《{{}}》 | {{}} | {{}} | {{}} | [ ] |

> 最终书名建议通过小范围测试（发给 3-5 个目标读者，看哪个更想点击）。

## 二、简介优化

### 简介的功能

让读者在 30 秒内决定"这本书值不值得追"。

### 简介结构公式

```
[钩子] → [主角状态] → [核心冲突] → [升级暗示] → [悬念/承诺]
```

**1. 钩子（1-2 句）**

- 用反常/悬念/冲突开场
- 不要"主角叫XXX，是一个..."

**示例**：
```
❌ 林尘是一个普通的杂役弟子，意外获得龙魂传承。
✅ 林尘被宗门追杀的那天，才发现自己体内沉睡着上古龙魂。
```

**2. 主角状态（1 句）**

- 主角现在的处境（不是"主角是谁"，是"主角面临什么"）

**3. 核心冲突（1-2 句）**

- 主角要做什么 + 被什么阻止

**4. 升级暗示（1 句）**

- 暗示主角会怎么变强/成长（让读者有期待）

**5. 悬念/承诺（1 句）**

- 给读者一个"必须要看"的理由

### 简介自检

- [ ] 前 2 句有钩子（不是平淡介绍）
- [ ] 有明确的冲突（不是"主角很厉害"）
- [ ] 有升级暗示（让读者期待变化）
- [ ] 有悬念或承诺（让读者想追）
- [ ] 字数在 100-200 字（不要太长）
- [ ] 没有剧透关键反转
- [ ] 没有"本书讲述了..."这种空话

### 简介示例

```markdown
# 简介示例（反差型）

林尘被宗门追杀的那天，才发现自己体内沉睡着上古龙魂。

从人人可欺的杂役，到被全宗追杀的"叛徒"，
林尘只用了一天。

但真正的麻烦不是追杀——
是他发现，宗门追杀他的理由，根本不是"偷学功法"那么简单。

一个关于龙族覆灭的千年阴谋，
一场涉及三界存亡的惊天棋局，
而林尘，正是那枚被隐藏了千年的棋子。

本书又名：《从杂役到龙族太子，我只用了一天》
```

## 三、开篇优化（黄金三章）

### 开篇的核心任务

前 3 章必须完成：

| 任务 | 第几章 | 说明 |
|------|--------|------|
| 建立主角 | Ch 1 | 让读者喜欢/同情/好奇主角 |
| 建立冲突 | Ch 1-2 | 让读者知道"核心矛盾是什么" |
| 建立期待 | Ch 2-3 | 让读者期待"接下来会发生什么" |
| 建立风格 | Ch 1-3 | 让读者知道"这是什么类型的书" |

### Ch 1 设计

**Ch 1 必须在 2000-2500 字内完成**：

- [ ] 主角登场（有动作，不是"介绍"）
- [ ] 反常事件发生（不是"平凡的一天"）
- [ ] 核心压力出现（主角被迫行动）
- [ ] 章尾有钩子（让读者必须看 Ch 2）

**Ch 1 自检**：

- [ ] 前 200 字有"反常"（不是"主角起床吃饭"）
- [ ] 主角在"做"什么（不是"在想"什么）
- [ ] 出现了至少 1 个"冲突信号"
- [ ] 章尾不是"平静收束"

### Ch 2 设计

**Ch 2 必须**：

- [ ] 延续 Ch 1 的压力（不是"新开始"）
- [ ] 展示主角的"反应方式"（让读者了解主角性格）
- [ ] 引入 1-2 个关键配角或关键设定
- [ ] 压力升级（比 Ch 1 更大/更复杂）

### Ch 3 设计

**Ch 3 必须**：

- [ ] 让主角做出 1 个"有后果的选择"
- [ ] 展示主角的"能力/特质"（让读者知道主角有什么）
- [ ] 引入更大的世界观/势力/规则
- [ ] 章尾抛出"中期悬念"（让读者想知道"这事最后怎么解决"）

### 开篇常见错误

| 错误 | 表现 | 修复 |
|------|------|------|
| 慢热 | Ch 1 全是背景和设定 | 从反常事件开始，设定随剧情释放 |
| 平铺 | Ch 1-3 节奏一样 | Ch 1 紧凑，Ch 2 升级，Ch 3 转折 |
| 无冲突 | 主角顺风顺水 | Ch 1 必须有压力/冲突/反常 |
| 介绍腔 | "这个世界有 XX 种族" | 用主角的遭遇展示世界，不是介绍 |
| 万能主角 | 主角什么都懂/什么都强 | Ch 1 展示弱点或局限 |

## 四、平台适配

### 平台差异

| 平台 | 读者特点 | 书名偏好 | 简介偏好 | 开篇偏好 | 节奏偏好 |
|------|----------|----------|----------|----------|----------|
| 起点 | 深度读者、喜欢复杂 | 直白/有信息量 | 详细、有世界观 | 可稍慢、重设定 | 可复杂、多线 |
| 番茄 | 快节奏、碎片化 | 反差/悬念 | 短、有钩子 | 快、有爽点 | 快、爽点密集 |
| 晋江 | 感情向、CP 导向 | 文艺/情绪 | 重感情线 | 重人物互动 | 感情线为主线 |
| 飞卢 | 脑洞向、猎奇 | 功能/反差 | 直白、有卖点 | 脑洞展示 | 快、脑洞密集 |

### 适配检查清单

根据目标平台调整：

**起点适配**：
- [ ] 书名有足够信息量
- [ ] 简介包含世界观暗示
- [ ] 开篇可包含适量设定
- [ ] 节奏可稍慢（Ch 3 内必须有爽点）

**番茄适配**：
- [ ] 书名有强烈反差或悬念
- [ ] 简介 100 字内必须有钩子
- [ ] Ch 1 必须有爽点或悬念
- [ ] 每 3 章必须有 1 个强钩子

**晋江适配**：
- [ ] 书名有情绪或 CP 暗示
- [ ] 简介突出感情线
- [ ] Ch 1-2 内 CP 双方出场
- [ ] 感情线节奏清晰

**多平台发布**：
- [ ] 书名：选最通用的版本
- [ ] 简介：准备 2-3 个版本（不同平台不同侧重）
- [ ] 开篇：按主平台优化，其他平台微调

## 五、追读优化

### 追读点布局

前 30 章的追读点分布：

| 位置 | 追读点类型 | 作用 |
|------|------------|------|
| Ch 1 章尾 | 悬念钩子 | 让读者看 Ch 2 |
| Ch 3 章尾 | 小爽点/小转折 | 确认"这本书有货" |
| Ch 5-7 | 信息释放 | 让读者想知道"真相" |
| Ch 10 | 小高潮 | 第一次大满足 |
| Ch 15 | 关系推进/代价支付 | 感情投入 |
| Ch 20 | 中高潮 | 中期满足 |
| Ch 25-30 | 卷末钩子 | 必须看下一卷 |

### 追读承诺

让读者知道"追下去能看到什么"：

```
读者追这本书，是因为想看到：
1. {{某个悬念的解答}}
2. {{主角的某个成长}}
3. {{CP 的某个结果}}
4. {{某个反派的结局}}
```

每 10 章检查一次：这些承诺还在推进吗？
