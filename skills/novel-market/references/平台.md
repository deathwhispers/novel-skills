## 起点

### 读者画像

| 维度 | 特点 |
|------|------|
| 年龄 | 25-35 岁为主 |
| 性别 | 男性 80%+ |
| 阅读经验 | 资深读者，看过大量网文 |
| 付费意愿 | 高（愿意为正版付费） |
| 阅读场景 | 晚上/周末，较完整时间段 |
| 偏好 | 复杂剧情、深度设定、群像、权谋 |
| 反感 | 套路化、小白文、无脑爽 |

### 平台机制

| 机制 | 说明 | 对作者的影响 |
|------|------|--------------|
| 签约 | 编辑推荐制，需要编辑认可 | 门槛高，但签约后资源多 |
| 推荐 | 编辑推荐 + 数据排名 | 需要数据好才能上推荐 |
| 上架 | 一般 20-30 万字上架 | 免费期长，考验追读 |
| 收入 | 订阅 + 打赏 + 月票奖励 | 订阅单价高（约 0.05 元/千字） |
| 完本 | 完本后仍有长尾收入 | 鼓励完本，不鼓励太监 |

### 适配策略

**适合的书**：
- 世界观复杂、设定多的玄幻/仙侠
- 节奏可稍慢、重视铺垫的权谋/历史
- 群像刻画好的作品
- 字数预期 200 万+

**不适合的书**：
- 纯爽文、无脑打脸
- 节奏极快、每章必须爽
- 字数预期少于 100 万

**书名偏好**：
- 直白、有信息量
- 示例：《大奉打更人》《诡秘之主》《道君》

**简介偏好**：
- 可稍长（150-250 字）
- 可包含世界观暗示
- 重设定、重氛围

**开篇偏好**：
- 可稍慢（但 Ch 1 必须有反常事件）
- 允许适量设定介绍
- Ch 3 内必须有爽点或悬念升级

## 番茄

### 读者画像

| 维度 | 特点 |
|------|------|
| 年龄 | 18-30 岁为主 |
| 性别 | 男女混合，略偏男 |
| 阅读经验 | 中轻度读者，看过一些网文 |
| 付费意愿 | 低（习惯免费+广告） |
| 阅读场景 | 碎片化（通勤/午休/睡前） |
| 偏好 | 快节奏、爽点密集、脑洞、反转 |
| 反感 | 慢热、铺垫长、设定多 |

### 平台机制

| 机制 | 说明 | 对作者的影响 |
|------|------|--------------|
| 签约 | 算法审核，门槛低 | 容易签约，但竞争激烈 |
| 推荐 | 纯算法推荐 | 需要数据好（完读率、留存率） |
| 验证期 | 8-20 万字验证期 | 验证期数据决定后续推荐 |
| 收入 | 广告分成 | 单价低，靠量取胜 |
| 完读率 | 关键指标 | 完读率 > 15% 算合格，> 25% 算优秀 |

### 适配策略

**适合的书**：
- 节奏快、爽点密集的玄幻/都市
- 脑洞大、设定新颖的作品
- 感情线推进快的作品
- 字数预期 100-200 万

**不适合的书**：
- 慢热、铺垫长
- 设定复杂、需要读者记很多东西
- 群像过多、人物关系复杂
- 正剧向、缺乏爽点

**书名偏好**：
- 反差强烈、悬念感强
- 示例：《开局签到荒古圣体》《我有一座冒险屋》《震惊！开局和女帝成亲》

**简介偏好**：
- 必须短（100 字以内）
- 必须有强烈钩子
- 必须有爽点暗示

**开篇偏好**：
- Ch 1 必须有爽点或强烈悬念
- 节奏必须快（不能铺垫超过 500 字）
- 每 3 章必须有 1 个强钩子

## 晋江

### 读者画像

| 维度 | 特点 |
|------|------|
| 年龄 | 18-30 岁女性为主 |
| 性别 | 女性 90%+ |
| 阅读经验 | 中高度读者，对感情线敏感 |
| 付费意愿 | 高（愿意为正版付费） |
| 阅读场景 | 晚上/周末，较完整时间段 |
| 偏好 | 感情线、人物塑造、CP 互动、文笔 |
| 反感 | 无感情线、工具人女主、虐女 |

### 平台机制

| 机制 | 说明 | 对作者的影响 |
|------|------|--------------|
| 签约 | 编辑推荐 + 读者数据 | 门槛中等 |
| 推荐 | 编辑推荐 + 榜单 | 感情线数据好容易上推荐 |
| 收入 | 订阅 + 打赏 | 单价高（约 0.03 元/字） |
| 评论 | 读者互动活跃 | 需要管理评论区 |
| 完结 | 完本有长尾 | 鼓励完本 |

### 适配策略

**适合的书**：
- 感情线清晰、CP 感强的言情/耽美
- 人物塑造细腻、互动多的作品
- 有明确 CP 关系的作品
- 字数预期 30-100 万

**不适合的书**：
- 无 CP 或 CP 极弱
- 纯升级、无感情线
- 虐女主/工具人女主
- 字数超过 150 万

**书名偏好**：
- 文艺、有情绪感
- 示例：《天官赐福》《魔道祖师》《二哈和他的白猫师尊》

**简介偏好**：
- 突出感情线和 CP 关系
- 可包含人物关系暗示
- 情绪感强

**开篇偏好**：
- Ch 1-2 内 CP 双方出场
- 用互动展示人物性格
- 感情张力从开篇建立

## 飞卢

### 读者画像

| 维度 | 特点 |
|------|------|
| 年龄 | 18-25 岁为主 |
| 性别 | 男性为主 |
| 阅读经验 | 中度读者，喜欢新鲜感 |
| 付费意愿 | 中（订阅单价低） |
| 阅读场景 | 碎片化 |
| 偏好 | 脑洞、猎奇、反差、同人 |
| 反感 | 老套路、慢热、无新意 |

### 平台机制

| 机制 | 说明 | 对作者的影响 |
|------|------|--------------|
| 签约 | 门槛低 | 容易签约 |
| 推荐 | 更新驱动 | 更新快容易上推荐 |
| 收入 | 订阅 | 单价低，靠量 |
| 风格 | 极度追求脑洞 | 必须新颖 |

### 适配策略

**适合的书**：
- 脑洞极大、设定新颖
- 同人/二次元改编
- 节奏极快
- 字数预期 50-150 万

**不适合的书**：
- 传统玄幻/仙侠套路
- 慢热、铺垫长
- 正剧向

## 常见平台错误

| 错误 | 表现 | 修复 |
|------|------|------|
| **平台错配** | 起点风格发到番茄，或反之 | 用"平台选择决策树"重新选择 |
| **不了解机制** | 不知道番茄算法推荐逻辑 | 研究平台推荐机制，按机制优化 |
| **忽视读者差异** | 晋江读者和起点读者当成同一批人 | 按平台读者画像调整风格 |
| **盲目多平台** | 同时发 5 个平台，精力分散 | 先主攻 1 个平台，数据稳定后再扩展 |
| **只看收入** | "哪个平台给钱多去哪个" | 考虑题材适配度、竞争强度、长期发展 |
| **不监控数据** | 发了书就不管数据 | 按"数据监控频率"定期检查 |

## 平台选择决策树

```
你的题材是什么？
  ├─ 玄幻/仙侠/科幻 → 起点（深度读者）或 番茄（泛读者）
  ├─ 都市/脑洞/系统 → 番茄 或 飞卢
  ├─ 言情/耽美/同人 → 晋江
  ├─ 二次元/同人/轻小说 → 刺猬猫
  └─ 历史/权谋/种田 → 起点 或 17K

你的风格是什么？
  ├─ 快节奏/爽点密集 → 番茄/飞卢
  ├─ 慢节奏/重铺垫 → 起点/17K
  ├─ 感情线为主 → 晋江
  └─ 日常/轻松 → 刺猬猫/番茄

你的更新能力？
  ├─ 日更 6000+ → 番茄/飞卢
  ├─ 日更 4000+ → 起点
  └─ 日更 2000-3000 → 晋江/七猫

你的目标？
  ├─ 高收入+高门槛 → 起点
  ├─ 快速验证+算法推荐 → 番茄
  ├─ 感情线深耕 → 晋江
  └─ 脑洞/猎奇 → 飞卢
```

## 快速检查（2 分钟）

1. 你的题材在哪个平台最火？（前 30 名出现几本）
2. 你的风格匹配目标平台的读者偏好吗？
3. 你的更新频率能达到目标平台的推荐门槛吗？

**任一题"否"→考虑换平台或调整风格**
