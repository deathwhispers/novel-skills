# 卷纲模板

> **使用说明**：每卷一个文件，按本模板填写。不是"写卷的故事"，是"定义这卷在全书中扮演的角色"。
> 适用于重构流程 Step 2（分卷规划）。
> 文件名：`20-大纲/分卷/volume-XX.md`

## 卷信息头

```markdown
# 第 {{X}} 卷：{{卷名}}

- **卷号**：第 X 卷（全书共 {{N}} 卷）
- **卷名**：{{简明卷名，2-6 字}}
- **功能定位**：{{起势 / 发展 / 转折 / 深化 / 收束 / 其他}}（选 1 个）
- **核心卖点**：{{这卷主要卖什么：爽点 / 虐点 / 悬念 / 感情线 / 其他}}（选 1-2 个）
```

## 一、卷级定位

### 1.1 在全书中扮演什么角色

用 1-2 句话说清这卷在全书中独一无二的功能：

```
这卷是 {{全书阶段}} 的 {{具体功能}}——
它负责 {{建立/升级/转折/深化/收束}} {{什么}}，
为 {{下一卷}} 的 {{什么}} 做铺垫。
```

**示例**：
```
这卷是全书"起势期"的"立人设+埋主线"——
它负责建立主角的初始状态和被追杀的核心矛盾，
为第 2 卷的"逃亡+成长"做铺垫。
```

### 1.2 读者读完这卷应该有什么感受

| 感受类型 | 具体描述 |
|----------|----------|
| 爽 | 读者觉得"主角终于……" |
| 虐 | 读者觉得"太惨了，希望主角能……" |
| 悬 | 读者觉得"到底怎么回事，下一卷必须看" |
| 甜 | 读者觉得"这对 CP 我磕了" |
| 燃 | 读者觉得"热血，主角冲啊" |
| 其他 | {{自定义}} |

**不要写"多种感受"**——每卷聚焦 1-2 种主感受。

## 二、开卷设计

### 2.1 开卷承诺（第 1 章必须给读者的）

读者读完第 1 章后，应该明确知道：

- [ ] 主角当前的状态（位置/能力/目标）
- [ ] 本卷的核心悬念/压力是什么
- [ ] 读者追下去能看到什么（爽点/虐点/悬念）

**开卷承诺的具体写法**：

```
第 1 章结束时，读者应该知道：
1. 主角 {{现在在做什么 / 面临什么}} 
2. 接下来会发生 {{什么类型的事}}（战斗/揭秘/感情推进/其他）
3. 这件事的 {{悬念/压力/期待点}} 是什么
```

### 2.2 开卷钩子（第 1 章的章尾发动机）

从 6 种中选 1 种：

- [ ] 新问题：开卷抛出新的核心问题
- [ ] 不可逆决定：主角做了无法回头的决定
- [ ] 倒计时：某件事即将发生
- [ ] 权力反转：局势突然改变
- [ ] 情绪余震：强烈的情绪冲击
- [ ] 关系破口：重要关系出现裂缝

## 九、自检清单

卷纲完成后，逐项检查：

- [ ] 功能定位明确（一句话能说清这卷在全书中扮演什么）
- [ ] 核心卖点聚焦（不超过 2 种主感受）
- [ ] 开卷承诺具体（读者读完第 1 章知道要追什么）
- [ ] 卷末钩子强（不是"平静收束"）
- [ ] 核心转折不可逆（不是"差点发生"）
- [ ] 铺垫分布散（不集中在一两章）
- [ ] 升级有代价（不是白得的）
- [ ] 字数预算合理（高潮卷可长，过渡卷可短）
- [ ] 跨卷衔接顺（本卷末接下一卷首）
- [ ] 角色弧线有变化（不只是主角变强）

> **核心规则**：10 项中若有 3 项以上说不清，回到对应阶梯重审。
