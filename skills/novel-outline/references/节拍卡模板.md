# 节拍卡模板

> **使用说明**：每章一个文件，按本模板填写 10 项。
> 不是"写章纲"——是"定义这章的戏剧任务"。
> 适用于重构流程 Step 3（章节节拍卡）。
> 文件名：`20-大纲/节拍卡/chapter-XXX.md`

## 章节信息头

```markdown
# 第 {{XXX}} 章：{{章节名}}

- **章节号**：第 {{XXX}} 章
- **所属卷**：第 {{X}} 卷 - {{卷名}}
- **视角人物**：{{主角/配角名}}
- **预计字数**：{{2000-2500}} 字
```

---

## 10 项必填

### 1. 这章在全卷中扮演什么角色

这章属于卷的哪个阶段？

- [ ] 开卷（立承诺 / 建语境）
- [ ] 推进（升级矛盾 / 释放信息）
- [ ] 小高潮（局部冲突解决）
- [ ] 大高潮（卷核心转折）
- [ ] 收束（余震 / 留钩子）
- [ ] 过渡（喘息 / 埋线）

用 1 句话说明：
```
这章是 {{卷阶段}} 的 {{具体功能}}——
它让 {{什么}} 从 {{X}} 变成 {{Y}}。
```

---

### 2. 这章是谁的戏（视角人物）

- **视角人物**：{{姓名}}
- **为什么选这个视角**：{{这个视角知道什么 / 不知道什么，这个信息差有什么用}}
- **视角限制**：{{这个视角不能知道什么（留给其他视角或读者）}}

> **核心规则**：不是"方便叙述"就选——是"这个视角的信息差最有戏"。

---

### 3. 这章必须完成什么（核心戏剧任务）

用 1 句话说清，不含糊：

```
{{视角人物}} 必须 {{做什么动作}}，
因为 {{被什么逼的}}，
但 {{被什么卡住}}。
```

**合格标准**：
- [ ] 有明确的动作（找/抢/说服/逃/杀/保护/揭露）
- [ ] 有被什么逼的（不是"角色想这么做"）
- [ ] 有被什么卡住（不是"有点难"）

**典型不合格**：
- ❌ "主角在这一章推进了主线"（没有具体动作）
- ❌ "主角遇到了敌人"（没有被什么逼的）
- ❌ "主角和敌人战斗"（没有被什么卡住）

**合格示例**：
- ✅ "主角必须在长老会上证明自己不是凶手，因为被诬陷了，但诬陷他的人已经买通了长老。"

---

### 4. 这章是被前文什么结果逼出来的（因果锚点）

```
上一章（Ch {{XXX}}）的最后，{{发生了什么结果}}，
这个结果逼出了本章：{{视角人物}} 必须 {{做什么}}。
```

**因果链检查**：
- [ ] 本章不是"凭空开始"——是被上一章的结果逼出来的
- [ ] 本章不是"作者想让角色这么做"——是角色被迫这么做的

---

### 5. 真正卡住角色的阻力是什么

**阻力定义**：

```
角色想 {{做 A}}，但 {{B 阻止了他}}。
B 是：
- [ ] 具体的人（谁？为什么阻止？）
- [ ] 具体的规则（什么规则？违反的代价？）
- [ ] 具体的环境（什么环境？怎么限制？）
- [ ] 具体的信息差（角色不知道什么？）
```

**阻力强度检查**：
- [ ] 阻力是"真实的"——不是"空气阻力"
- [ ] 阻力有代价——角色突破阻力要付出什么
- [ ] 阻力会变形——不是同一阻力重复，是阻力随剧情升级

---

### 6. 这章局面怎么变了

章初和章末对比：

| 维度 | 章初 | 章末 | 变化 |
|------|------|------|------|
| 信息 | {{角色知道什么}} | {{角色知道了什么新信息}} | {{}} |
| 关系 | {{角色间关系}} | {{关系变成了什么}} | {{}} |
| 地位 | {{角色地位}} | {{地位变成了什么}} | {{}} |
| 局势 | {{当前局势}} | {{局势变成了什么}} | {{}} |

> **核心规则**：至少 1 个维度有不可逆变化。章末和章初必须"不一样"。

---

### 7. 情绪 / 关系怎么变了

**情绪变化**：

```
章初：{{角色情绪}}
章中：{{情绪因为什么事件变化}}
章末：{{角色情绪}}
```

**关系变化**（如有）：

```
关系对：{{角色 A}} ↔ {{角色 B}}
章初状态：{{}}
章末状态：{{}}
变化触发事件：{{}}
```

> **核心规则**：情绪不是"抽象词"（他很生气）——是"因为什么具体事件变的"。

---

### 8. 本章写完后必须记住什么（新增的信息债 / 关系债 / 物件债）

**信息债**：本章揭示了什么新信息，读者和角色知道了但还没消化：
```
新信息：{{}}
谁知道了：{{角色 / 读者 / 两者}}
什么时候会用到：{{计划在哪章回收}}
```

**关系债**：本章改变了什么关系，这个变化还没完全展开：
```
关系变化：{{}}
未解决的：{{}}
计划在哪章深化：{{}}
```

**物件债**：本章出现了什么重要物件，这个物件还没发挥全部作用：
```
物件：{{}}
当前状态：{{}}
计划在哪章回收：{{}}
```

> **核心规则**：每章至少新增 1 条"债"——没债的章是"空转"。

---

### 9. 章尾拿什么继续吊读者（章尾发动机）

从 6 种中选 1 种：

- [ ] **新问题**：抛出新的核心问题（"但赵长老的信里还提到了什么？"）
- [ ] **不可逆决定**：主角做了无法回头的决定（"苏晚握紧了剑——她做了一个决定。"）
- [ ] **倒计时**：某件事即将发生（"三天后的月圆之夜，封印就要破了。"）
- [ ] **权力反转**：局势反转（"李昭本想伏击主角——但他不知道，伏击他的人已经在身后。"）
- [ ] **情绪余震**：强烈的情绪震撼（"主角看着师父的墓，许久没有说话。"）
- [ ] **关系破口**：关系出现裂缝（"苏晚的眼睛里，第一次有了恨意。"）

**章尾的具体设计**：

```
最后 1-2 段写什么：
{{}}

读者读完会想什么：
{{}}

注意：不能是"平静收束"或"总结"。
```

---

### 10. 本章碰了哪几条埋线（旧线升级 / 新线埋下 / 信息回收 / 关系债）

**旧线升级**（本章让之前的伏笔/悬念发生了什么变化）：
```
旧线：{{之前埋的什么}}
本章怎么升级的：{{}}
```

**新线埋下**（本章埋了什么新伏笔）：
```
新线：{{}}
埋设方式：{{细节 / 行为 / 对话 / 环境}}
计划回收：{{哪章}}
```

**信息回收**（本章回收了什么之前的信息债）：
```
回收的信息：{{}}
原埋设章节：{{}}
回收方式：{{明示 / 暗示 / 翻案}}
```

**关系债**（本章推动了什么关系变化）：
```
关系对：{{}}
变化：{{}}
未解决的：{{}}
```

> **核心规则**：每章至少碰 1 条旧线或埋 1 条新线。连续 3 章都不碰线 = 追读会跌。

---

## 自检清单

10 项填完后，逐项检查：

- [ ] **1. 角色明确**：有明确的视角人物，不是"群像无主角"
- [ ] **2. 动作具体**：核心戏剧任务有"做什么"，不是"推进剧情"
- [ ] **3. 阻力真实**：被什么卡住是具体的人/规则/环境，不是"很难"
- [ ] **4. 因果清楚**：本章是被上一章逼出来的，不是凭空开始
- [ ] **5. 局面变了**：章末和章初至少 1 个维度不同
- [ ] **6. 情绪有因**：情绪变化有具体事件触发，不是"突然生气"
- [ ] **7. 有债可追**：至少新增 1 条信息债/关系债/物件债
- [ ] **8. 章尾有钩**：发动机从 6 种中选 1 种，不是"平静收束"
- [ ] **9. 有线可碰**：至少碰 1 条旧线或埋 1 条新线
- [ ] **10. 字数合理**：预计字数和卷字数预算匹配

> **核心规则**：10 项中若有 2 项以上说不清，不要开写——回到 `写前检查表.md` 补齐。

---

## 跨章衔接检查

本章节拍卡填完后，和前后章对比：

### 与上一章的衔接

- [ ] 上一章的章尾发动机，本章开头接住了吗？
- [ ] 上一章留下的角色状态，本章延续了吗？
- [ ] 上一章的悬念/压力，本章在推进吗？

### 与下一章的衔接

- [ ] 本章的章尾发动机，下一章能接住吗？
- [ ] 本章新增的信息债，下一章有计划碰吗？
- [ ] 本章的局面变化，下一章有后果吗？

> **核心规则**：单章的节拍卡合格 ≠ 跨章衔接合格。必须和前/后章放一起检查。
