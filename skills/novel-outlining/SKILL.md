# 铺线列纲

## 功能定位

把小说从“有点子”变成“能稳稳往下写”。这个 skill 负责搭主线、分卷、章节推进线和埋回线，让中文长篇常见的断层、灌水、失焦和回收失踪尽量在写前暴露。

## 大纲阶梯

按从粗到细的顺序工作：

1. 主线与收束方向
2. 分卷或阶段性推进
3. 每卷起止状态变化
4. 章节节拍卡
5. 伏笔、信息债、关系债的埋设与回收位置

短篇时可以压缩为：

1. 题眼
2. 关键推进
3. 转折
4. 收束或回响

## 章节 beat 最低要求

每一章至少要写清：

- 本章视角人物
- 本章要完成什么事
- 真正的拦路阻力是什么
- 这一章局面怎么变了
- 情绪或关系怎么变了
- 章尾拿什么吊住读者
- 本章写完后必须记住什么
- 本章碰到了哪几条埋线

如果目标是提高写作质量，还要把这些信息同步进两张表：

- `scene-causality-map.md`
- `payoff-ledger.md`

## 改纲原则

- 尽量做局部改纲，不轻易重置整本书。
- 改纲后明确指出哪些卷、哪些章节 beats 需要联动修改。
- 如果用户只想写下一章，不要把任务扩大成重列整本书。
- 长篇优先保留追更循环、升级感、关系线拉扯和阶段性回收，短篇优先保留题眼和结尾力量。
- 改纲时不要只改章节卡，要同步检查因果链表和伏笔回收表是否失效。
