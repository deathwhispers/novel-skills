# 项目初始化与立项

## 在重构流程中的位置

本 skill 承担重构流程的 **Step 0**（立项定核）：

| 步骤 | 在本 skill 的位置 | 转入下步条件 |
|------|------------------|--------------|
| Step 0 立项定核 | 第一步到第三步（项目初始化→立项定核→篇幅规划） | 立项单通过 + 故事核心能用 50 字说清 + 篇幅预算已确认 |

**前后衔接**：
- 准入：用户明确要"开一本书"或"初始化项目"
- 转出：Step 0 通过后进入 `novel-market`（如果需要市场分析）或 `novel-worldbuilding`（如果需要先建设定）
- 如果用户已有明确题材和市场认知，可直接进入 `novel-worldbuilding`

## 功能定位

把项目初始化、立项定核、篇幅规划整合成一个连贯的流程。这个 skill 负责：

1. **项目初始化**：创建小说工作区、统一目录结构
2. **立项定核**：明确故事核、读者对象、题材定位
3. **篇幅规划**：与用户互动确认小说篇幅、分卷数量、更新频率

## 工作流程

### 第一步：创建项目

使用初始化脚本创建项目：

```bash
python3 scripts/init_novel_project.py \
  --output <target-dir> \
  --title "<novel-title>" \
  --genre "<genre>" \
  --premise "<one-line-premise>"
```

### 第二步：立项定核（与用户互动）

**一次性确认以下信息**：

1. **故事核心**：请用一句话说清您的故事核
2. **目标读者**：这本书写给谁看？
3. **题材定位**：这本书的核心卖点是什么？
4. **新鲜感来源**：这本书有什么独特之处？

### 第三步：篇幅规划（与用户互动）

**一次性确认以下信息**：

1. **小说篇幅**：短篇/中篇/长篇/超长篇
2. **分卷数量**：计划分几卷
3. **每章字数**：每章写多少字
4. **更新频率**：日更/隔日更/周更/不固定
5. **确认总体预算**：生成总体预算并让用户确认

### 第四步：导入旧项目（可选）

如果用户有旧稿需要导入，按导入流程处理。

## 常见立项错误

| 错误 | 表现 | 修复 |
|------|------|------|
| **核心太泛** | "主角修仙成神"——没有具体的追求和阻力 | 用一句话模板：一个{{状态}}的{{主角}}，因为{{触发事件}}，必须在{{限制}}内{{追求}}，但{{阻力}}让这件事不断升级 |
| **读者不清** | "所有人都能看"——没有目标读者 | 明确年龄层+性别倾向+阅读偏好+阅读场景 |
| **卖点模糊** | "和别的书差不多"——没有独特之处 | 回答：读者选这本书而不是其他同类书，是因为什么？ |
| **篇幅不合理** | "我想写 10 万字的超长篇"——概念混淆 | 用篇幅规划指南确认：短篇(1-10万)/中篇(10-30万)/长篇(30-100万)/超长篇(100万+) |
| **更新频率太高** | "日更 1 万字"——不可持续 | 根据实际时间确认：日更/隔日更/周更，并给出合理字数建议 |
| **跳过立项** | "直接写第一章吧"——没有立项单 | 坚持先立项——没有立项单的项目=没有地基的房子 |

## 快速检查（5 分钟立项体检）

回答这 5 个问题，每个不超过 20 字：

1. 书名是什么？
2. 主角是谁？（名字 + 身份）
3. 他想要什么？（具体目标）
4. 什么在阻止他？（具体阻力）
5. 读者为什么选这本书？（独特卖点）

**评分**：
- 能回答 5 题：立项健康，可进入下一步
- 能回答 3-4 题：需要补强对应部分
- 能回答 <3 题：需要回到立项单模板重填
