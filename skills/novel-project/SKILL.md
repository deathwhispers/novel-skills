# 项目初始化与立项

## 功能定位

把项目初始化、立项定核、篇幅规划整合成一个连贯的流程。这个 skill 负责：

1. **项目初始化**：创建小说工作区、统一目录结构
2. **立项定核**：明确故事核、读者对象、题材定位
3. **篇幅规划**：与用户互动确认小说篇幅、分卷数量、更新频率

## 工作流程

### 第一步：创建项目

使用初始化脚本创建项目：

```bash
python3 scripts/init_novel_project.py \
  --output <target-dir> \
  --title "<novel-title>" \
  --genre "<genre>" \
  --premise "<one-line-premise>"
```

### 第二步：立项定核（与用户互动）

**一次性确认以下信息**：

1. **故事核心**：请用一句话说清您的故事核
2. **目标读者**：这本书写给谁看？
3. **题材定位**：这本书的核心卖点是什么？
4. **新鲜感来源**：这本书有什么独特之处？

### 第三步：篇幅规划（与用户互动）

**一次性确认以下信息**：

1. **小说篇幅**：短篇/中篇/长篇/超长篇
2. **分卷数量**：计划分几卷
3. **每章字数**：每章写多少字
4. **更新频率**：日更/隔日更/周更/不固定
5. **确认总体预算**：生成总体预算并让用户确认

### 第四步：导入旧项目（可选）

如果用户有旧稿需要导入，按导入流程处理。

## 产物要求

完成本流程后，至少应生成：

- `00-书核/作品总表.md`
- `00-书核/立项单.md`（包含完整的篇幅规划）
- `90-运行/当前进度.md`
