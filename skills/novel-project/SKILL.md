# 项目初始化与立项

## 在重构流程中的位置

本 skill 承担重构流程的 **Step 0**（立项定核）：

| 步骤 | 在本 skill 的位置 | 转入下步条件 |
|------|------------------|--------------|
| Step 0 立项定核 | 第一步到第三步（项目初始化→模式与书核→必要规划） | 已能支持当前任务；探索起草允许保留未决项 |

**前后衔接**：
- 准入：用户明确要"开一本书"或"初始化项目"
- 转出：Step 0 通过后进入 `novel-market`（如果需要市场分析）或 `novel-worldbuilding`（如果需要先建设定）
- 如果用户已有明确题材和市场认知，可直接进入 `novel-worldbuilding`

## 功能定位

把项目初始化、立项定核、篇幅规划整合成一个连贯的流程。这个 skill 负责：

1. **项目初始化**：创建小说工作区、统一目录结构
2. **立项定核**：明确故事核、读者对象、题材定位
3. **必要规划**：按写作模式决定是否需要篇幅、分卷、更新频率和平台约束

## 工作流程

### 第一步：创建项目

使用初始化脚本创建项目：

```bash
python3 scripts/init_novel_project.py \
  --output <target-dir> \
  --title "<novel-title>" \
  --genre "<genre>" \
  --premise "<one-line-premise>" \
  --profile serial
```

选择最小资料集用 `minimal`，常规连载用 `serial`，百万字、多主角或复杂势力项目用 `longform`。模板档位不等于写作模式；文学、短篇和探索项目通常从 `minimal` 开始。

### 第二步：立项定核

从用户已有信息先填，只有缺口会改变当前写作方向时才集中确认。探索起草的最小书核只有三项：

1. **写作模式**：商业连载 / 类型长篇 / 文学叙事 / 短篇 / 探索起草
2. **当前写作意图**：这次最想写成什么
3. **不可轻易丢失的东西**：人物声音、关系、谜团、体验、形式或 canon

目标读者、卖点和新鲜感在商业发行或作者明确需要定位时再补；私人创作不强行商业化表述。

### 第三步：篇幅规划（与用户互动）

只在当前模式和当前任务需要时确认：

1. **小说篇幅**：短篇/中篇/长篇/超长篇/暂不确定
2. **分卷数量**：计划分几卷
3. **每章字数**：每章写多少字
4. **更新频率**：日更/隔日更/周更/不固定
5. **总体预算**：数字影响交付或平台约束时再确认

### 第四步：导入旧项目（可选）

如果用户有旧稿需要导入，按导入流程处理。

## 产物要求

完成本流程后，生成支持当前任务的最小产物：

- `00-书核/作品总表.md`

## 常见立项错误

| 错误 | 表现 | 修复 |
|------|------|------|
| **意图太泛** | 只知道题材，不知道这次想探索什么 | 先确定人物、处境、关系、体验或形式中的一个抓手 |
| **过早商业化** | 私人试写也被要求填写平台和卖点 | 先用探索起草验证人物与文字，发行信息后补 |
| **篇幅不合理** | "我想写 10 万字的超长篇"——概念混淆 | 用篇幅规划指南确认：短篇(1-10万)/中篇(10-30万)/长篇(30-100万)/超长篇(100万+) |
| **更新频率太高** | "日更 1 万字"——不可持续 | 根据实际时间确认：日更/隔日更/周更，并给出合理字数建议 |
| **用立项阻止试写** | 用户想先找人物声音，却要求填完整立项单 | 使用 `minimal` + 探索起草；只锁不可逆决定，其余保留未定 |

## 快速检查（5 分钟）

尽量回答；未知项可以标为“通过试写发现”：

1. 当前是哪种写作模式？
2. 这次最想写清哪个人物、处境或问题？
3. 当前文字要给人什么体验？
4. 哪些事实和人物选择不能随意改？
5. 下一步最小写作动作是什么？

只要已有足够信息支持下一步，就可以进入写作；不按填空数量给立项打分。
