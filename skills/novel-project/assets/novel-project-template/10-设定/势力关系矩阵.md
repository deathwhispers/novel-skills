# 势力关系矩阵

> **使用说明**：本表是势力之间的横向关系层，对应 `10-设定/势力/势力名.md` 的纵向单势力档案。
> 当书内有 5 个以上核心势力时，必须建立本表——否则合纵连横、势力洗牌、阵营翻转这类关键情节没有可追溯的底图。
> 行=势力 A，列=势力 B，格子=关系。格子里的内容要同时写关系类型和亲疏强度，不要只写"敌对"或"友好"。

## 关系类型枚举

| 缩写 | 全称 | 说明 |
|------|------|------|
| 同盟 | 同盟/结盟 | 公开或半公开的合作关系 |
| 从属 | 从属/附庸 | 上级势力对下级势力的控制关系 |
| 敌对 | 敌对/对抗 | 公开冲突，可能正在战争或冷战 |
| 紧张 | 紧张/猜忌 | 没有公开冲突但相互不信任 |
| 中立 | 中立/无交集 | 当前没有显著互动 |
| 暧昧 | 暧昧/利用 | 一方在利用另一方或双方各怀心思 |
| 同源 | 同源/同门 | 历史上是同一组织分裂而来 |
| 血仇 | 血仇/世仇 | 有历史血债，难以化解 |

## 关系亲疏强度（1-5）

- **5**：铁盟/血盟，生死与共
- **4**：紧密合作，重大决策会同步
- **3**：常规合作，平时有利益交换
- **2**：冷合作/冷对立，利益相同时合作，否则各走各的
- **1**：表面文章，随时可能翻脸
- **-1**：轻度紧张
- **-2**：紧张/冷战
- **-3**：敌对/小规模冲突
- **-4**：全面对抗
- **-5**：血仇/必须消灭

## 势力关系矩阵

> 单元格写法：关系类型 / 亲疏强度（1-5/-1-5）/ 关系概述（一句话）
> 自反格子（势力A ↔ 势力A）填"—"。

|  | 势力A | 势力B | 势力C | 势力D | 势力E | 势力F |
|--|------|------|------|------|------|------|
| **势力A** | — | | | | | |
| **势力B** | | — | | | | |
| **势力C** | | | — | | | |
| **势力D** | | | | — | | |
| **势力E** | | | | | — | |
| **势力F** | | | | | | — |

### 填写示例

> 假设"青云宗"与"天剑宗"是同源死敌，与"大周皇室"是从属关系，与"散修联盟"是紧张状态。

|  | 青云宗 | 天剑宗 | 大周皇室 | 散修联盟 |
|--|------|------|------|------|
| **青云宗** | — | 血仇 / -5 / 同源分裂，三百年前夺嫡之仇 | 从属 / 4 / 受皇室册封，受调令约束 | 紧张 / -2 / 散修抢过本宗弟子 |
| **天剑宗** | 血仇 / -5 / 三百年前夺嫡之仇 | — | 中立 / 1 / 不受皇室管辖 | 暧昧 / 1 / 暗中收买散修 |
| **大周皇室** | 从属 / 4 / 受皇室册封 | 中立 / 1 / 不受皇室管辖 | — | 中立 / 0 / 默认不干预 |
| **散修联盟** | 紧张 / -2 / 散修抢过本宗弟子 | 暧昧 / 1 / 暗中收买散修 | 中立 / 0 / 默认不干预 | — |

## 关系大事记

> 关系不是静态的。每一次重大事件（结盟/破裂/翻脸/联姻/刺杀）都要落一行。
> 这一栏是关系矩阵的"动态层"。

| 章节/事件 | 涉及势力 | 关系变化 | 触发事件 | 后续影响 |
|----------|---------|---------|---------|---------|
| 例：第 18 章 | 青云宗 ↔ 散修联盟 | 从中立 → 紧张 | 散修联盟挖走本宗一名内门弟子 | 双方进入冷战，第 25 章需要做一次正面接触化解或升级 |

## 势力合纵连横当前格局

> 用一段话描述当前书内"势力博弈"的全景。这段话是写势力戏时的快速参照。

```
（用文字画出当前势力格局——谁是盟主、谁被孤立、哪几个势力在暗中勾连、哪几个势力是潜在的火药桶。
 比如：当前青云宗与大周皇室深度绑定，对天剑宗保持高压；散修联盟被天剑宗暗中渗透；新兴的「烬火门」
 在两强之间走钢丝，是当前最不稳定的变量。）
```

## 关系自检

- [ ] 关系矩阵里是否有长时间不变的死敌（如果不是主线对抗设计，提示关系可能太静态）
- [ ] 关系大事记里是否每 10-20 章至少有一条关系变化
- [ ] 是否有势力被孤立（与所有势力关系 ≤ 0）——这种势力是定时炸弹，要么引爆要么淘汰
- [ ] 主角所在势力的关系网络是否够复杂（少于 3 段不同性质关系，提示剧情可能太线性）
- [ ] 上一卷与本卷相比，关系矩阵里至少有几格发生了变化
