# 前文索引

> **使用说明**：百万级长篇写到中后期，每次续写都要回看前文——但靠翻章节文件效率极低。
> 本表是"前文快速索引"——把每章的关键名场面、关键对话、关键设定抽出来，按主题/角色/设定分类。
> 配套 `scripts/extract_index.py` 脚本会自动从章节中抽取关键句写入本表。
> 这是写作时最该高频回看的"运行面板"之一。

## 索引说明

> 本表按以下三类组织：

1. **关键名场面**：本章有哪个读者能记住的画面（高潮/转折/重要对话）
2. **关键对话**：本章有哪些"被反复引用"的话（约定/誓言/遗言/承诺）
3. **关键设定/物件**：本章出现了哪些被反复提及的物品/规则/概念

## 自动抽取脚本

> 运行 `python3 scripts/extract_index.py /path/to/novel` 会从 `30-正文/第X卷-卷名/` 中：
> - 抽取包含引号"对话"的内容
> - 抽取每章的 H2 标题作为"关键名场面"
> - 抽取角色名+动作/对话的组合
> - 自动更新本表

> 脚本生成的索引会标记 `[AUTO]` 提示是自动生成，需要人工校对。

## 使用流程

1. 写新章节前，先翻本表查"上一章做了什么/说了什么"
2. 写完后，运行 `extract_index.py` 自动补全本表
3. 每月做一次"索引体检"——删除过期/无效条目，合并重复项
4. 重大事件（卷末、卷间）发生时手动补充

## 索引自检

- [ ] 真正重要且后文可能调用的场面是否已记录
- [ ] 关键对话是否被原文引用（不要凭记忆改写）
- [ ] 关键设定/物件是否第一次出现 + 关键使用 + 揭示都被记录
- [ ] 索引是否每 20 章更新一次（过期太久就失效）
- [ ] 是否有"读者会想反复看"的内容没被记录
- [ ] 索引是否过大（> 1000 行考虑分卷）
