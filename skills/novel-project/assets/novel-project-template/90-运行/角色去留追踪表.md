# 角色去留追踪表

> **使用说明**：百万级长篇会有大量角色进出——某些角色中途退场（如死亡、隐退、被囚禁），某些角色一段时间不出现。
> 没有这张表，写到中后期作者会自然"写丢"角色——读者会问"那个 A 章出场的 X 角色后来去哪了"。
> 本表追踪每个角色的"在场状态"和"消失/回归计划"。

## 角色在场状态枚举

| 状态 | 含义 | 追踪重点 |
|------|------|---------|
| **活跃** | 当前正在出场或近期出场 | 最近出场章节、下次出场计划 |
| **冷藏** | 故意一段时间不出现 | 冷藏原因、计划回归章节 |
| **退场** | 已经死亡/退隐/消失 | 退场方式、是否有复活可能 |
| **悬空** | 不知道还在不在 | 计划如何处理（确认退场/回归） |
| **预设出场** | 还没出场但有计划 | 计划出场章节、引入方式 |

### 退场方式的写法

- **死亡**：是当场死、伤重不治、还是被人救走又死了
- **退隐**：是主动退隐、还是被动退隐（被迫）
- **失踪**：是真的失踪了，还是被作者故意"放飞"
- **被囚**：被谁囚、囚在哪里、什么时候能出来
- **出奔**：是自己跑、还是被人带走

## 角色去留自检

- [ ] 是否所有"应该退场"的角色都已经明确标记（避免"假退场"）
- [ ] 是否所有"应该回归"的角色都有回归计划（避免"假冷藏"）
- [ ] 冷藏角色是否超过 30 章没回归（超过要警惕读者遗忘）
- [ ] 退场角色的"最终状态"是否被记录（避免后文写错）
- [ ] 预设出场的角色是否有具体的引入计划（避免"到时候现想"）
- [ ] 是否有"实际已经写丢"的角色（应该在表里但作者忘了）
- [ ] 卷末的活跃角色列表是否与本表一致（避免矛盾）
