# 日更/连载工作流

> **使用说明**：90% 网文作者是日更/周更——但现有项目只覆盖"创作"不覆盖"连载运营"。
> 本表是连载工作流——日更时间表、字数追踪、卡文应对、月度回顾。
> 配 `90-运行/连载驾驶舱.md` 和 `90-运行/当前进度.md` 使用。

## 一、连载节奏的 4 种模式

> 不同作者有不同的连载节奏——本表覆盖 4 种常见模式。

| 模式 | 字数 | 频率 | 适合 |
|------|------|------|------|
| **日更** | 2000-2500/天 | 每天 | 全职作家 |
| **双更** | 4000-5000/天 | 每天 2 更 | 高产作者 |
| **周更** | 8000-15000/周 | 每周 | 兼职作家 |
| **月更** | 20000-30000/月 | 每月 | 慢工细活 |

> **关键**：选定一个模式后，**不要中途频繁切换**——读者会被"突然不更"激怒。

## 二、日更时间表（标准 2000-2500 字/天）

> 假设日更 2000-2500 字、每天 1 章、每章 2000-2500 字。

### 时间表模板

| 时段 | 时长 | 任务 |
|------|------|------|
| 08:00-08:30 | 30 分钟 | 回顾上一章 + 看连载驾驶舱 |
| 08:30-09:00 | 30 分钟 | 写节拍卡（如未完成） |
| 09:00-11:30 | 2.5 小时 | 写正文（2000-2500 字） |
| 11:30-12:00 | 30 分钟 | 自检 + 章尾钩子检查 |
| 12:00-13:00 | 1 小时 | 午休 |
| 13:00-14:00 | 1 小时 | 修改昨晚的稿 + 回复评论 |
| 14:00-15:00 | 1 小时 | 写节拍卡/伏笔更新/设定更新 |
| 15:00-15:30 | 30 分钟 | 看读者评论 + 思考下一步 |

> **核心规则**：每天至少 4 小时深度写作时间 + 1 小时回顾+更新。

## 三、字数追踪表

> 日更作者每天都要追踪字数——避免某天突然写少。

| 日期 | 章节 | 字数 | 计划 vs 实际 | 备注 |
|------|------|------|------------|------|
| 2024-01-01 | 第 5 章 | 3500 | +500 | 状态好 |
| 2024-01-02 | 第 6 章 | 2800 | -200 | 卡文 |
| 2024-01-03 | 第 7 章 | 3200 | +200 | 正常 |
| ... | | | | |

> **关键**：每天字数差异不超过 30%——稳定比爆更更重要。

## 四、卡文应对表（按日更场景设计）

> 日更场景下卡文是最常见的问题——本表是 5 分钟卡文应对。

### 卡文类型 1：开篇卡文

- **场景**：不知道这一章怎么开始
- **应对**：
  1. 翻上一章最后一段——接续写
  2. 翻节拍卡——按"被前文什么结果逼出来的"写
  3. 用"动作开场"——先写一个动作（进屋/拔剑/说话）
  4. 用"环境开场"——先写一个环境（雨/雪/集市）

### 卡文类型 2：中段卡文

- **场景**：写了一半不知道怎么往下
- **应对**：
  1. 翻节拍卡——核心戏剧任务是什么
  2. 写"小高潮"——冲突最激烈的瞬间
  3. 写"反常"——主角做出意外的事
  4. 加"新角色"——临时引入新视角
  5. 加"反转"——读者没想到的事

### 卡文类型 3：章尾卡文

- **场景**：不知道怎么收尾
- **应对**：
  1. 6 种发动机选一个：新问题/不可逆决定/倒计时/权力反转/情绪余震/关系破口
  2. 写一句"最难忘的话"
  3. 写一个"画面感强"的场景定格
  4. 章尾"留白"——用动作代替说明

### 卡文类型 4：整体卡文

- **场景**：连续 3 天写不动
- **应对**：
  1. **跳到下一章**——先写卡文之后的章
  2. **加新角色/新事件**——用新元素激活剧情
  3. **改变节奏**——从战斗换成对话
  4. **强制收尾**——把这一章写短（1500-2000 字）作为过渡章
  5. **AI 协助**——用 `novel-writing` 的卡文急救 prompt

## 五、每日必做清单

> 日更作者每天必须做的事。

| 必做 | 时长 | 用途 |
|------|------|------|
| 回顾上一章 | 5 分钟 | 接续 |
| 看连载驾驶舱 | 5 分钟 | 知道整体进度 |
| 写正文 | 2-3 小时 | 主要工作 |
| 章尾检查 | 5-10 分钟 | 保证章尾有钩子 |
| 更新当前进度 | 5 分钟 | 记录今天做了什么 |
| 回复评论（可选） | 30 分钟 | 维护读者 |
| 更新伏笔/设定 | 10 分钟 | 防止漏线 |

> **核心规则**：每天必须更新 `90-运行/当前进度.md` 和 `连载驾驶舱.md`。

## 六、每周回顾

> 周末做 1 次回顾——避免中段空转。

| 必看 | 用途 |
|------|------|
| 连载驾驶舱 | 整体进度 |
| 弧线追踪 | 哪些线推进了 |
| 读者评论 | 读者反馈 |
| 章间推进密度曲线 | 节奏是否塌 |
| 伏笔监控 | 是否失联 |

| 必做 | 用途 |
|------|------|
| 调整下周计划 | 灵活应对 |
| 备份稿子 | 防丢失 |
| 自我评估 | 是否需要调整 |
| 休息 1 天 | 防止疲劳 |

## 七、月度体检

> 每月做 1 次完整体检——避免后期失控。

| 必做 | 用途 |
|------|------|
| 卷体检 | 当前卷节奏 |
| 长线扫描 | 伏笔失联 |
| 追读排查 | 最近追读 |
| 中期审视 | 整体健康度 |
| 反馈整理 | 读者反馈分类 |

| 必更新 | 用途 |
|------|------|
| 全角色卷末快照 | 角色状态 |
| 角色去留追踪 | 角色分布 |
| 角色立场漂移 | 立场变化 |
| 信息债监控 | 债的状态 |
| 关系矩阵 | 关系网络 |

## 八、突发情况应对

> 连载中可能遇到的 5 种突发情况。

| 突发 | 应对 |
|------|------|
| **平台故障/被封** | 立刻备份全稿 + 联系平台 + 准备备用渠道 |
| **被骂/网暴** | 截图保留证据 + 屏蔽 + 找朋友倾诉 + 继续写 |
| **灵感枯竭** | 看其他作品 + 散步 + 写小片段 + 强行续写 |
| **身体不适** | 立刻休息 + 减少更新（写短章）+ 告知读者 |
| **家人/工作** | 提前沟通 + 储备稿子 + 接受"断更" |

> **核心规则**：**健康第一**——断更一天比烂更一周好。

## 九、长期可持续的 5 个建议

> 长篇连载是"马拉松"——必须可持续。

| 建议 | 说明 |
|------|------|
| **固定时间写作** | 每天同一个时间，养成习惯 |
| **不追热点** | 自己的节奏比追热点重要 |
| **有备稿** | 至少存 3-5 天稿子 |
| **不熬夜** | 健康第一 |
| **定期休息** | 每周至少 1 天不写 |

## 十、日更的 5 个红线

> 日更作者最容易犯的 5 个错误。

| 红线 | 表现 | 危害 |
|------|------|------|
| **每天硬更 3000** | 字数不够硬凑 | 质量崩 |
| **周末不更** | 突然断更 | 读者流失 |
| **熬夜更** | 牺牲健康 | 后期无力 |
| **卡文硬写** | 不做调整硬写 | 烂尾 |
| **不更新设定** | 写完不更新文档 | 后期失联 |

## 十二、日更的"AI prompt"

### Prompt：日更规划

```
请帮我规划日更时间表。

【项目信息】
- 题材：{{}}
- 字数要求：{{2000-2500/天}}
- 工作时间：{{}}
- 当前进度：{{第 X 章}}

【要求】
1. 制定每天的写作时间表
2. 列出每天必做的事
3. 列出卡文时的应对方法
4. 给出每周/月的回顾节奏

【输出】具体的时间表。
```

### Prompt：卡文急救

```
我卡文了，{{具体位置：开篇/中段/章尾/整体}}。

【已有内容】
{{}}

【节拍卡要求】
{{}}

【5 分钟卡文应对】
1. 如果是开篇卡文：{{具体应对}}
2. 如果是中段卡文：{{具体应对}}
3. 如果是章尾卡文：{{具体应对}}

【输出】3 种不同的"接下来写什么"方案。
```
