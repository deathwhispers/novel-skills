# 导入检查清单

> **使用说明**：用户有旧稿需要导入新项目时，按本清单执行。
> 不是"复制粘贴"——是"分类整理+缺口识别"。
> 适用于重构流程的立项阶段（导入旧项目）。

## 导入总流程

```
收集旧稿 ──→ 分类整理 ──→ 识别缺口 ──→ 补充缺失 ──→ 验证完整性
```

---

## 第一步：收集旧稿

### 检查旧稿来源

| 来源类型 | 常见位置 | 处理方式 |
|----------|----------|----------|
| 本地文件 | 电脑文件夹、云盘 | 全部收集到临时目录 |
| 在线平台 | 起点/番茄/晋江后台 | 导出或复制 |
| 笔记软件 | 印象笔记/Notion/语雀 | 导出为 Markdown |
| 手写/纸质 | 笔记本、打印稿 | 扫描或手打 |
| 聊天记录 | 微信/QQ/Discord | 提取关键内容 |

### 收集清单

- [ ] 已发布的章节（正文）
- [ ] 未发布的草稿
- [ ] 大纲/设定笔记
- [ ] 角色设定
- [ ] 世界观设定
- [ ] 弃稿/废稿
- [ ] 读者评论/反馈
- [ ] 自己的写作笔记

---

## 第二步：分类整理

### 2.1 正文分类

按章节整理到 `30-正文/`：

```
30-正文/
└── 旧稿导入/
    ├── 第001章-旧标题.md
    ├── 第002章-旧标题.md
    └── ...
```

**整理规则**：
- 按自然顺序编号（第001, 第002...）
- 保留原标题（后续可修改）
- 删除重复章节（保留最新版本）
- 标记章节状态：
  - `【已发布】` 已在外部平台发布
  - `【草稿】` 未发布
  - `【废稿】` 废弃不用

### 2.2 设定分类

整理到 `10-设定/`：

| 旧稿内容 | 目标位置 | 处理方式 |
|----------|----------|----------|
| 角色设定 | `10-设定/角色/姓名.md` | 按 `角色卡模板.md` 格式整理 |
| 世界观 | `10-设定/硬设定.md` / `世界规则.md` | 合并到对应文件 |
| 势力设定 | `10-设定/势力/势力名.md` | 每个重要势力一份动态档案 |
| 地点设定 | `10-设定/地点/地点名.md` | 每个重要地点一份动态档案 |
| 时间线 | `10-设定/时间线.md` | 按时间顺序整理 |

### 2.3 大纲分类

整理到 `20-大纲/`：

| 旧稿内容 | 目标位置 | 处理方式 |
|----------|----------|----------|
| 全书大纲 | `20-大纲/全书总纲.md` | 提取核心冲突和演化路径 |
| 卷大纲 | `20-大纲/分卷/volume-XX.md` | 按 `卷纲模板.md` 格式整理 |
| 章节大纲 | `20-大纲/节拍卡/chapter-XXX.md` | 按 `节拍卡模板.md` 格式整理 |
| 伏笔记录 | `20-大纲/回收/回收总账.md` | 整理为表格 |

## 第三步：识别缺口

### 3.1 正文缺口

| 检查项 | 缺口类型 | 处理方式 |
|--------|----------|----------|
| 章节编号不连续 | 缺章 | 标记为"待补" |
| 章节内容过短 | < 1000 字 | 标记为"待扩" |
| 章节内容过散 | 无核心任务 | 标记为"待修" |
| 章节间衔接断 | 前后不接 | 标记为"待补衔接" |
| 卷级分布不均 | 某卷过多/过少 | 标记为"待调" |

### 3.2 设定缺口

| 检查项 | 缺口类型 | 处理方式 |
|--------|----------|----------|
| 无硬设定 | 世界规则不明确 | 补充 `硬设定.md` |
| 无角色卡 | 角色只有名字 | 按模板补角色卡 |
| 无时间线 | 时间混乱 | 建立 `时间线.md` |
| 无地点 | 场景模糊 | 补充 `地点.md` |
| 无力量体系 | 升级逻辑不清 | 补充 `力量体系.md` |

### 3.3 大纲缺口

| 检查项 | 缺口类型 | 处理方式 |
|--------|----------|----------|
| 无全书总纲 | 核心冲突不清 | 按 `大纲阶梯.md` 补 |
| 无卷纲 | 卷功能不明 | 按 `卷纲模板.md` 补 |
| 无节拍卡 | 章节任务不清 | 按 `节拍卡模板.md` 补 |
| 无因果图 | 前后因果断 | 建立 `场景因果图.md` |
| 无回收总账 | 伏笔混乱 | 建立 `回收总账.md` |

---

## 第四步：补充缺失

### 4.1 正文补充

| 缺失类型 | 补充方式 | 优先级 |
|----------|----------|--------|
| 缺章 | 根据大纲补写 | P0 |
| 衔接断 | 在两章间加过渡段 | P1 |
| 章节过短 | 补具体细节/扩冲突 | P1 |
| 章节过散 | 重写核心任务 | P2 |
| 卷级不均 | 调整卷分配 | P2 |

### 4.2 设定补充

| 缺失类型 | 补充方式 | 负责 skill |
|----------|----------|------------|
| 硬设定 | 提取正文中的规则，整理成文 | novel-worldbuilding |
| 角色卡 | 根据正文中的角色表现反推 | novel-worldbuilding |
| 时间线 | 按正文中的时间线索整理 | novel-worldbuilding |
| 地点 | 根据正文中的场景描写整理 | novel-worldbuilding |

### 4.3 大纲补充

| 缺失类型 | 补充方式 | 负责 skill |
|----------|----------|------------|
| 全书总纲 | 从已写章节反推核心冲突 | novel-outline |
| 卷纲 | 按已写章节反推卷功能 | novel-outline |
| 节拍卡 | 从已写章节中提取任务 | novel-outline |
| 因果图 | 按章节顺序梳理因果链 | novel-outline |

---

## 第五步：验证完整性

### 5.1 正文完整性

- [ ] 章节编号连续（无跳号）
- [ ] 每章有标题
- [ ] 每章有核心任务（能从内容中看出）
- [ ] 章间衔接顺（读完一章知道下一章该看什么）
- [ ] 卷级分布合理（无异常长短卷）

### 5.2 设定完整性

- [ ] 硬设定有文件
- [ ] 主角有角色卡
- [ ] 主要反派有角色卡
- [ ] 关键地点有描述
- [ ] 力量体系有定义

### 5.3 大纲完整性

- [ ] 有全书总纲
- [ ] 有分卷规划
- [ ] 已写章节有节拍卡
- [ ] 有伏笔回收记录

## 导入决策树

```
开始导入
  ↓
收集旧稿齐？
  ├─ 否 → 继续收集
  └─ 是 → 分类整理
             ↓
        识别缺口
             ↓
        缺口多？
          ├─ 是（>10 个）→ 建议先走重构流程（Step 1-3）
          └─ 否 → 逐条补充
                      ↓
                 验证完整性
                      ↓
                 通过？
                   ├─ 否 → 回补充
                   └─ 是 → 导入完成，进入正常写作流程
```

---

## 导入后的下一步

导入完成后，根据项目状态决定下一步：

| 项目状态 | 下一步 |
|----------|--------|
| 已写 1-10 章 | 补齐大纲（Step 1-3），然后继续写作（Step 4） |
| 已写 10-50 章 | 做卷体检（novel-quality），然后补缺口 |
| 已写 50+ 章 | 做全身体检（novel-quality），然后决定重构范围 |
| 大纲/设定缺失多 | 先走重构流程 Step 1-3，补齐骨架 |
