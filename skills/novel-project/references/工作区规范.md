# 工作区规范

> **使用说明**：novel-project 初始化时创建的标准目录结构。
> 每个目录的用途、命名规则、文件格式都在本文定义。
> 适用于重构流程的立项阶段（Step 0）。

## 目录结构总览

```
my-novel/
├── 00-书核/              # 核心目标与承诺
├── 05-市场/              # 市场与对标研究
├── 10-设定/              # 世界观、角色、规则
├── 20-大纲/              # 故事结构与推进
│   ├── 分卷/
│   ├── 节拍卡/
│   ├── 因果/
│   └── 回收/
├── 30-正文/              # 正文资产
│   ├── 第一卷-卷名/
│   ├── 第二卷-卷名/
│   └── ...
├── 40-修订/              # 体检和修稿报告
│   ├── 体检报告/
│   └── 修稿报告/
├── 50-归档/              # 已完成的内容
└── 90-运行/              # 当前进度、决策记录
```

---

## 00-书核/

**用途**：存放"整本书不会改变"的核心文件。

| 文件 | 用途 | 格式 |
|------|------|------|
| `作品总表.md` | 书名、作者、题材、核心冲突、一句话简介 | Markdown |
| `立项单.md` | 详细立项信息（见 `立项单模板.md`） | Markdown |
| `读者承诺.md` | 这本书给读者的核心承诺（爽点/虐点/悬念） | Markdown |

**命名规则**：
- 文件名固定，不随意更改
- 所有文件使用 Markdown 格式
- 文件头使用 YAML front matter

## 10-设定/

**用途**：世界观、角色、势力、地点等设定资产。

### 核心设定文件

| 文件 | 用途 | 创建时机 |
|------|------|----------|
| `硬设定.md` | 不可违反的世界底层规则 | 开书前 |
| `世界规则.md` | 能力系统、社会机制、禁忌与代价 | 开书前 |
| `力量体系.md` | 等级、能力、突破路径 | 开书前 |
| `地点.md` | 核心场景地点总表 | 开书前 |
| `势力.md` | 核心势力组织总表 | 开书前 |
| `时间线.md` | 历史事件+当前时间 | 卷一写作中 |
| `母题.md` | 主题、象征、反复出现的意象 | 中后期 |
| `文风指南.md` | 写作风格、POV规则、禁用语 | 开书前 |

### 角色目录

```
10-设定/
├── 角色/
│   ├── 角色卡模板.md
│   ├── 主角.md
│   ├── 反派.md
│   └── 配角-姓名.md
```

**角色卡最低要求**（见 `novel-worldbuilding/SKILL.md`）：
- role, desire, fear, contradiction, secret, leverage, voice markers, relationship pressure, arc direction

### 其他模板文件

| 文件 | 用途 |
|------|------|
| `角色关系矩阵.md` | 横向追踪所有角色关系（角色>10时必建） |
| `角色阵营归属表.md` | 追踪阵营、立场强度、变节可能 |
| `势力关系矩阵.md` | 横向追踪势力关系（势力>5时必建） |
| `势力动态史.md` | 追踪势力兴衰史 |
| `世界层级图.md` | 界-域-国-城-区-场所层级树 |
| `地点距离表.md` | 关键地点两两距离/交通时间 |
| `地点模板.md` | 新增地点时复用 |
| `势力模板.md` | 新增势力时复用 |

## 20-大纲/

**用途**：故事结构、卷纲、节拍卡、因果链、伏笔回收。

### 目录结构

```
20-大纲/
├── 全书总纲.md
├── 升级阶梯.md
├── 弧线追踪.md
├── 分卷/
│   ├── volume-01.md
│   ├── volume-02.md
│   └── ...
├── 节拍卡/
│   ├── chapter-001.md
│   ├── chapter-002.md
│   └── ...
├── 因果/
│   └── 场景因果图.md
└── 回收/
    └── 回收总账.md
```

### 文件用途

| 文件 | 用途 | 更新频率 |
|------|------|----------|
| `全书总纲.md` | 核心冲突、冲突演化路径 | 重构时 |
| `升级阶梯.md` | 主角三维升级路径 | 每卷末更新 |
| `弧线追踪.md` | 关键角色弧线起点终点 | 每卷末更新 |
| `分卷/volume-XX.md` | 每卷骨架（见 `卷纲模板.md`） | 分卷规划时 |
| `节拍卡/chapter-XXX.md` | 每章 10 项节拍卡 | 每章写作前 |
| `因果/场景因果图.md` | 跨章因果关系 | 每章写作后更新 |
| `回收/回收总账.md` | 伏笔埋设与回收记录 | 每章写作后更新 |

## 30-正文/

**用途**：章节正文资产。

### 目录结构

```
30-正文/
├── 第一卷-卷名/
│   ├── 第001章-章节名.md
│   ├── 第002章-章节名.md
│   └── ...
├── 第二卷-卷名/
│   ├── 第010章-章节名.md
│   └── ...
└── ...
```

### 命名规则

**卷目录**：
- 格式：`第X卷-卷名/`
- 示例：`第一卷-青云入门/`、`第二卷-宗门大比/`

**章节文件**：
- 格式：`第XXX章-章节名.md`
- 示例：`第001章-初入宗门.md`、`第002章-入门测试.md`
- 使用三位数编号（001, 002, ... 010, 011）
- 章节名简明，2-6 字

### 文件格式

```markdown
# 第XXX章-章节名

正文内容...

---

## 元数据

- 字数：{{实测字数}}
- 状态：草稿/定稿
- 节拍卡：chapter-XXX.md
- 体检报告：chXXX-checkup.md（如有）
```

## 90-运行/

**用途**：当前进度、决策记录、连载驾驶舱。

| 文件 | 用途 | 更新频率 |
|------|------|----------|
| `当前进度.md` | 当前章节、未决问题、下一步 | 每章写作后 |
| `决策记录.md` | 重要决策和原因 | 每次决策时 |
| `连载驾驶舱.md` | 近 5 章任务表、追读数据 | 每周 |
| `人物状态变迁日志.md` | 核心人物状态追踪 | 每章写作后 |

---

## 命名规范

### 文件命名

- 使用中文命名
- 序号使用三位数（001, 002, ...）
- 避免特殊字符（除 `-` 和 `_`）
- 使用 `.md` 扩展名

### 目录命名

- 使用 `XX-描述/` 格式
- 序号使用两位数（00, 05, 10, ...）
- 卷目录使用 `第X卷-卷名/`

新建项目时，按以下清单创建目录和初始文件：

- [ ] 创建根目录 `my-novel/`
- [ ] 创建 8 个一级目录（00-书核/ 到 90-运行/）
- [ ] 创建 `20-大纲/` 子目录（分卷/ 节拍卡/ 因果/ 回收/）
- [ ] 创建 `10-设定/角色/` 目录
- [ ] 创建 `30-正文/`（空，等卷目录）
- [ ] 创建 `40-修订/` 子目录（体检报告/ 修稿报告/）
- [ ] 创建 `50-归档/` 子目录
- [ ] 创建初始文件：`00-书核/作品总表.md`、`00-书核/立项单.md`
- [ ] 创建模板文件：`10-设定/角色/角色卡模板.md`、`10-设定/地点模板.md`、`10-设定/势力模板.md`
