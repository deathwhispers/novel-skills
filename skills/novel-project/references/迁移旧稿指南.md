# 迁移旧稿指南

> **使用说明**：将已有的长篇小说旧稿迁移到本工作区的详细步骤。
> 比"导入检查清单"更具体——是按步骤操作的手册。
> 适用于已有完整/半完整旧稿的作者。

## 迁移前评估

### 评估旧稿状态

在迁移前，先评估旧稿的"健康度"：

| 维度 | 评估项 | 健康 | 亚健康 | 不健康 |
|------|--------|------|--------|--------|
| 完整性 | 章节是否连续 | 无跳跃 | 少量跳跃 | 大量跳跃 |
| 一致性 | 设定是否前后一致 | 一致 | 少量矛盾 | 大量矛盾 |
| 格式 | 文件格式是否统一 | 统一 | 2-3 种格式 | 多种格式混杂 |
| 组织 | 是否有目录结构 | 有 | 部分有 | 无 |
| 大纲 | 是否有大纲 | 有 | 有部分 | 无 |
| 设定 | 是否有设定文档 | 有 | 有部分 | 无 |

**决策**：
- 3 项以上"健康" → 直接迁移
- 3 项"亚健康" → 迁移 + 补建
- 2 项以上"不健康" → 建议先整理再迁移，或按重构流程重来

## 第一阶段：准备（30 分钟）

### 1.1 创建新工作区

```bash
# 创建项目目录
mkdir -p my-novel-migrated
cd my-novel-migrated

# 按工作区规范创建基础目录
mkdir -p 00-书核 10-设定/角色 20-大纲/分卷 20-大纲/节拍卡 20-大纲/因果 20-大纲/回收 30-正文 50-归档/旧稿 50-归档/废稿 90-运行
```

### 1.2 复制旧稿到临时目录

```bash
# 创建临时整理区
mkdir -p temp_old_drafts

# 复制所有旧稿文件
cp /path/to/old/novel/* temp_old_drafts/

# 记录原始文件清单
ls -R temp_old_drafts/ > 90-运行/旧稿原始清单.md
```

### 1.3 安装必要工具

```bash
# 检查 pandoc（用于格式转换）
pandoc --version

# 如无，安装：
# macOS: brew install pandoc
# Ubuntu: sudo apt-get install pandoc
```

## 第二阶段：正文迁移（2-4 小时）

### 2.1 格式转换

#### Word → Markdown

```bash
# 批量转换
for f in temp_old_drafts/*.docx; do
    pandoc "$f" -o "temp_old_drafts/$(basename $f .docx).md"
done
```

**转换后清理**：
- 删除 Word 格式残留（`{...}` 样式标记）
- 统一标题层级（`#` 为章标题，`##` 为节）
- 统一对话格式（`「对话」` 或 `"对话"`，选一个）

#### txt → Markdown

```bash
# 直接改后缀，添加标题
for f in temp_old_drafts/*.txt; do
    mv "$f" "temp_old_drafts/$(basename $f .txt).md"
done
```

**添加标题**：
```markdown
# 第XXX章-章节名

（原内容）
```

### 2.2 章节重命名

#### 提取章节信息

```bash
# 列出所有章节文件
ls temp_old_drafts/*.md | sort
```

#### 创建映射表

```markdown
## 章节迁移映射

| 旧文件名 | 旧章节号 | 新章节号 | 新文件名 | 状态 |
|----------|----------|----------|----------|------|
| 第1章.docx | 1 | 001 | 第001章-初入青云.md | ✅ |
| chapter_5.txt | 5 | 005 | 第005章-待命名.md | ⚠️ |
| ... | ... | ... | ... | ... |
```

#### 执行重命名

```bash
# 按映射表重命名
mv temp_old_drafts/第1章.md 30-正文/未分类/第001章-初入青云.md
mv temp_old_drafts/chapter_5.md 30-正文/未分类/第005章-待命名.md
# ...
```

### 2.3 卷划分

如果旧稿有卷概念：

```bash
# 创建卷目录
mkdir -p 30-正文/第一卷-卷名
mkdir -p 30-正文/第二卷-卷名

# 移动章节到对应卷
mv 30-正文/未分类/第001章-*.md 30-正文/第一卷-卷名/
mv 30-正文/未分类/第025章-*.md 30-正文/第二卷-卷名/
```

如果旧稿无卷概念：

1. 按 `卷纲拆章指引.md` 的拆章原则重新划分
2. 或暂时全部放入 `30-正文/未分类/`，后续重构时再划分

### 2.4 内容清理

每章迁移后执行：

- [ ] 删除空行（连续空行保留 1 个）
- [ ] 统一段落缩进（无缩进或 2 空格缩进）
- [ ] 统一对话格式
- [ ] 删除页眉/页脚/页码
- [ ] 删除 Word 的格式标记

## 第三阶段：设定迁移（1-2 小时）

### 3.1 角色设定迁移

**旧稿来源**：
- 角色设定文档 → 拆分为独立文件
- 正文中的角色描写 → 提取关键信息
- 脑图/表格 → 文字化

**迁移步骤**：

1. 列出所有角色
2. 为主要角色创建独立文件：`10-设定/角色/角色名.md`
3. 按 `novel-worldbuilding` 的角色卡最低要求填写
4. 次要角色可合并到 `10-设定/角色/配角群像.md`

### 3.2 世界观迁移

**旧稿来源**：
- 世界观文档 → 分类整理
- 正文中的设定揭示 → 提取为规则
- 零散笔记 → 归类

**迁移步骤**：

1. 硬设定（不可违反的规则）→ `10-设定/硬设定.md`
2. 力量体系 → `10-设定/力量体系.md`
3. 世界规则 → `10-设定/世界规则.md`
4. 地点 → `10-设定/地点/地点名.md`
5. 势力 → `10-设定/势力/势力名.md`

### 3.3 时间线迁移

**从正文中提取**：
- 历史事件（正文提到的"三年前""百年前"）
- 当前时间线（事件顺序）

**迁移到**：`10-设定/时间线.md`

## 第四阶段：大纲补建（2-4 小时）

### 4.1 全书总纲补建

**从旧稿中提取**：

```markdown
# 全书总纲（补建）

## 核心冲突
[从正文第 1-3 章提取主角的初始状态和触发事件]

## 冲突演化路径
[根据已有章节总结起承转合]

## 主角弧线
[从正文提取主角的起点状态和当前状态]
```

**填写方式**：
1. 读第 1 章：提取主角初始状态
2. 读最后一章：提取当前状态
3. 读关键转折章：提取升级节点
4. 按 `大纲阶梯.md` 第 1-2 阶格式填写

### 4.2 卷纲补建

**如果已有卷划分**：

按 `卷纲模板.md` 格式，逐卷填写：

```markdown
# 第 X 卷：卷名（补建）

- **功能定位**：根据本卷内容判断（起势/发展/转折/深化/收束）
- **核心卖点**：本章最突出的感受（爽/虐/悬/甜）
- **开卷承诺**：第 1 章给读者什么期待
- **卷末钩子**：最后一章的悬念
```

**如果无卷划分**：

1. 按 `卷纲拆章指引.md` 的"从字数预算反推"重新划分
2. 或标记 `[待重构]`，后续按重构流程 Step 2 处理

### 4.3 节拍卡补建

**对已写章节**：

按 `节拍卡模板.md` 格式， retroactively 填写：

```markdown
# 第 XXX 章：章节名（补建）

- **视角人物**：[从正文判断]
- **核心戏剧任务**：[本章主角做了什么]
- **因果锚点**：[被上一章什么逼出来的]
- **阻力**：[被什么卡住]
- **局面变化**：[章末和章初相比变了什么]
- **章尾发动机**：[6 种之一]
```

**注意**：补建的节拍卡标记 `[补建]`，后续重构时优先审查。

## 迁移后建议

### 立即做的事

1. **补建缺失文件**：按 `导入检查清单.md` 的 Step 4 补建
2. **做一轮体检**：用 `novel-quality` 对迁移后的章节做连续性检查
3. **更新当前进度**：`90-运行/当前进度.md`

### 一周内做的事

1. **重审全书总纲**：按 `大纲阶梯.md` 第 1 阶检查核心冲突
2. **统一设定**：解决迁移中发现的一致性问题
3. **补建角色卡**：为主要角色补建完整角色卡

### 重构时做的事

1. **按重构流程复核必要阶段**：旧稿完整时跳过无关步骤，只验证当前风险
2. **重新划分卷**：如果旧稿无卷概念，按 `卷纲拆章指引.md` 重新划分
3. **补建节拍卡**：为所有已写章节补建完整节拍卡

## 迁移工具脚本（可选）

### 批量重命名脚本

```bash
#!/bin/bash
# rename_chapters.sh

# 将旧章节文件按新规范重命名
# 使用方式：./rename_chapters.sh old_dir new_dir

OLD_DIR=$1
NEW_DIR=$2

for f in "$OLD_DIR"/*; do
    # 提取章节号
    ch_num=$(echo "$f" | grep -oE '[0-9]+' | head -1)
    # 补零到 3 位
    ch_padded=$(printf "%03d" $ch_num)
    # 提取章节名（简化处理）
    ch_name=$(basename "$f" | sed 's/第[0-9]*章//g' | sed 's/\.md//g' | sed 's/ //g')
    # 新文件名
    new_name="第${ch_padded}章-${ch_name}.md"
    
    cp "$f" "$NEW_DIR/$new_name"
    echo "Renamed: $f -> $new_name"
done
```

### 格式清理脚本

```bash
#!/bin/bash
# clean_format.sh

# 清理 Markdown 文件格式

for f in 30-正文/*/*.md; do
    # 删除连续空行（保留 1 个）
    sed -i '/^$/N;/^\n$/D' "$f"
    # 删除 Word 格式标记
    sed -i '/{\.\w*}/d' "$f"
    echo "Cleaned: $f"
done
```

> **注意**：脚本为示例，实际使用前请备份文件。
