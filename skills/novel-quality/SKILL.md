# 质量检查与修订

## 在重构流程中的位置

本 skill 承担重构流程的 **Step 5 和 Step 6**（见 `../novel-studio/references/重构流程.md`）：

| 步骤 | 在本 skill 的位置 | 转入下步条件 |
|------|------------------|--------------|
| Step 5 质量检测 | 第一步（章节体检）+ 第二步（问题诊断）+ 叙事质量审计 | 硬门禁全过 + 本章选定的 2-3 个软目标有可引用证据 |
| Step 6 最小修订 | 第三步（按授权修订）+ 第四步（语言与形式复核） | 改写未引入新硬伤，且没有为消除表面特征损害原有 voice |

**前后衔接**：
- 准入：Step 4 由 `novel-writing` 起稿完成，写后自检已通过
- 转出：Step 6 通过后回 `novel-writing` 做 Step 7 状态与学习；篇幅只在存在项目硬约束时复核
- 失败回退：硬门禁问题 → 优先局部修复，只有章节功能或因果骨架失效时才回 Step 3/4；软目标不足不自动触发整章重写

## 功能定位

把章节体检、结构修订、语言与形式校准整合成一个连贯的流程。这个 skill 负责：

1. **章节体检**：检查单章质量、前后照应、人物状态、伏笔回收
2. **结构修订**：修复逻辑问题、细节问题、回收问题
3. **语言与形式校准**：判断语言选择是否服务人物、场景、节奏和写作模式

## 工作流程

### 第一步：确定授权与检查范围

- 用户只要求审查：只诊断，不改正文。
- 用户要求修订、润色或重写：在指定范围内直接修改，并保留必要的前后对照。
- 用户未指定检查项：默认检查硬门禁，再根据写作模式选择最相关的 2-3 个软目标，不重复询问。
- 只有修订范围会显著改变剧情、人物立场或 voice 时才暂停确认。

### 第二步：问题诊断

根据体检结果，诊断问题类型：

1. **叙事成立问题**：连续性、人物动机、因果、canon 或非预期视角漂移
2. **场景与信息问题**：人物压力和选择是否可感，信息时机是否服务当前目的
3. **语言与形式问题**：具体语言选择是否造成重复解释、人物同声、节奏失真或形式偏离

### 第三步：按授权修订

先写明必须保留的戏剧功能、人物选择、信息和 voice，再按“结构/人物 → 场景与信息 → 语言与形式”顺序做最小修改。不要为了让所有指标变整齐而统稿。

### 第四步：语言与形式复核

读取 `references/去味手法.md`。词语、句长和比例只负责定位，不单独构成问题。每个修改都要说明受损效果，并验证修改后仍像这个人物、这个叙述者和这本书。

### 第五步：读者反馈处理（转 novel-feedback）

> 本步转交给 `novel-feedback` skill 处理。
> 读者反馈追踪、分析、决策是独立的专业领域，由 `novel-feedback` 统一处理。

**商业连载模式的触发条件**（满足任一即转交；其他模式不使用追读阈值作为质量门禁）：
- 单章追读率 < 50%
- 连续 2 章追读率下跌
- 读者集中骂某类问题（节奏/角色/感情线/套路）
- 每卷结束后 3 天内
- 用户主动要求"分析读者反馈"

**转交流程**：
1. 将当前体检发现的问题 + 相关章节 + 追读数据传递给 `novel-feedback`
2. `novel-feedback` 执行：收集 → 追踪 → 分析 → 决策 → 输出指令
3. `novel-feedback` 可能向本 skill 返回修订指令（具体改哪章/怎么改）
4. 本 skill 执行修订后，`novel-feedback` 验证效果

**产物要求**：
- `90-运行/决策记录.md` 中记录本次反馈处理决策和验证结果

## 产物要求

按任务生成必要产物：

- 深度体检或发布前精修：`40-修订/体检报告/{chapter}-checkup.md`
- 实际进行了修订：`40-修订/修稿报告/{chapter}-revision.md`
- 更新后的正文

## 硬门禁与软目标

先判断“是否成立”，再判断“这一章要在哪些方面出色”：

1. **硬门禁（全部通过）**：连续性、人物动机、因果链、硬设定、视角控制；失败时必须修复
2. **软目标（按章选择）**：模式与目的、人物真实、压力与情绪运动、视角与信息控制、语言形式与回响；选择 2-3 项，用原文证据判断
3. **状态回写**：硬门禁通过后更新运行状态；软目标不足可记录为下轮训练项

> 不要求每章集齐所有文学效果。过渡章、揭示章、高潮章和实验性章节应采用不同软目标。

## 快速诊断（5 分钟体检）

不确定该用哪个模板时，先回答这 5 个问题：

1. **这章最让读者困惑的是什么？**（逻辑/角色/设定/伏笔）
   - 逻辑问题 → `逻辑修补模板.md`
   - 角色问题 → `细节修补模板.md`
   - 设定问题 → `连续性检查清单.md`
   - 伏笔问题 → `回收修补模板.md`

2. **读者说"最近不想追了"→** `追读排查.md`
3. **读者说"这段像 AI 写的"→** 先让反馈落到具体句子和受损效果，再用 `AI味目录.md` + `去味手法.md`
4. **长篇写到中段觉得"哪里断了"→** `长线扫描模板.md`
5. **不知道先修哪个→** `修订阶梯.md`（按逻辑→回收→细节→句子顺序）

**核心原则**：先诊断，再修稿。不要上来就改句子。

## 常见质量错误

| 错误 | 症状 | 修复模板 |
|------|------|----------|
| 因果断裂 | "然后他就赢了"——怎么赢的？ | `逻辑修补模板.md` |
| 动机悬浮 | "他突然背叛了"——为什么？ | `逻辑修补模板.md` |
| 伏笔丢失 | 埋了 20 章没收 | `回收修补模板.md` |
| 回收太硬 | "他突然想起来……" | `回收修补模板.md` |
| 情绪表达失效 | 情绪命名、动作与语境重复或互相矛盾 | `细节修补模板.md` |
| 情绪悬空 | "她哭了"——没有触发 | `细节修补模板.md` |
| 连续断连 | 上一章的伤这章没了 | `连续性检查清单.md` |
| 章尾失效 | 结尾没有完成当前模式需要的推动、回响、封闭或留白 | `追读排查.md` |
| 语言模板化 | 多处句子可无损换到任何人物、任何场景 | `AI味目录.md` |
| 角色同声 | 所有人说话方式一样 | `去味手法.md` |
| 主线遗忘 | 核心目标 10 章没推进 | `长线扫描模板.md` |
| 追读跌 | 最近几章读者不想追了 | `追读排查.md` |
