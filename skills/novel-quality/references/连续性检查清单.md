# 连续性检查表

优先检查：

- 人物称呼是否稳定
- 视角人物知道与不知道的信息是否稳定
- 时间顺序是否说得通
- 地点移动是否有交代
- 物件是否凭空出现或消失
- 伤势、能力消耗、关系裂痕是否持续生效
- 设定规则是否被正文自己打破

如果是长篇，再补查：

- 支线人物是否无故失踪
- 旧线是否长时间失联又无提醒
- 本章新增的信息债是否记进了后续跟踪表
- 近期最值钱的读点是否被连续几章写散

---

## 常见连续性错误

| 错误 | 表现 | 修复 |
|------|------|------|
| **称呼漂移** | 前章叫"林师兄"，这章叫"林尘"，没交代关系变化 | 统一称呼或加过渡说明 |
| **伤势消失** | 前章主角手臂骨折，这章用手持剑无困难 | 加"忍着痛"或改变动作 |
| **物件凭空出现** | 主角突然拿出从未提及的丹药 | 前章预埋或本章交代来源 |
| **时间跳变** | "三天后"但没有过渡 | 加"三日间……"或时间标记 |
| **地点瞬移** | 主角从 A 城到 B 城无过程 | 加移动过程或时间跳跃说明 |
| **能力超限** | 主角使用了还没学会的能力 | 提前铺垫学习过程 |
| **信息错乱** | 角色 A 知道的事，角色 B 也知道了（没交代） | 补信息传递场景 |

## 快速检查（1 分钟）

读完一章后，问自己：
1. 上一章的伤势/状态/关系，这章还在吗？
2. 这章出现的新物件/能力，之前有过吗？
3. 时间/地点变化有交代吗？

**任一题"没有"→必须修复**
