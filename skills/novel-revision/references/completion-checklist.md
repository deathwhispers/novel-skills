# 完本检查清单

适用于一卷结束或全书收尾时使用。按顺序执行，不要跳跃。

## 第一步：线头收束检查

### 主线检查
- [ ] 主线最终目标是否已经达成或明确失败
- [ ] 主角的成长弧线是否完整（起点→终点有可感知的变化）
- [ ] 全书最大的悬念是否已经解答
- [ ] 所有中期提出的核心问题是否已有答案

### 副线检查
- [ ] 每条打开的副线是否都有收束（不一定完美收束，但不能凭空消失）
- [ ] 所有重要配角的故事线是否都有交代
- [ ] 被长时间搁置的线索是否最少有一次提醒或解释

### 关系线检查
- [ ] 每段核心关系的最终状态是否清晰
- [ ] 读者最在意的那段关系是否有足够篇幅的收束
- [ ] 有没有冗余关系线没有处理

## 第二步：伏笔回收清单

- [ ] pull 出 `20-outline/payoff-tracking/payoff-ledger.md` 逐条检查每一条线
- [ ] 标记每条线的最终状态：已回收 / 有提醒 / 故意留白
- [ ] 未回收的线头中，哪些是续集伏笔，哪些是纯粹遗漏

## 第三步：情绪收束

- [ ] 结局的情绪基调是否符合全书的整体气质
- [ ] 读者读完最后的情绪是否和开书时的承诺一致
- [ ] 最后几章是否有足够的余韵空间（不是信息堆叠）
- [ ] 是否给读者留下了"这本书值得回味"的结尾力量

## 第四步：硬伤排查

- [ ] 检查 10-bible/canon.md 与正文是否冲突
- [ ] 检查 20-outline/causality/scene-causality-map.md 是否所有因果链都已闭合
- [ ] 检查角色名称、地点名称、能力名称在全书中是否一致
- [ ] 检查时间线是否存在矛盾

## 第五步：完本后操作

- [ ] 最后通读一遍，修正错别字和语病
- [ ] 将完成的分卷正文移入 `50-archive/`
- [ ] 在 `90-ops/decisions.md` 记录完本信息
- [ ] 更新 `00-story-core/project-meta.md` 的状态为「已完本」
- [ ] 如果存在续集计划，在 `00-story-core/series-promise.md` 记录衔接思路

## 第六步：续集/新书规划（可选）

- [ ] 留下的续集钩子是否足够让人期待
- [ ] 续集需要的设定变化是否已经记录
- [ ] 新书的构思是否已经独立记录，不被完本情绪带偏
