# 设定总表

## 功能定位

把创意变成可以长期复用的设定资产。所有设定都应服务剧情与角色，而不是为了炫耀百科知识。

## 核心原则

1. 先冲突，后设定。设定只有在影响剧情或角色选择时才值得记录。
2. 先 hard canon，后柔性补充。不可违反的核心规则优先于 flavor text。
3. 人物优先于世界说明。读者通过角色感知世界，不是通过百科。
4. 每次新增设定都要能回答「它会影响什么剧情或选择」。
5. 渐进式丰富。不要开书第一天就把所有设定填满，按阶段逐步深化。

## 主要产物

- `10-设定/硬设定.md` —— 不可违反的世界底层规则
- `10-设定/世界规则.md` —— 能力系统、社会机制、禁忌与代价
- `10-设定/力量体系.md` —— 等级、能力、突破路径
- `10-设定/地点.md` —— 核心场景地点（或独立文件）
- `10-设定/势力.md` —— 核心势力组织（或独立文件）
- `10-设定/时间线.md` —— 历史事件+当前时间
- `10-设定/母题.md` —— 主题、象征、反复出现的意象
- `10-设定/文风指南.md` —— 写作风格、POV规则、语言习惯
- `10-设定/角色/*.md` —— 每个核心角色一个独立文件
- `10-设定/地点模板.md` —— 新增地点时复用的通用模板
- `10-设定/势力模板.md` —— 新增势力时复用的通用模板

## 角色卡最低要求

每个主要角色至少要有（按优先级排序）：

| 字段 | 必须 | 说明 |
|------|------|------|
| role | ✓ | 角色在故事中承担的功能 |
| desire | ✓ | 驱动他行动的核心欲望 |
| fear | ✓ | 他真正不能承受的代价 |
| contradiction | ✓ | 最拧巴的地方 |
| secret | ✓ | 不想让人知道的秘密 |
| leverage | ✓ | 可以拿捏他的软肋 |
| voice markers | ✓ | 说话/做事的辨识度 |
| relationship pressure | ✓ | 与关键角色的关系张力 |
| arc direction | ✓ | 他在这本书里会怎么变 |

## 渐进式丰富的工作流程

### 新建角色
1. 复制 `10-设定/角色/角色卡模板.md` → 命名为 `姓名.md`
2. 先填「基本档案」和「核心驱动力」
3. 随剧情推进逐步补充「性格地图」「关系网络」「状态追踪」
4. 重要角色逐步补全「成长弧线」和「读者投入点」

### 新建地点
1. 复制 `10-设定/地点模板.md` → 命名为 `地点名.md`
2. 先填「基本描述」和「登场情况」
3. 随场景出场逐次补充「感官氛围」和「变化追踪」

### 新建势力
1. 复制 `10-设定/势力模板.md` → 命名为 `势力名.md`
2. 先填「基本档案」「组织架构」「信条」
3. 随剧情推进补充「关系」「内部矛盾」「动态变化」

### 阶段性深化

**开书前**：主角档案 + 卷一反派 + 开场地点 + 一条核心世界规则
**卷一写作中**：出场配角 + 关键场景地点 + 接触到的势力 + 时间线
**中后期**：深度关系 + 地点变化 + 势力动态 + 角色状态追踪
**完本前**：弧线验证 + 时间线核对 + 硬设定回溯 + 归档

## 变更规则

- 如果修订了 hard canon，同步在 `90-运行/决策记录.md` 记录原因。
- 如果只是补充 flavor text，不要伪装成 hard canon。
- 如果用户要求「随便补一点设定」，优先补会影响当前大纲或下一章的设定。
