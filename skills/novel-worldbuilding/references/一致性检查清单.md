# 一致性检查清单

> **用途**：系统性检查设定是否自洽，防止前后矛盾。
> **使用时机**：每卷结束时、完本前、发现疑似矛盾时。

---

## 一、角色一致性

### 性格一致性

- [ ] 角色的行为是否符合其核心驱动力（欲望/恐惧/矛盾）？
- [ ] 角色的决策风格是否前后一致（冲动/权衡/直觉/理性）？
- [ ] 角色在压力下的反应是否一致（战斗/逃跑/僵硬/讨好）？
- [ ] 角色的标志性动作是否持续出现？
- [ ] 角色的口头禅/说话风格是否稳定？

### 能力一致性

- [ ] 角色的能力是否在 hard canon 范围内？
- [ ] 角色的能力使用是否遵守代价系统？
- [ ] 角色的能力升级是否有合理的晋升过程？
- [ ] 角色是否有能力限制被忘记（如只在雨天有效）？

### 关系一致性

- [ ] 角色间的关系状态是否与最新记录一致？
- [ ] 角色对某人的态度变化是否有明确的事件触发？
- [ ] 角色关系的变化是否产生连锁反应？

---

## 二、设定一致性

### 硬设定检查

- [ ] 已写内容是否违反了任何 hard canon 规则？
- [ ] 如果有违反，是否已在变更日志中记录并得到合理解释？
- [ ] 新揭示的规则是否与已有规则冲突？

### 地理一致性

- [ ] 角色的移动是否符合地理距离和通行时间？
- [ ] 不同地点的时间/季节是否一致？
- [ ] 地点的状态变化（被毁/易主）是否在所有相关文件中更新？

### 时间一致性

- [ ] 事件的因果顺序是否正确（先因后果）？
- [ ] 多线叙事的时间是否同步？
- [ ] 角色的年龄/成长时间是否与现实时间线匹配？

### 势力一致性

- [ ] 势力的状态变化是否在所有相关文件中更新？
- [ ] 势力间的关系变化是否有明确的事件触发？
- [ ] 势力的资源/实力是否与描述一致？

---

## 三、剧情一致性

### 伏笔回收

- [ ] 已埋下的伏笔是否都有回收计划？
- [ ] 已回收的伏笔是否满足读者预期？
- [ ] 未回收的伏笔是否已标记为"废弃"？

### 信息一致性

- [ ] 角色知道的信息是否与读者知道的一致（或合理地不一致）？
- [ ] 信息的传递时间是否合理（没有未卜先知）？
- [ ] 角色基于信息的决策是否合理？

### 后果一致性

- [ ] 角色的行动是否产生了合理的后果？
- [ ] 重要决策是否有长期影响（不是决定后就消失）？
- [ ] 世界的变化是否反映了剧情事件的后果？

---

## 四、风格一致性

### Voice 一致性

- [ ] 叙述距离是否稳定？
- [ ] 主角的标志性动作是否持续出现？
- [ ] 禁用语是否被遵守？
- [ ] 对话风格是否因角色而异？

### 氛围一致性

- [ ] 场景的氛围描述是否与地点卡一致？
- [ ] 情感基调是否与母题系统一致？
- [ ] 节奏变化是否符合卷级节奏型？

---

## 五、检查流程

### 每卷检查（30 分钟）

1. 通读本卷，标记疑似矛盾
2. 对照设定文件核实
3. 如有矛盾，决定：修正正文 / 修正设定 / 记录为例外
4. 更新一致性记录

### 完本检查（2-4 小时）

1. 通读全文，标记所有疑似矛盾
2. 系统性检查每个检查项
3. 整理矛盾清单，按优先级修复
4. 更新所有设定文件到最终版本

---

## 六、矛盾处理决策树

```
发现矛盾
  → 是硬设定矛盾？
    → 是 → 必须修复（修改正文或变更 hard canon 并记录）
    → 否 → 是 flavor text 矛盾？
      → 是 → 可忽略或轻微修正
      → 否 → 是伏笔矛盾？
        → 是 → 决定回收或废弃
        → 否 → 记录并评估影响
```

---

## 使用提示

1. **每卷结束**：执行快速检查（30 分钟）
2. **发现矛盾**：立即标记，不要等
3. **修复矛盾**：优先修复 hard canon 矛盾
4. **完本前**：执行系统性检查
5. **严禁**：发现矛盾后假装没看见——矛盾会累积
