# 世界观构建指南

> **用途**：从零开始搭建小说世界的顶层框架，确保世界观自洽、有层次、能驱动剧情。
> **使用时机**：立项后、写第 1 章前。
> **预计耗时**：2-4 小时（只需搭建骨架，细节随写随补）。

---

## 一、核心原则

**先冲突，后设定。** 设定只有让角色的选择变难时才有价值。

```
冲突需求 → 设定支撑 → 剧情验证
```

| 原则 | 说明 | 反例 |
|------|------|------|
| 先冲突后设定 | 先确定核心矛盾，再设计支撑它的世界规则 | 先写 5000 字世界历史，再发现用不上 |
| 先 hard canon 后 flavor | 不可违反的规则优先，细节可以后补 | 花三天设计货币体系，主角根本不用钱 |
| 人物优先于世界 | 读者通过角色感知世界，不是百科全书 | 开篇 3 章纯世界说明，主角还没出场 |
| 渐进式丰富 | 开书只建骨架，写到哪补到哪 | 开书第一天就把所有设定填满 |

## 三、从零构建五步流程

### Step 1：确定世界类型（30 分钟）

回答三个问题：
1. **规模**：个人冒险 / 国家斗争 / 世界存亡 / 宇宙战争？
2. **基调**：光明 / 灰暗 / 残酷 / 荒诞 / 史诗？
3. **核心差异**：这个世界与现实最大的不同是什么？

### Step 2：画权力地图（1 小时）

1. 列出所有势力（至少 3 个）：
   - 势力 A：欲望 __ / 恐惧 __ / 资源 __ / 弱点 __
   - 势力 B：欲望 __ / 恐惧 __ / 资源 __ / 弱点 __
   - 势力 C：欲望 __ / 恐惧 __ / 资源 __ / 弱点 __

2. 画出势力关系网：
   ```
   A ←[盟友]→ B ←[敌对]→ C
   ```

3. 找出**结构性矛盾**（不可调和的利益冲突）：____

### Step 3：锁死 Hard Canon（30 分钟）

写出 5-10 条绝对不可违反的规则：

| 类型 | 规则 | 约束对象 |
|------|------|---------|
| 物理上限 | | |
| 社会禁忌 | | |
| 资源限制 | | |
| 绝对代价 | | |

> 这些规则将约束后续所有创作，防止战力崩坏。

### Step 4：设计主角的初始位置（30 分钟）

- **社会层级**：底层 / 中层 / 顶层？
- **资源状况**：有 __ / 缺 __
- **信息状况**：知道 __ / 被 __ 蒙蔽
- **行动空间**：能做 __ / 不能做 __

### Step 5：建立最小可运转集（1 小时）

创建以下文件（仅填核心字段，其余留白）：

1. `硬设定.md` —— 5-10 条规则
2. `主角.md` —— 欲望、恐惧、矛盾、秘密
3. `核心冲突.md` —— 对立双方 + 争夺对象 + 不可调和性
4. `开场地点.md` —— 感官细节 + 叙事功能
5. `初始势力.md` —— 欲望、恐惧、与主角关系

> **红线**：这 5 个文件不完成，不开写第 1 章。

## 四、常见陷阱与修复

| 陷阱 | 症状 | 修复 |
|------|------|------|
| 百科全书陷阱 | 设定写 5000 字，第 1 章还没开始 | 问：这个设定会让谁的选择变难？ |
| 铁板一块陷阱 | 势力内部没有矛盾 | 每个势力至少设计 1 个内部裂痕 |
| 免费超能力陷阱 | 力量没有代价，战力无限膨胀 | 每次突破必须有身体/精神/关系/道德代价 |
| 时间混乱陷阱 | 不同地点事件时间对不上 | 建立统一时间轴 |
| 文化真空陷阱 | 世界只有政治和战斗，没有日常 | 设计 3 个日常细节（饮食/节日/禁忌） |
| 设定孤岛陷阱 | 各设定间无关联 | 每个新设定必须链接到已有 2 个设定 |

## 五、世界的动态演化

世界不是静态的——它应该随剧情**被改变**。

**变化触发器**：

| 类型 | 示例 | 追踪方式 |
|------|------|---------|
| 主角造成的 | 主角推翻帝国 | `世界状态变更日志.md` |
| 独立发生的 | 天灾、政变 | `势力兴衰曲线.md` |
| 揭示的 | 原本存在的真相 | `地点变化追踪.md` |

## 六、自检清单

开写前检查：
- [ ] 核心冲突能让主角至少做 10 次重大选择
- [ ] 硬设定不超过 10 条（太多记不住）
- [ ] 主角有明确的欲望和恐惧
- [ ] 至少 1 个势力与主角立场对立
- [ ] 世界有 3 个可感知的日常细节
- [ ] 代价系统已经定义（获得力量的必须代价）

---

## 使用提示

1. **不要一次性填满**：Step 5 的 5 个文件完成后即可开写
2. **写到哪补到哪**：新角色出场 → 建角色卡；新势力出场 → 建势力卡
3. **定期回溯**：每卷结束时检查 hard canon 是否被违反
4. **记录变更**：任何 hard canon 的修改必须记录原因
5. **保持神秘**：底层设定（作者掌握的）不要完全揭示给读者
