# 力量体系设计指南

> **使用说明**：设计一套自洽、有代价、可升级的力量体系。
> 力量体系是**冲突引擎**——不是为了让角色变强，而是为了制造选择困境。

---

## 一、力量体系的三要素

```
来源 → 表现 → 代价
```

| 要素 | 问题 | 示例 |
|------|------|------|
| **来源** | 力量从哪里来？ | 内力、血统、科技、信仰、契约 |
| **表现** | 力量如何显现？ | 火焰、念力、变形、预知 |
| **代价** | 使用力量要付出什么？ | 寿命、理智、关系、道德 |

> **规则**：缺少任何一个要素，力量体系都会崩坏。
> **核心原则**：没有免费的力量。每次使用都必须有代价。

---

## 二、来源设计

### 来源类型
| 类型 | 特点 | 适合的故事 | 风险 |
|------|------|-----------|------|
| **修炼型** | 后天努力，等级分明 | 武侠、修仙 | 升级疲劳 |
| **血统型** | 先天注定，无法选择 | 奇幻、超英 | 宿命论 |
| **科技型** | 外部装备，可复制 | 科幻 | 装备依赖 |
| **信仰型** | 与神/灵契约 | 宗教、暗黑 | 信仰崩溃 |
| **情感型** | 情绪驱动，不稳定 | 奇幻、言情 | 逻辑混乱 |
| **知识型** | 理解规则即力量 | 克苏鲁、科幻 | 门槛过高 |

### 来源的限制
- 来源是否可以被剥夺？
- 来源是否可以被转移？
- 来源是否有副作用？
- 不同来源之间是否冲突？

### 等级设计原则
1. **金字塔结构**：等级越高，人数越少（避免顶尖战力泛滥）
2. **能力跃迁**：每个等级要有质变（不是量变）
3. **晋升仪式**：晋升需要标志性事件（考试、战斗、觉醒）
4. **晋升代价**：每次晋升都要失去什么（ Innocence / 关系 / 身体部分）

### 等级与剧情的绑定
| 等级 | 对应的剧情阶段 | 主角的目标 | 对应的对手等级 |
|------|---------------|-----------|---------------|
| 1-2 | 开局 | 生存、学习 | 2-3 |
| 3-4 | 发展 | 建立势力、复仇 | 4-5 |
| 5 | 高潮 | 改变世界 | 5 |

## 四、代价系统设计

### 代价类型
| 类型 | 描述 | 示例 |
|------|------|------|
| **身体代价** | 使用力量伤害身体 | 减寿、残疾、变异 |
| **精神代价** | 使用力量侵蚀心智 | 疯狂、失忆、人格分裂 |
| **关系代价** | 使用力量伤害他人 | 亲友受伤、被孤立 |
| **道德代价** | 使用力量需要作恶 | 杀人、背叛、牺牲无辜 |
| **资源代价** | 使用力量消耗稀缺资源 | 灵石、生命力、时间 |

### 代价的层次
- **轻微代价**：偶尔使用无大碍（日常使用）
- **中度代价**：频繁使用会累积伤害（战斗中使用）
- **重度代价**：极限使用会永久损伤或死亡（绝境中使用）

### 代价与选择的绑定
> 最好的力量体系制造"没有完美选择"的困境：
- 用力量救人 → 代价是伤害自己
- 不用力量 → 代价是看着人死
- 寻找第三种方法 → 代价是冒险和时间

|---------|---------|---------|---------|------|-----------|
| | | | | | |

### 能力设计原则
1. **每个能力都有反制方法**（没有无敌的能力）
2. **能力有使用场景限制**（下雨天不能火攻）
3. **能力之间有克制关系**（水克火，但火也可以蒸发水）
4. **能力的强度与代价成正比**

### 组合能力
> 当多个能力组合时，产生新的效果或代价：
- 能力 A + 能力 B = 新效果
- 能力 A + 能力 B = 代价倍增

## 六、力量体系的平衡检查

### 战力膨胀检查
- [ ] 顶级战力有明确上限（不能毁灭世界）
- [ ] 低级战力在特定条件下可以战胜高级战力
- [ ] 战力差距不是绝对的（情报、准备、环境可以弥补）
- [ ] 战力升级速度合理（不是一卷从 1 级跳到 10 级）

### 代价一致性检查
- [ ] 每次使用力量的代价都被记录
- [ ] 累积代价有显现（频繁使用的角色身体/精神恶化）
- [ ] 代价不能被轻易消除（没有万能解药）
- [ ] 代价驱动剧情（角色因为代价而不得不做选择）

### 体系完整性检查
- [ ] 力量的来源、表现、代价三者一致
- [ ] 不同角色的力量体系可以比较
- [ ] 力量体系有历史（谁创造的？为什么？）
- [ ] 力量体系有文化（不同地区对力量的看法不同）

## 使用提示

1. **开书前**：确定来源类型、设计 3-5 个等级、确定核心代价
2. **新增能力时**：检查是否与现有体系冲突
3. **角色升级时**：记录代价的累积
4. **战斗设计时**：利用能力限制和克制关系制造智斗
5. **严禁**：战力无限膨胀——每次升级必须有对应的代价和限制
