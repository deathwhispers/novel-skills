# 设定实战指南

> **使用说明**：按时间线管理设定文件的创建和更新。
> 不是"开书第一天就建完所有设定"——是"按阶段逐步丰富"。
> 适用于 novel-worldbuilding 的渐进式丰富工作流程。

## 设定建设总原则

```
先冲突，后设定 → 先 hard canon，后 flavor text
人物优先于世界说明 → 每次新增设定都要能回答"它会影响什么剧情或选择"
渐进式丰富 → 不要开书第一天就把所有设定填满
```

---

## 第一阶段：开书前（必须完成）

> **时间**：立项后、写第 1 章前
> **目标**：建立最小可执行的设定集
> **预计时间**：2-4 小时

### 必须建立的文件（按优先级）

| 优先级 | 文件 | 必须填写的字段 | 原因 |
|--------|------|----------------|------|
| P0 | `00-书核/立项单.md` | 核心冲突、主角、触发事件 | 没有 = 不知道写什么 |
| P0 | `10-设定/硬设定.md` | 3-5 条不可违反的规则 | 防止写到一半自相矛盾 |
| P0 | `10-设定/角色/主角.md` | desire、fear、contradiction | 主角行动的逻辑基础 |
| P1 | `10-设定/角色/卷一反派.md` | role、desire、leverage | 冲突需要对立面 |
| P1 | `10-设定/世界规则.md` | 1 条核心规则 + 代价 | 世界运行的基础 |
| P2 | `10-设定/力量体系.md` | 等级划分（如有） | 升级文必备 |
| P2 | `10-设定/地点/开场地点.md` | 基本描述 + 登场情况 | 第 1 章需要 |
| P2 | `10-设定/文风指南.md` | 叙述距离、禁用语 | 保持 voice 一致 |

### 第一阶段检查清单

- [ ] 主角有明确的欲望和恐惧
- [ ] 有至少 1 个反派/阻力
- [ ] 有 3-5 条不可违反的规则
- [ ] 第 1 章的场景已定义
- [ ] 知道主角的初始状态和终点状态

> **红线**：P0 文件不填完，不开写第 1 章。

---

## 第二阶段：卷一写作中（按需补充）

> **时间**：写第 1-30 章期间
> **目标**：为出场的内容建立设定
> **原则**：写到哪，设定补到哪

### 触发条件 → 建立文件

| 触发条件 | 建立/更新文件 | 内容 |
|----------|--------------|------|
| 新角色出场 | `10-设定/角色/角色名.md` | 基本档案 + 核心驱动力 |
| 新地点出场 | `10-设定/地点/地点名.md` | 基本描述 + 登场情况 |
| 新势力出场 | `10-设定/势力/势力名.md` | 基本档案 + 组织架构 + 信条 |
| 新规则揭示 | `10-设定/世界规则.md` | 追加规则 + 代价 |
| 历史事件提及 | `10-设定/时间线.md` | 追加历史事件 |
| 能力升级 | `10-设定/力量体系.md` | 追加等级/能力描述 |
| 发现伏笔线索 | `20-大纲/回收/回收总账.md` | 记录伏笔等级和计划回收 |

### 第二阶段检查清单（每 10 章检查一次）

- [ ] 所有出场角色有角色卡（主角 + 配角）
- [ ] 所有出场地点有描述
- [ ] 所有出场势力有基本设定
- [ ] 时间线无矛盾
- [ ] 硬设定未被违反（如有违反，记录到 `90-运行/决策记录.md`）

---

## 第三阶段：中后期（深度完善）

> **时间**：写到卷 3+，角色/势力/地点大量出场后
> **目标**：建立横向追踪文件，防止设定失控
> **原则**：横向关系比单个设定更重要

### 触发条件 → 建立横向追踪文件

| 触发条件 | 建立文件 | 原因 |
|----------|----------|------|
| 角色超过 10 个 | `10-设定/角色关系矩阵.md` | 防止关系混乱 |
| 角色超过 10 个 | `10-设定/角色阵营归属表.md` | 防止立场混乱 |
| 势力超过 5 个 | `10-设定/势力关系矩阵.md` | 防止势力关系混乱 |
| 势力超过 5 个 | `10-设定/势力动态史.md` | 防止势力设定前后矛盾 |
| 关键地点超过 5 个 | `10-设定/地点距离表.md` | 防止地理矛盾 |
| 世界嵌套 4 层以上 | `10-设定/世界层级图.md` | 防止层级混乱 |
| 大事件涉及多地点 | `10-设定/关键事件地理足迹表.md` | 防止时间/地理矛盾 |
| 反派体系复杂 | `10-设定/反派/反派体系图.md` | 防止反派关系混乱 |
| 反派跨卷升级 | `10-设定/反派/反派递进阶梯.md` | 防止反派强度失控 |

### 横向追踪文件的使用方法

**角色关系矩阵**：
```
        林尘  苏晚  李昭  赵长老  ...
林尘     —    盟友   敌人   师徒   ...
苏晚    盟友   —    中立   疏远   ...
李昭    敌人  中立   —    利用   ...
...
```

**势力关系矩阵**：
```
        青云宗  魔教   朝廷   ...
青云宗   —     敌对   合作   ...
魔教    敌对   —     敌对   ...
...
```

### 第三阶段检查清单（每卷结束后）

- [ ] 角色关系矩阵已更新（新增角色 + 关系变化）
- [ ] 势力关系矩阵已更新（势力变化 + 新联盟/敌对）
- [ ] 地点距离表已更新（新地点 + 距离计算）
- [ ] 时间线已更新（新事件 + 时间顺序检查）
- [ ] 角色弧线追踪表已更新（角色状态变化）

---

## 第四阶段：完本前（验证与归档）

> **时间**：写到最后一卷
> **目标**：验证所有设定一致性，归档

### 完本前必须做的事

| 任务 | 文件 | 检查内容 |
|------|------|----------|
| 弧线验证 | `10-设定/角色/角色弧线追踪表.md` | 每个角色的起点和终点是否匹配 |
| 时间线核对 | `10-设定/时间线.md` | 所有事件的时间顺序是否正确 |
| 硬设定回溯 | `10-设定/硬设定.md` | 全文是否违反了硬设定 |
| 伏笔回收检查 | `20-大纲/回收/回收总账.md` | 所有伏笔是否已回收或明确废弃 |
| 地理一致性 | `10-设定/地点距离表.md` | 所有路程描述是否一致 |
| 关系一致性 | `10-设定/角色关系矩阵.md` | 最终关系状态是否合理 |

### 完本前检查清单

- [ ] 主角弧线完整（起点 → 终点，有变化）
- [ ] 主要配角弧线完整
- [ ] 反派弧线完整（不是"突然变弱"）
- [ ] 硬设定无违反
- [ ] 伏笔全部回收（或明确标记"未回收"）
- [ ] 时间线无矛盾
- [ ] 地理无矛盾
- [ ] 关系无矛盾

## 设定更新纪律

### 何时更新

| 情况 | 更新动作 | 记录到 |
|------|----------|--------|
| 新设定出场 | 新建文件 | — |
| 设定细节补充 | 追加内容 | 文件末尾更新记录 |
| 硬设定变更 | 修改文件 + 记录原因 | `90-运行/决策记录.md` |
| 发现设定矛盾 | 修正矛盾 + 记录 | `90-运行/决策记录.md` |
| 角色关系变化 | 更新矩阵 | 关系矩阵文件 |
| 势力变化 | 更新矩阵 + 动态史 | 势力文件 |

### 何时不更新

- ** flavor text 补充**：不影响剧情的小细节，直接补，不记录
- **设定深化**：同一设定的更详细描述，直接补，不记录
- ** typo 修正**：错别字/格式修正，直接改，不记录

### 何时必须记录到决策记录

- **hard canon 变更**：任何不可违反规则的修改
- **角色核心设定变更**：欲望/恐惧/身份等核心字段修改
- **时间线重大调整**：影响多章节的事件时间修改
- **关系状态重大变化**：盟友变敌人/敌人变盟友等

## 设定与写作的协作流程

```
写作前
  ↓
查设定：本章涉及的角色的卡、地点描述、硬设定
  ↓
写作中
  ↓
新设定出场？→ 暂停写作，先建设定文件
  ↓
写作后
  ↓
更新设定：角色状态变化、新伏笔、新信息债
  ↓
检查一致性：硬设定是否被遵守、关系是否更新
```

> **核心规则**：设定不是"写完再补"——是"写到哪补到哪"。
> 但也不要"为了设定而设定"——设定服务于剧情和角色选择。
