# 渐进式丰富指南

> **用途**：按阶段逐步构建设定，不是开书第一天就填满所有内容。
> **核心原则**：设定随剧情推进而深化——写到哪，补到哪。

---

## 一、总体原则

```
先冲突，后设定 → 先 hard canon，后 flavor text
人物优先于世界说明 → 每次新增设定都要能回答"它会影响什么剧情或选择"
渐进式丰富 → 不要开书第一天就把所有设定填满
```

---

## 二、四阶段建设流程

### 第一阶段：开书前（必须完成）

> **时间**：立项后、写第 1 章前
> **目标**：建立最小可执行的设定集
> **预计时间**：2-4 小时

#### 必须建立的文件（按优先级）

| 优先级 | 文件 | 必须字段 | 原因 |
|--------|------|---------|------|
| P0 | `硬设定.md` | 3-5 条不可违反的规则 | 防止自相矛盾 |
| P0 | `主角.md` | desire、fear、contradiction | 行动逻辑基础 |
| P1 | `卷一反派.md` | role、desire、leverage | 冲突需要对立面 |
| P1 | `世界规则.md` | 1 条核心规则 + 代价 | 世界运行基础 |
| P2 | `力量体系.md` | 等级划分（如有） | 升级文必备 |
| P2 | `开场地点.md` | 基本描述 + 登场情况 | 第 1 章需要 |
| P2 | `文风指南.md` | 叙述距离、禁用语 | 保持 voice 一致 |

#### 第一阶段检查清单

- [ ] 主角有明确的欲望和恐惧
- [ ] 有至少 1 个反派/阻力
- [ ] 有 3-5 条不可违反的规则
- [ ] 第 1 章的场景已定义
- [ ] 知道主角的初始状态和终点状态

> **红线**：P0 文件不完成，不开写第 1 章。

---

### 第二阶段：卷一写作中（按需补充）

> **时间**：写第 1-30 章期间
> **目标**：为出场的内容建立设定
> **原则**：写到哪，设定补到哪

#### 触发条件 → 建立文件

| 触发条件 | 建立/更新文件 | 内容 |
|----------|--------------|------|
| 新角色出场 | `角色/角色名.md` | 基本档案 + 核心驱动力 |
| 新地点出场 | `地点/地点名.md` | 基本描述 + 登场情况 |
| 新势力出场 | `势力/势力名.md` | 基本档案 + 组织架构 + 信条 |
| 新规则揭示 | `世界规则.md` | 追加规则 + 代价 |
| 历史事件提及 | `时间线.md` | 追加历史事件 |
| 能力升级 | `力量体系.md` | 追加等级/能力描述 |
| 发现伏笔线索 | `回收/回收总账.md` | 记录伏笔等级和计划回收 |

#### 第二阶段检查清单（每 5 章检查一次）

- [ ] 新出场角色有基本档案
- [ ] 新出场地点有感官描述
- [ ] 新揭示的规则已记录
- [ ] 硬设定未被违反（如有违反，记录原因）

---

### 第三阶段：中后期（横向追踪）

> **时间**：第 3 卷起
> **目标**：防止设定膨胀导致的混乱
> **原则**：当某类设定超过阈值时，建立横向追踪文件

#### 横向追踪触发条件

| 触发条件 | 必须补建 | 用途 |
|---------|---------|------|
| 角色超过 10 个 | `角色关系矩阵.md` | 防止关系混乱 |
| 角色超过 10 个 | `角色阵营归属表.md` | 防止立场混乱 |
| 势力超过 5 个 | `势力关系矩阵.md` | 防止势力关系混乱 |
| 势力超过 5 个 | `势力动态史.md` | 防止势力设定矛盾 |
| 关键地点超过 5 个 | `地点距离表.md` | 防止地理矛盾 |
| 世界嵌套 4 层以上 | `世界层级图.md` | 防止层级混乱 |
| 大事件涉及多地点 | `关键事件地理足迹.md` | 防止时间/地理矛盾 |
| 反派体系复杂 | `反派/反派体系图.md` | 防止反派关系混乱 |
| 反派跨卷升级 | `反派/反派递进阶梯.md` | 防止反派强度失控 |

#### 第三阶段定期检查（每卷结束时）

- [ ] 角色数量是否触发横向追踪？
- [ ] 势力数量是否触发横向追踪？
- [ ] 地点数量是否触发横向追踪？
- [ ] 所有设定文件是否最新？

---

### 第四阶段：完本前（一致性验证）

> **时间**：最后 1-2 卷
> **目标**：确保所有设定闭合，没有烂尾
> **原则**：系统性检查，不是凭记忆

#### 完本前验证清单

| 检查项 | 检查方法 | 通过标准 |
|--------|---------|---------|
| 弧线验证 | `角色/角色弧线追踪表.md` | 每个角色的起点和终点匹配 |
| 时间线核对 | `时间线.md` | 所有事件时间顺序正确 |
| 硬设定回溯 | `硬设定.md` | 全文没有违反硬设定 |
| 伏笔回收 | `回收/回收总账.md` | 所有伏笔已回收或明确废弃 |
| 地理一致性 | `地点距离表.md` | 所有路程描述一致 |
| 关系一致性 | `角色关系矩阵.md` | 最终关系状态合理 |

#### 完本前必须更新的文件

- [ ] 所有角色的最终状态
- [ ] 所有势力的最终状态
- [ ] 时间线的最终版本
- [ ] 设定变更日志的归档

---

## 三、设定优先级系统

### P0（必须立即完成）
- 硬设定（3-5 条规则）
- 主角核心驱动力
- 核心冲突定义

### P1（开书前完成）
- 卷一反派
- 世界核心规则
- 开场地点

### P2（卷一写作中补充）
- 力量体系框架
- 文风指南
- 初始势力

### P3（中后期补充）
- 完整角色档案
- 势力详细历史
- 完整时间线

### P4（完本前验证）
- 横向追踪文件
- 一致性检查
- 设定归档

## 四、常见错误

| 错误 | 症状 | 修复 |
|------|------|------|
| 设定先行 | 写了 100 条设定，还没开写 | 只建 P0，其余随写随补 |
| 设定孤岛 | 各设定间无关联 | 每个新设定必须链接到已有 2 个设定 |
| 设定膨胀 | 设定文件比正文还多 | 删除不影响剧情的设定 |
| 设定矛盾 | 后文与前文设定冲突 | 建立硬设定，违反时记录原因 |
| 设定遗忘 | 写了设定但后文没用 | 定期回溯，删除未使用的设定 |

---

## 使用提示

1. **开书前**：完成 P0 和 P1
2. **卷一写作中**：按需补充，不提前建设
3. **中后期**：触发横向追踪，防止混乱
4. **完本前**：系统性验证，不留漏洞
5. **严禁**：一次性建完所有设定——渐进式是核心原则
