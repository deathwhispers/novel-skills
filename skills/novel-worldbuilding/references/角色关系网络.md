# 角色关系网络

> **用途**：追踪所有角色之间的动态关系，以及关系随剧情的推进和变化。
> **核心原则**：关系是活的——会升温、降温、断裂、修复、变异。

---

### 核心关系矩阵

| 角色 A | 角色 B | 关系性质 | 初始亲密度 | 当前亲密度 | 关系弧线 |
|--------|--------|---------|-----------|-----------|---------|
| | | | | | |

**关系性质分类**：
- 盟友 / 敌对 / 师徒 / 恋人 / 主仆 / 竞争者 / 亲人 / 利用 / 其他

**亲密度尺度**：
- -5：死敌（不死不休）
- -3：敌对（利益冲突）
- -1：冷淡（互不关心）
- 0：中立（陌生人）
- +1：认识（泛泛之交）
- +3：亲近（信任/友谊）
- +5：生死之交（愿意为对方牺牲）

## 四、关系推进节奏

### 卷级关系推进

| 卷数 | 推进的关系对 | 推进方式 | 预期读者情绪 |
|------|------------|---------|-------------|
| 卷一 | | | |
| 卷二 | | | |

### 关系推进的 5 种方式

1. **共同事件**：一起经历危险/成功/失败
2. **信息交换**：分享秘密/谎言被揭穿
3. **牺牲行为**：为对方付出代价
4. **背叛/欺骗**：关系破裂的触发器
5. **第三方介入**：新角色改变原有关系动态

### 变化的连锁反应

> 当 A 和 B 的关系变化时，检查是否影响 A-C、B-C 的关系：

| 变化源 | 直接影响 | 间接影响 | 三级影响 |
|--------|---------|---------|---------|
| A↔B 破裂 | A 需要新盟友 | C 趁机接近 A | D 感到威胁 |

## 六、关系网络检查

### 常见问题

- [ ] 关系推进只依赖对话，缺乏共同事件支撑
- [ ] 配角之间的关系统统通过主角传递
- [ ] 长期没有新关系注入或旧关系变异
- [ ] 关系状态长时间不变，失去张力
- [ ] 关系变化缺乏铺垫，显得突兀

### 健康的关系网络特征

- [ ] 至少 3 对关系在同时推进
- [ ] 每卷至少有 1 段关系发生质变
- [ ] 配角之间也有独立的关系线
- [ ] 关系变化有连锁反应
- [ ] 读者能预测关系走向（但猜不到具体方式）

---

## 使用提示

1. **开书前**：画出主角周围的 5-7 个核心关系
2. **每卷规划**：确定本卷要推进哪 2-3 段关系
3. **写场景时**：检查场景中有多少段关系在同时变化
4. **制造高潮时**：让关系变化成为高潮的燃料
5. **严禁**：关系网是静态的——每卷必须有关系变质
