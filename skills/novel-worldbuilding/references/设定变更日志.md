> **核心原则**：设定不是一成不变的——但变化必须被记录。

### 变更类型

- **新增**：创建新的设定文件
- **修改**：修改已有设定的内容
- **删除**：删除不再需要的设定
- **合并**：将多个设定合并为一个
- **拆分**：将一个设定拆分为多个

## 二、Hard Canon 变更规则

### 变更前必须回答

1. **为什么必须改？**（不修改的后果是什么？）
2. **变更后是否更符合剧情？**
3. **变更会影响多少已写内容？**
4. **是否有更简单的替代方案？**

### 变更后必须做

1. 在变更日志中记录
2. 更新所有受影响的文件
3. 检查已写章节是否需要修改
4. 在 `90-运行/决策记录.md` 中记录原因

### 禁止的变更

- [ ] 为了"让角色更强"而修改 hard canon
- [ ] 为了"解决当前困境"而临时添加新规则
- [ ] 未经记录的秘密变更
- [ ] 变更后不更新受影响的内容

| | | | |

### 连锁反应检查

> 当 A 设定变更时，检查是否影响 B、C、D：

```
A（硬设定）变更
  → B（力量体系）是否需要调整？
  → C（角色能力）是否需要调整？
  → D（已写剧情）是否需要修改？
```

## 五、变更自检

每次变更后检查：
- [ ] 变更已在日志中记录
- [ ] 所有受影响文件已更新
- [ ] 已写章节已检查是否需要修改
- [ ] 变更原因已记录
- [ ] 变更没有引入新的矛盾

---

## 使用提示

1. **每次修改设定**：立即记录到变更日志
2. **修改 hard canon**：必须经过影响评估
3. **定期回顾**：每月回顾一次变更日志，检查是否有遗漏
4. **完本前**：审查所有变更，确保一致性
5. **严禁**：悄悄修改设定不记录——这是设定矛盾的根源
