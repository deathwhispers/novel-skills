# 设定圣经地图与渐进式丰富指南

## 文件结构

```
10-设定/
├── 角色/
│   ├── 角色卡模板.md     ← 通用模板，新增角色时复制
│   ├── 主角.md           ← 主角角色卡（深度+追踪）
│   ├── 配角.md           ← 重要配角卡
│   └── 反派.md           ← 反派卡（含对决阶梯）
├── 地点模板.md            ← 新增地点时复制
├── 势力模板.md            ← 新增势力时复制
├── 地点.md                ← 核心地点汇总（或独立文件）
├── 势力.md                ← 核心势力汇总（或独立文件）
├── 世界规则.md            ← 世界底层规则
├── 力量体系.md            ← 能力、等级、代价
├── 时间线.md              ← 历史+当前时间
├── 母题.md                ← 主题与象征
├── 硬设定.md              ← 不可违反的设定
└── 文风指南.md            ← 写作风格与语言习惯
```

## 渐进式丰富策略

设定不是开书第一天就写满的。按优先级分阶段完善。

### 第一阶段：开书前必须填（骨架级）

- 主角.md：基本档案 + 核心驱动力 + 性格地图
- 反派.md（第一卷的）：基本档案 + 核心欲望 + 压迫方式
- 一个开场地点：基本描述 + 登场情况
- 世界规则.md：最核心的一条规则

这一阶段的目标是「能开写」，不是「世界观百科全书」。

### 第二阶段：写第一卷时补充（血肉级）

- 配角.md：第一卷出现的配角
- 地点.md：每新增一个关键场景，记录其描述和氛围
- 势力.md：主角接触到的势力，记录关系和内部矛盾
- 时间线.md：记录已发生的关键事件
- 力量体系.md：主角正在使用的能力

这一阶段的目标是「不写丢」，随写随记。

### 第三阶段：进入中后期（网络级）

- 深度关系：角色关系表模板中的「张力来源」开始发挥
- 地点变化：关键地点随剧情产生变化
- 势力动态：势力内部矛盾和联盟变化
- 角色追踪表：主角和关键配角的状态变更日志
- 伏笔网络：回收总账和场景因果图的联动

这一阶段的目标是「线不乱」，靠记录维持长篇一致性。

### 第四阶段：完本/修订（闭环级）

- 弧线验证：所有角色的成长弧线是否完整
- 时间线核对：全本时间线是否一致
- 硬设定回溯：检查设定与正文是否有冲突
- 归档：不再活跃的角色和地点文件存档

这一阶段的目标是「不烂尾」，靠检查保证质量。

## 角色卡最低标准

每个主要角色至少要有（按优先级排序）：

| 字段 | 必须 | 说明 |
|------|------|------|
| role | ✓ | 角色在故事中承担的功能 |
| desire | ✓ | 驱动他行动的核心欲望 |
| fear | ✓ | 他真正不能承受的代价 |
| contradiction | ✓ | 最拧巴的地方 |
| secret | ✓ | 不想让人知道的秘密 |
| leverage | ✓ | 可以拿捏他的软肋 |
| voice markers | ✓ | 说话/做事的辨识度 |
| relationship pressure | ✓ | 与关键角色的关系张力 |
| arc direction | ✓ | 他在这本书里会怎么变 |

## 常见陷阱

- 把设定当成百科全书来写，而不是为剧情服务
- 一次性把所有设定写完，结果写正文时全部推翻
- 角色只有功能没有欲望（如：配角只为主角存在）
- 地点只有名字没有氛围（如：场景只是一个背景板）
- 势力只有标签没有内部矛盾（如：铁板一块的组织不存在）
- 设定写了不追踪（如：写了一个伏笔但忘了回收）
