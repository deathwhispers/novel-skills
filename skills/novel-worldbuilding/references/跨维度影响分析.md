# 跨维度影响分析指南

> **用途**：分析世界观各维度之间的相互影响，确保设定不是孤立的岛屿，而是相互关联的网络。
> **核心原则**：改变一个设定，必然影响其他设定。忽略这种影响会导致设定矛盾。

---

## 一、影响分析矩阵

### 六维度交叉影响表

用此表检查：当你修改某个维度的设定时，其他维度是否需要同步调整？

| 修改的维度 | 受影响的维度 | 影响方式 | 需要调整的内容 |
|-----------|------------|---------|---------------|
| 物理世界（新增一种魔法金属） | 经济 | 货币体系 | 新增货币/资源定价 |
| | 社会 | 阶层结构 | 掌握矿脉的势力崛起 |
| | 力量 | 体系规则 | 该金属对力量的影响 |
| 社会世界（废除奴隶制） | 经济 | 劳动力来源 | 替代劳动力的出现 |
| | 历史 | 重大事件 | 记录废奴运动 |
| | 势力 | 内部矛盾 | 依赖奴隶的势力衰落 |
| 历史世界（发现古代超级武器） | 势力 | 格局变化 | 拥有武器的势力崛起 |
| | 价值 | 伦理困境 | 是否使用该武器？ |
| | 核心冲突 | 升级 | 冲突从人际升级为存亡 |
| 规则世界（力量代价翻倍） | 角色 | 能力档案 | 所有角色重新评估代价 |
| | 战斗 | 系统规则 | 战斗策略需调整 |
| | 经济 | 资源消耗 | 治疗/恢复资源涨价 |
| 价值世界（主流价值观从秩序转向自由） | 社会 | 法律结构 | 法律宽松化 |
| | 势力 | 合法性 | 维护秩序的势力失势 |
| | 角色 | 立场 | 守序角色面临选择 |
| 神秘世界（证实神不存在） | 社会 | 信仰体系 | 宗教势力崩溃 |
| | 价值 | 意义来源 | 虚无主义兴起 |
| | 势力 | 合法性 | 神授王权的势力瓦解 |

## 二、变更影响评估流程

当你要修改一个设定时：

### Step 1：识别直接受影响领域

```
修改：____
直接受影响：____
影响方式：____
```

### Step 2：评估连锁影响（问三个问题）

1. **谁会因此变强/变弱？**
   - 变强的：____ → 他们会不会打破平衡？
   - 变弱的：____ → 他们会如何反应？

2. **什么规则因此失效/生效？**
   - 失效的规则：____ → 是否有替代规则？
   - 生效的规则：____ → 是否会制造新的约束？

3. **什么剧情因此不再合理？**
   - 已写剧情中：____ 需要修改
   - 计划中剧情：____ 需要调整

### Step 3：决定修改范围

| 影响范围 | 处理方式 | 示例 |
|---------|---------|------|
| 仅影响 1 个文件 | 直接修改 | 修改某个角色的年龄 |
| 影响 2-3 个文件 | 同步修改 + 记录 | 修改力量等级定义 |
| 影响 5+ 个文件 | 必须经过变更评估 | 修改 hard canon |
| 影响已写剧情 | 必须回溯修改 | 修改通行时间 |

## 三、常见的设定矛盾类型

### 类型 1：时间矛盾

**症状**：角色在不同地点的时间对不上
**检测**：检查角色的行程表 vs 事件时间表
**修复**：调整行程时间 / 增加延迟事件 / 使用传送等设定

### 类型 2：战力矛盾

**症状**：角色突然变强/变弱没有合理解释
**检测**：对照力量体系检查升级路径
**修复**：增加修炼过程 / 解释临时爆发的原因 / 增加代价

### 类型 3：地理矛盾

**症状**：地点之间的距离/方向不一致
**检测**：对照地理层级和世界层级图
**修复**：统一地理描述 / 增加地图说明

### 类型 4：社会矛盾

**症状**：社会规则在某些场景被忽略
**检测**：对照社会结构与文化检查
**修复**：解释例外情况 / 修改规则使其更灵活

### 类型 5：关系矛盾

**症状**：角色关系状态前后不一致
**检测**：对照角色关系网络检查
**修复**：补充关系变化的事件 / 修正状态描述

## 四、设定网络密度检查

### 密度评估

| 指标 | 低密度（危险） | 中密度 | 高密度（理想） |
|------|-------------|--------|--------------|
| 角色与势力的关联 | 角色没有势力归属 | 角色有势力但无互动 | 角色与势力深度绑定 |
| 地点与事件的关联 | 地点只是背景 | 地点影响部分事件 | 地点参与剧情推进 |
| 力量与社会的关联 | 力量只用于战斗 | 力量影响部分社会 | 力量塑造整个社会 |
| 历史与当前的关联 | 历史只是背景 | 历史影响部分当前 | 当前是历史的直接结果 |

### 提升密度的方法

1. **给每个设定至少 2 个链接**：
   - 角色 A → 链接到势力 X 和地点 Y
   - 地点 Z → 链接到历史事件 H 和势力 X

2. **让设定相互制约**：
   - 力量的代价 = 社会禁忌
   - 历史事件 = 当前势力格局的原因

3. **让变化产生连锁反应**：
   - 角色 A 死亡 → 势力 X 失去领袖 → 地点 Y 陷入混乱 → 历史事件 H 的阴影重现

---

## 五、使用提示

1. **新增设定时**：立即在矩阵中填写它与其他设定的关系
2. **修改设定时**：执行变更影响评估流程
3. **发现矛盾时**：归类矛盾类型，使用对应修复方法
4. **定期审查**：每卷结束时检查设定网络密度
5. **严禁**：孤立地创建设定——每个设定必须有至少 2 个关联
