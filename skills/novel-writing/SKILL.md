# 写作流程

## 功能定位

把起稿执行、质量门禁、状态回写整合成一个连贯的流程。这个 skill 负责：

1. **起稿执行**：按最小上下文包写场景或章节
2. **质量门禁**：强制执行完整的写作流程
3. **状态回写**：更新进度、伏笔、因果关系

## 工作流程

### 第一步：写前准备（与用户互动）

**一次性确认以下信息**：

1. **加载上下文包**：写哪一章？按大纲继续写还是写指定章节？
2. **确认本章任务**：根据节拍卡确认本章任务
3. **确认写作重点**：完成核心戏剧任务/推进主线/深化人物关系/兑现追读承诺

### 第二步：起稿执行

按以下顺序起稿：

1. 先锁本章最重要的一场戏
2. 先决定角色这次主动做什么
3. 先把冲突和阻力摆上桌
4. 先让局面发生变化
5. 先保章尾发动机

### 第三步：写后自检（与用户互动）

**一次性确认以下信息**：

> 请问您需要检查哪些方面？
> - [ ] 本章是否完成了节拍卡规定的戏剧任务
> - [ ] 本章是否有明确的章尾推进器
> - [ ] 本章是否碰了该碰的旧线
> - [ ] 本章是否留下了下一章非写不可的后果
> - [ ] 全面自检

### 第四步：质量门禁（与用户互动）

根据自检结果，决定下一步：

> 自检结果：通过/未通过
> 
> 请问您需要如何处理？
> - A. 继续写下一章
> - B. 转 `novel-quality` 进行质量检查
> - C. 转 `novel-quality` 进行修订
> - D. 先不处理，继续写作

### 第五步：状态回写

写完正文后至少更新：

- `30-正文/第X卷-卷名/第XXX章.md` 中对应章节
- `90-运行/当前进度.md` 中的当前进度、未决问题、下一步
- 如出现新 hard canon，补到 `10-设定/硬设定.md` 或记入 `90-运行/决策记录.md`
- 如本章推进了伏笔、人物状态、因果债或关系债，联动更新 `20-大纲/因果/场景因果图.md` 或 `20-大纲/回收/回收总账.md`

## 质量底线

起草时默认守住这些底线：

1. 不用整段总结代替戏
2. 不用作者解释代替人物行动
3. 不用抽象情绪词代替身体反应和关系变化
4. 不让角色在没有代价的情况下突然想通、突然变强、突然改变立场
5. 不让章尾只做平静收束，至少要留下问题、压力、决定或余震中的一种
