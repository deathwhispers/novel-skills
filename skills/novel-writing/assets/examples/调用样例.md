# AI 协作写作调用样例

> 样例展示输入与交付顺序，不是正文验收模板。数字、钩子和结构只在对应模式需要时使用。

## 样例 1：探索人物声音

用户输入：

```text
写作模式：探索起草。
试写一个十七岁学生第一次坐夜班公交的片段，目标是发现人物声音。
不要确定性别、城市或最终去向，不必形成完整剧情。
```

期望交付：直接给正文；停在人物显出新的注意力或可能性之处。正文后只在擅自补全信息的风险存在时列待确认项。

## 样例 2：商业连载写作支点

用户输入：

```text
写作模式：商业连载。
上一章：债主在雨夜敲门，妹妹躲在灶后暗格。
本章需要沈砚拖延到巡夜人经过，但不能泄露妹妹位置，也不能让他轻易许诺。
先给最小写作支点。
```

期望支点：视角注意出口；人物想保护妹妹并拖延；债主掌握钱款信息形成压力；结尾动力来自选择的后果。无需填写十项节拍卡。

## 样例 3：潜台词对话

用户输入：

```text
林尘想隐瞒伤势，苏晚已经察觉但不愿逼问。
林尘先观察退路，受到关心时会把话题转向对方；苏晚用具体小事确认别人是否说谎。
写 600 字以内的对话，不靠口头禅制造辨识度。
```

期望交付：动作、停顿和信息策略与人物一致；不附“潜台词解析”，除非用户要求。

## 样例 4：证据型体检

用户输入：

```text
写作模式：文学叙事。只审查，不改写。
重点看父女收拾遗物的场景是否把沉默写成了通用煽情。
```

期望交付：

```markdown
## 硬门禁

- 未发现连续性、动机、因果、canon 或视角硬伤。

## 最影响阅读效果的问题

- 引用原文：……
- 受损效果：旁白提前宣布理解，削弱了物件选择和天气谈话承担的关系压力。
- 最小建议：删去解释句，让女儿挑出坏物件的动作继续承担判断。
```

不输出综合分，不把安静结尾标为 P0。

## 样例 5：最小修订

用户输入：

```text
保留人物没有道歉、女儿已经察觉和天气谈话这三点，只修掉解释过度的句子。
```

期望交付：先给修订正文，再用 1-3 条说明实际改动；不顺手重写人物选择、节奏或结尾。

## 使用原则

1. 写新文本时先给正文。
2. 审查与改写权限分开。
3. P0 只表示叙事硬伤，不表示审美偏好。
4. 人物声音以注意力、回避方式、信息策略、句法节奏和可靠样本为证据。
5. 状态增量记录真实变化；没有变化不制造总账工作。
