# AI Prompt 路由

不要整库加载。先按任务选择一个库，再只读取对应二级标题下的 Prompt。

| 当前任务 | 读取范围 |
|---|---|
| 节拍卡、章纲、设定核对、伏笔、关系、卷纲 | `写前-prompt.md` 对应小节 |
| 对话、战斗、感情、章尾、心理、环境、节奏、续写 | `写中-prompt.md` 对应小节 |
| 体检、连续性、修订、语言风险、voice | `写后-prompt.md` 对应小节 |
| 角色对戏、卡文、批量任务、灵感、平台实验 | `特殊场景-prompt.md` 对应小节 |

加载步骤：

1. 用目录或搜索定位任务标题，例如 `rg -n '^## .*对话|^### Prompt' <file>`。
2. 只读取目标标题到下一个同级标题之间的内容。
3. 替换项目变量，并补充当前写作支点、最近状态增量、必要上下文和不可破坏项。
4. 把 Prompt 中的字数、段数、比例和结构视为可配置参考；项目约束与章节功能优先。
5. AI 产出后先过硬门禁，再判断软目标；不要仅凭词表或比例机械改写。
6. Prompt 正文若与 `写作模式契约.md` 冲突，以模式契约为准；不要因为只加载一个小节而丢失适用范围。

如果任务可直接用当前上下文完成，不为了“用 Prompt”而额外加载模板。
