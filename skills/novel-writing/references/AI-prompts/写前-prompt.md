# 写前 Prompt

> 每次只使用一个 Prompt，并把“写作模式、当前单位、作者意图”一并交给模型。除 hard canon 外，所有结构、数量和篇幅都可被作者意图覆盖。

## 目录

通用输入头；最小写作支点；支点检查；场景或章节；设定与因果；类型兑现；伏笔与信息；关系；结构与反转；长线规划；上下文裁剪；最终判断。

## 通用输入头

```text
【写作模式】商业连载 / 类型长篇 / 文学叙事 / 短篇 / 探索起草
【当前单位】场景 / 章节 / 段落 / 片段 / 卷
【作者意图】这次最想完成、发现或试验什么
【可靠上下文】最近正文、人物状态、hard canon、voice 样本
【允许未知】哪些内容可以通过正文发现
```

## Prompt 1：生成最小写作支点

```text
基于通用输入头，为当前单位生成最小写作支点。

只保留会真正改变起稿选择的项目：当前视角如何注意世界、人物想靠近或逃避什么、压力来自哪里、已知因果与 canon、希望形成的阅读效果。缺失项标记为待探索，不补成事实。

商业连载可额外写本次承诺兑现与继续阅读动力；其他模式不强制钩子、升级或事件推进。
```

## Prompt 2：检查写作支点

```text
检查以下写作支点是否足以起稿。只指出会导致人物选择失真、因果断裂、视角不明或 hard canon 冲突的缺口。

不要因为没填满模板、没有高潮、没有新债或选择安静收束而阻止起稿。输出“可以起稿”或最多 3 个真正阻塞项。
```

## Prompt 3：设计场景或章节

```text
为当前单位设计一条可写的运动轨迹：开始时人物如何理解局面，什么细节或行动施加压力，人物怎样选择，结束时关系、理解、局面、声音或形式发生了什么。

按内容需要决定场景数和篇幅。文学叙事、短篇或探索起草允许停顿、歧义、回响和未完成感；商业连载可选择更明确的兑现与推动。
```

## Prompt 4：设定与因果预检

```text
对照给定正文计划与设定，分开列出：
1. 已证实冲突：引用双方证据并说明为什么不能同时成立；
2. 可解释差异：可能来自视角局限、传闻或时间变化；
3. 尚未确定：留给正文探索。

只把第一类作为写前硬阻塞，不把资料空白误判成冲突。
```

## Prompt 5：类型兑现或情绪高压（按需）

```text
【适用】商业连载、类型长篇，或作者明确要求爽感、恐惧、悬疑、虐感等类型体验。

设计体验成立所需的前因、人物投入、压力变化、兑现瞬间和真实后果。避免把人物降格为反应装置，也不要为了强度添加与主线无关的羞辱、秘密或反转。

输出“体验如何从人物和因果中长出来”，不套固定模式或强度等级。
```

## Prompt 6：伏笔与信息释放

```text
根据当前视角和读者已知信息，判断这次应显露、延迟、误读或保持空白的内容。

每条建议说明：谁能感知、读者会怎样理解、现在释放产生什么效果、延后有什么代价。伏笔只在服务人物、因果或主题时加入，不要求每个单位新埋线。
```

## Prompt 7：关系变化

```text
设计一段关系运动。先写双方各自想得到、隐藏或避免什么，再用行动、误读、交换、沉默或拒绝让距离发生变化。

变化可以是靠近、疏远、重新命名关系，也可以是表面不变但读者理解改变。不要用固定阶段表替代人物的具体历史。
```

## Prompt 8：结构或反转选择

```text
比较 2-3 种真正不同的组织方式，例如顺叙、缺口、并置、循环、视角限制或延迟揭示。说明每种方式会改变什么阅读效果和代价。

只有当反转能重写既有证据且人物需要承担后果时才建议反转；“不反转”必须是合法选项。
```

## Prompt 9：卷级或长线规划

```text
【适用】确有长线生产需要时。

围绕作品承诺、人物弧线、关系与因果建立阶段节点。只规划当前可见范围，把未知项和可变项明确标出。

商业连载可补充追读承诺与更新约束；文学长篇不自动加载留存、升级或章尾强度指标。
```

## Prompt 10：上下文包裁剪

```text
为当前写作任务裁剪上下文。优先保留写作模式、hard canon、当前人物状态、最近有效正文、当前任务和可靠 voice 证据；其次保留相关关系、地点、物件与状态增量。

列出被省略但可能需要回查的来源。不要把模板占位、无关世界观或旧总结塞进正文上下文。
```

## Prompt 11：写前最终判断

```text
判断现在是否可以下笔。硬阻塞只包括：关键人物选择无法判断、已知因果断裂、hard canon 冲突、非预期视角不明。

其余问题列为“可在正文中发现”或“写后复核”。不要用字数、模板完成率、钩子、伏笔数量或固定结构阻止起稿。
```
