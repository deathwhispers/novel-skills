## 一、角色对戏

### Prompt 1：单角色回应

```text
你现在扮演 {{角色名}}。

【当前场景与关系】
{{}}

【人物声音证据】
- 优先注意：{{}}
- 回避或误解：{{}}
- 信息策略：{{}}
- 压力下语言变化：{{}}
- 可靠原文样本：{{}}

【对方刚说】
{{}}

只输出角色此刻的回应与必要动作。不要解释人设，不跳出角色，不用口头禅代替人物声音。
```

### Prompt 2：多角色对戏

```text
【写作模式】{{}}
【场景】{{}}
【参与人物及各自想要/回避的东西】
{{}}
【每人的 voice 证据】
{{}}

让人物按各自注意力和信息策略回应。同一时刻允许有人沉默、打断或不参与；不要求每人平均发言。输出连续场景，不附分析。
```

## 二、批量任务

### Prompt 3：批量写作支点

```text
【写作模式】商业连载 / 类型长篇 / 文学叙事 / 短篇 / 探索起草
【当前阶段计划】{{}}
【需要生成的写作单位】{{场景/章节及范围}}
【必须继承】{{人物状态、因果、canon、voice}}

为每个单位只生成真正需要的支点：视角与注意力、人物趋向、压力或未知、信息变化、预期效果、下一单位继承内容。

商业连载可补承诺兑现和继续阅读动力；其他模式不强制钩子、高潮、固定场景数或十项节拍卡。
```

### Prompt 4：批量初稿

```text
【写作模式】{{}}
【写作支点】{{}}
【项目篇幅约束】{{没有则写“自然篇幅”}}
【voice 与 hard canon】{{}}

逐个生成初稿。每个写作单位先完成自身作用；不统一套四段式，不为凑字数增加事件，不把所有结尾写成悬念。正文后只列真正需要作者确认的事项。
```

> 批量正文适合低风险初稿，不适合一次生成跨越重大人物选择或未知 canon 的长段落。

## 三、voice 校准

### Prompt 5：提取 voice 证据

```text
【可靠项目原文】
{{}}

分别引用原文，提取：叙述注意力、人物回避方式、信息策略、价值判断、句法节奏、叙述距离、压力下变化。区分稳定特征与场景偶发现象，不用“细腻、自然、冷峻”等空泛标签。
```

### Prompt 6：校准待写文本

```text
【可靠 voice 样本】{{}}
【待校准文本】{{}}
【必须保留的内容与节奏】{{}}

先指出最影响辨识度的 1-3 个偏差并引用证据，再做最小改写。不要复制样本表面句子，不靠重复口头禅或标志动作制造一致性。
```

## 四、灵感与卡文

### Prompt 7：提供不同方向

```text
【当前文本停点】{{}}
【人物想靠近、逃避、理解或维持什么】{{}}
【写作模式】{{}}
【不能改变】{{}}

给出 3 个真正不同的继续方向：人物选择方向、信息/关系方向、叙述距离或形式方向。说明各自会产生什么阅读效果与代价，不随机添加陌生人、秘密或反转。
```

### Prompt 8：五分钟恢复

```text
【卡住的位置】{{}}
【最后一段正文】{{}}
【我不确定的是】{{下一句/人物行动/结尾/voice/结构}}

按 `../写中急救指南.md` 只给一个五分钟动作，并提供不超过 200 字的可撤销试写。不要扩写完整章节。
```

## 五、开篇与连载

### Prompt 9：模式化开篇检查

```text
【写作模式】{{}}
【开篇】{{}}
【作品希望读者首先注意什么】{{}}

检查开篇是否建立了当前模式需要的注意力、人物、问题、声音或类型承诺。引用最有效和最失效的原文各一处。商业连载可额外检查进入速度；不使用统一前 200 字、固定字数或强钩子门槛。
```

### Prompt 10：商业连载日更计划

```text
【适用模式】商业连载
【本周可用时间与可持续字数】{{}}
【最近状态增量】{{}}
【阶段承诺】{{}}

生成可持续的 7 天计划：每次写作目的、最小上下文、预计篇幅区间和缓冲日。数字是生产预算；正文自然完成时允许偏离并记录原因。
```

## 六、反馈与平台实验

### Prompt 11：把反馈变成可验证问题

```text
【原文】{{}}
【读者反馈或数据】{{}}
【写作模式与作者意图】{{}}

区分事实、读者偏好、重复信号和单点意见。把反馈改写成最多 3 个可验证假设，并指出需要交给 `novel-feedback` 的证据。不要直接按评论重写正文。
```

### Prompt 12：平台适配实验

```text
【目标平台与已验证规范】{{}}
【原文】{{}}
【必须保留的人物、信息、节奏与 voice】{{}}

只选择一个平台变量做 A/B 小样，例如进入速度、段落长度或信息前置。输出原版、实验版和预期影响；不把“网文化”简化为夸张反应、固定爽虐或每章强钩子。
```
