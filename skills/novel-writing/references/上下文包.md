# 起稿上下文包

## 默认读取

1. `00-书核/作品总表.md`
2. `00-书核/立项单.md`
3. 对应的 chapter beat
4. 对应的 volume 文件

## 条件读取

出现这些情况再追加：

- 角色冲突较复杂 -> 读角色文件
- 规则制约明显 -> 读 `world-rules.md` 或 `canon.md`
- 强连续续写 -> 读上一章
- 会话接续不稳定 -> 读 `90-运行/当前进度.md`
- 正在连载中、怕续写漂 -> 读 `90-运行/连载驾驶舱.md`
- 想提高逻辑完整性 -> 读相关拆文分析中的"因果怎么闭合"
- 想提高细节有效性 -> 读相关拆文分析中的"细节抓手"
- 长篇推进跨章因果或伏笔 -> 读 `20-大纲/因果/场景因果图.md` 和 `20-大纲/回收/回收总账.md`
- 想提高起稿质量下限 -> 读 `references/章节质量标准.md`
- 写的是长篇续章 -> 读 `references/连载续写卡.md`

上下文包选完后，最后再读一次 `references/写前检查表.md`，把章节目标、阻力、因果、细节载体和结尾推进器锁死，再开始起草。

## 拆文反哺

如果项目已经做过对标拆文，优先把拆文结果当成"写法约束"而不是"抄写参考"：

- 从"因果怎么闭合"继承因果闭合方式
- 从"细节抓手"继承细节承担信息和情绪的方法
- 从"迁移到我项目里的注意点"继承哪些可借、哪些必须变形

## 禁忌

- 不默认全量读取整个 `10-设定/`
- 不默认把所有历史章节都读一遍
- 不在写作前先做全面修订
- 不把拆文样例当成表层模仿稿
- 不在写前自检仍有大块空白时强行开写
- 不为了"写够字数"牺牲局面变化和章尾推动力

---

## 常见上下文错误

| 错误 | 表现 | 修复 |
|------|------|------|
| **读太多** | 把 50 章设定全部读一遍 | 只读当前章相关的设定 |
| **读太少** | 不读上一章直接写 | 至少读上一章最后 3 段 |
| **忽略进度** | 不读 `90-运行/当前进度.md` | 必读——这是连载的"驾驶舱" |
| **模仿拆文** | 把拆文样章当模板抄 | 只学因果闭合和细节抓手 |
| **跳过自检** | 不读 `写前检查表.md` 就写 | 必须读完再写 |
| **不锁 voice** | 不读文风指南 | 至少读 `文风快速自检卡.md` 前 4 条 |

## 快速组装（1 分钟）

写前确认上下文包：
1. **必读**：作品总表 + 立项单 + 本章节拍卡
2. **续写**：+ 上一章最后 3 段 + 连载续写卡
3. **复杂**：+ 角色文件 / 世界规则 / 场景因果图
4. **最后**：+ 写前检查表（锁死目标/阻力/因果/结尾）

**上下文包太大→砍掉非必读项。上下文包太小→补读相关文件。**
