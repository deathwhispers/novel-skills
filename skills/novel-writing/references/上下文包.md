# 起稿上下文包

优先用脚本生成可审计的候选包，再人工删减和补充：

```bash
python3 scripts/build_context_pack.py /path/to/my-novel \
  --chapter 12 \
  --task "当前要写的人物、场景与意图" \
  --output /path/to/my-novel/90-运行/上下文包-chapter-012.md
```

复杂人物场景可重复使用 `--include 10-设定/角色/某角色.md`。脚本会区分已纳入与因预算省略的来源；未填写的通用实体卡不参与自动召回。`--max-chars` 包含标题、来源状态、连接换行和截断标记，是硬上限。

## 默认优先级

1. 项目配置、写作模式与作品承诺
2. 相关 hard canon 与当前进度
3. 对应的 chapter beat 或当前写作任务
4. 上一章末段或当前场景前文
5. 最近 1-5 份未合并的章节状态增量
6. 从任务、写作支点、有效前文和增量识别的有效人物、地点、物件及显式附加文件
7. 文风样本、相关因果/回收总账与分卷摘要

书核和立项单内容重复时只读作品总表摘要，不让项目说明挤占正文上下文。

## 条件读取

出现这些情况再追加：

- 角色冲突较复杂 -> 读角色文件
- 规则制约明显 -> 读 `world-rules.md` 或 `canon.md`
- 续写 -> 上一章或当前场景前文属于最高优先级
- 会话接续不稳定 -> 读 `90-运行/当前进度.md`
- 正在连载中、怕续写漂 -> 读 `90-运行/连载驾驶舱.md`
- 想提高逻辑完整性 -> 读相关拆文分析中的"因果怎么闭合"
- 想提高细节有效性 -> 读相关拆文分析中的"细节抓手"
- 长篇推进跨章因果或伏笔 -> 读 `20-大纲/因果/场景因果图.md` 和 `20-大纲/回收/回收总账.md`
- 想提高起稿质量下限 -> 读 `references/章节质量标准.md`
- 写的是长篇续章 -> 读 `references/连载续写卡.md`

上下文包选完后，用 `references/写前检查表.md` 做最小确认。探索起草只需锁当前意图与不可改事实；商业连载再补章级推进目标。

## 拆文反哺

如果项目已经做过对标拆文，优先把拆文结果当成"写法约束"而不是"抄写参考"：

- 从"因果怎么闭合"继承因果闭合方式
- 从"细节抓手"继承细节承担信息和情绪的方法
- 从"迁移到我项目里的注意点"继承哪些可借、哪些必须变形

## 禁忌

- 不默认全量读取整个 `10-设定/`
- 不默认把所有历史章节都读一遍
- 不在写作前先做全面修订
- 不把拆文样例当成表层模仿稿
- 不把未知项自动补写成 hard canon；可逆的探索空白不阻止试写
- 不为了"写够字数"牺牲人物、节奏或作品真正需要的结尾效果

---

## 常见上下文错误

| 错误 | 表现 | 修复 |
|------|------|------|
| **读太多** | 把 50 章设定全部读一遍 | 只读当前章相关的设定 |
| **读太少** | 续写却不读紧邻前文 | 优先读当前场景前文或上一章尾部 |
| **忽略进度** | 长篇续写不知道当前状态 | 按需读 `90-运行/当前进度.md` |
| **模仿拆文** | 把拆文样章当模板抄 | 只学因果闭合和细节抓手 |
| **检查过载** | 未开写先加载全部清单 | 只做当前模式的最小写前确认 |
| **不锁 voice** | 人物声音任务没有可靠样本 | 补读一段同人物正文或 voice 校准记录 |

## 快速组装（1 分钟）

写前确认上下文包：
1. **当前任务**：本章节拍卡或用户指定场景
2. **连续性**：上一章最后 3 段或当前场景前文
3. **实体**：出场人物、地点、物件和相关 canon
4. **风格与长线**：voice 样本、相关因果/伏笔；卷纲和书核只取摘要

**上下文包太大→砍掉非必读项。上下文包太小→补读相关文件。**
