# 作者训练闭环

目标是让检查逐渐减少，而不是让作者永久依赖更多检查表。

## 每 5 章复盘一次

1. 汇总最近章节重复出现的问题，只统计有原文证据的问题。
2. 选一个最影响阅读效果、同时可在下一章练习的目标。
3. 保存一组修改前后片段，说明改动为什么更有效。
4. 下一轮写作只主动提醒这个训练目标，其他软问题写后再看。
5. 连续 3 次稳定通过后移出高频提醒。

## 训练记录

```markdown
## 周期：chapter-XXX 至 chapter-XXX

- 当前唯一训练目标：
- 反复出现的原文证据：
- 它损害的阅读效果：
- 修改前：
- 修改后：
- 下一章可执行练习：
- 连续通过次数：0/3
```

训练目标应写成可观察行为，例如“关键对话中减少直接回答，让回避来自人物关系”，不要写成“文笔更好”“更有张力”。保存到 `90-运行/写作训练日志.md`。
