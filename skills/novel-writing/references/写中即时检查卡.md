# 场景边界检查卡

> 默认在完成一个场景或显著节拍后检查，不按段落或固定字数强制打断。目标是尽早发现会污染后文的硬伤，同时保留连续起草产生的节奏和直觉。

## 默认模式：场景级检查

1. 连续写完当前场景，不在句子层面来回润色。
2. 在视角、地点、时间或主要行动目标切换时停下检查。
3. 只立即修复硬门禁问题；语言和比例问题先标记。
4. 全章完成后统一处理节奏、voice、字数和句子。

## 硬门禁

任一项失败都应先处理，再继续写下一场景：

- [ ] 视角没有无意跳转；有意切换具备清楚边界
- [ ] 人物选择符合当前动机、已知信息和能力边界
- [ ] 没有违反 hard canon、时间线、伤势、位置或资源约束
- [ ] 关键结果能从行动、阻力和规则中推出，不靠作者强行安排
- [ ] 本场景没有让上一场景的直接后果凭空消失

## 场景功能

用一句话回答每项；说不清时回看节拍卡：

- 视角人物此刻在做什么：争取、维持、回避、拖延，还是观察？
- 具体的人、规则、环境或内在矛盾如何阻止他？
- 场景结束时，信息、关系、位置、资源或风险改变了什么？
- 哪个后果推动下一场景，而不是只靠“然后”？
- 本场景最值得保留的动作、画面或潜台词是什么？

场景可以承担呼吸、铺垫或余震功能，不强制每场都出现高潮或反转；但它必须改变读者理解、人物压力或后续选择中的至少一项。

## 软观察

这些信号不自动判失败。先判断偏离是否服务当前场景：

- 叙述距离、句子节奏或描写密度和项目样本不同
- 本场景明显长于或短于计划
- 对话、心理、动作或说明的比例发生变化
- 角色暂时没有使用惯常动作、句式或口头禅
- 出现疑似解释腔、抽象情绪词或常见套语

如果偏离有明确目的，例如高潮处压短句、创伤场景拉远叙述距离、审讯场景提高对话比例，记录意图后保留。不要为了指标统一而抹平变化。

## 三种工作模式

### 默认模式

- 每个场景检查一次硬门禁和场景功能。
- 软观察留到全章完成后处理。

### 高频护栏模式

适用于新手、复杂战斗、多人对话或批量 AI 起草：

- 在关键节拍或约 600-1000 字处快速检查硬门禁。
- 连续两次无硬伤后恢复默认模式。
- 不因字数偏差单独停笔。

### 自由起草模式

适用于作者主动探索、实验性章节或已稳定掌握项目 voice：

- 全场景完成后再检查。
- 写作中只标记疑点，不立即修改。
- 结束后仍必须通过硬门禁。

## 决策顺序

```text
发现问题
  ├─ 硬门禁失败 → 局部修复；骨架失效才回节拍卡
  ├─ 场景功能说不清 → 明确目标/阻力/变化，必要时合并或删除场景
  ├─ soft signal 有创作理由 → 记录理由并保留
  ├─ soft signal 无创作理由 → 标记，完成场景后统一修
  └─ 卡文 → 使用《写中急救指南》，仍无效再补上下文
```

核心规则：先保护因果、人物和设定，再保护场景功能，最后处理语言表面。不要让检查本身切碎作品的呼吸。
