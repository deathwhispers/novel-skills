# 章名/小标题设计技法

> **使用说明**：章名是网文的"第一印象"——读者翻目录时第一眼看到的就是章名。
> 本表是章名设计技法——让章名成为"追读钩子"。
> 配 `novel-market` 使用——商业化改稿也涉及章名。

## 一、章名的 6 大功能

| 功能 | 说明 | 例子 |
|------|------|------|
| **点题** | 让读者知道这章讲什么 | "青云入门" |
| **悬念** | 让读者想看内容 | "那一夜" |
| **情绪** | 调动读者情绪 | "绝望" |
| **记忆** | 让读者记住这章 | "林尘的选择" |
| **暗示** | 暗示后续剧情 | "再见青云宗" |
| **品牌** | 形成个人风格 | 每章都是"一句话"风格 |

> **核心规则**：章名是"微缩版的内容"——读者扫一眼就知道这章讲什么。

## 二、章名的 10 种类型

### 类型 1：地点型

- **特点**：以地点命名
- **例子**：
  - "青云宗"
  - "临渊城"
  - "禁地"
  - "天剑峰"
- **适合**：地点变化/地图扩展
- **优点**：清晰
- **缺点**：可能枯燥

### 类型 2：人物型

- **特点**：以人物命名
- **例子**：
  - "李昭"
  - "苏晚"
  - "赵长老"
- **适合**：重要人物出场
- **优点**：让读者记得人物
- **缺点**：不能太多

### 类型 3：事件型

- **特点**：以事件命名
- **例子**：
  - "内门大比"
  - "秘境开启"
  - "师门决裂"
- **适合**：重要事件
- **优点**：直接告诉读者内容
- **缺点**：可能剧透

### 类型 4：情绪型

- **特点**：以情绪命名
- **例子**：
  - "绝望"
  - "希望"
  - "愤怒"
  - "告别"
- **适合**：情感场景
- **优点**：调动情绪
- **缺点**：可能太抽象

### 类型 5：悬念型

- **特点**：用问题/省略号/反问命名
- **例子**：
  - "谁？"
  - "原来如此"
  - "……"
  - "不可能"
- **适合**：悬疑/反转
- **优点**：钩子强
- **缺点**：用多会疲劳

### 类型 6：决定型

- **特点**：以"决定"命名
- **例子**：
  - "林尘的决定"
  - "反出师门"
  - "最后一战"
- **适合**：关键决策
- **优点**：凸显选择
- **缺点**：可能剧透

### 类型 7：对比型

- **特点**：用对比命名
- **例子**：
  - "生与死"
  - "敌人与朋友"
  - "昨日与今日"
- **适合**：角色弧线/反差
- **优点**：戏剧性
- **缺点**：可能生硬

### 类型 8：意象型

- **特点**：用具体意象命名
- **例子**：
  - "那枚玉佩"
  - "窗前的月光"
  - "满地的落叶"
- **适合**：情感场景
- **优点**：文学性强
- **缺点**：可能太文艺

### 类型 9：问句型

- **特点**：用问题命名
- **例子**：
  - "他是谁？"
  - "为什么？"
  - "接下来怎么办？"
- **适合**：悬疑/转折
- **优点**：钩子强
- **缺点**：不要每章都用

### 类型 10：组合型

- **特点**：多种类型组合
- **例子**：
  - "青云入门 - 第一天"（地点+时间）
  - "李昭的反扑"（人物+事件）
  - "苏晚的眼泪"（人物+意象）
- **适合**：复杂章节
- **优点**：信息量大
- **缺点**：可能太长

## 三、章名的 5 个反模式

| 反模式 | 表现 | 危害 |
|--------|------|------|
| **剧透章名** | "林尘发现苏晚的真实身份" | 失去悬念 |
| **平淡无奇** | "第一章" / "第二天" | 失去记忆点 |
| **太长** | "林尘在青云宗遇到苏晚并拜师" | 失去章名意义 |
| **完全一致** | 每章都是"X" | 失去辨识度 |
| **模仿他人** | 直接抄别人的章名 | 失去原创性 |

## 四、章名的 5 个最佳实践

| 实践 | 说明 |
|------|------|
| **保持 2-8 个字** | 简洁 |
| **动词优先** | 比名词有动感 |
| **具体 > 抽象** | "那枚玉佩" > "往事" |
| **有钩子** | 让读者想看 |
| **形成系列风格** | 比如每章都是"主语+动词" |

## 五、章名设计的工作流

> 写章节前先想好章名——再写正文。

| 步骤 | 操作 |
|------|------|
| 1 | 看完节拍卡——本章的核心戏剧任务是什么 |
| 2 | 选 1 种章名类型（地点/人物/事件/情绪/悬念/决定/对比/意象/问句/组合） |
| 3 | 写 3-5 个候选章名 |
| 4 | 选 1 个最有钩子的 |
| 5 | 写正文 |
| 6 | 写完后检查章名——是否还能更好 |

## 六、章名的"风格统一"

> 全书的章名最好有"风格"——让读者一看就知道是你的书。

| 风格 | 例子 | 适合 |
|------|------|------|
| **极简风** | 全部 2-4 字 | 都市爽文 |
| **四字风** | 全部 4 字 | 古典题材 |
| **文艺风** | 全部意象/诗句 | 文艺题材 |
| **数字风** | "第一章 入门" "第二章 比试" | 传统网文 |
| **混合风** | 不同章用不同风格 | 长篇 |

> **关键**：**选 1 个风格**——全本保持。

## 七、章名的"AI prompt"

### Prompt：设计章名

```
请按以下信息，为 chapter-{{章节号}} 设计 5 个候选章名。

【章节信息】
- 核心戏剧任务：{{}}
- 关键事件：{{}}
- 关键人物：{{}}
- 关键情绪：{{}}

【章名的 10 种类型】
1. 地点型
2. 人物型
3. 事件型
4. 情绪型
5. 悬念型
6. 决定型
7. 对比型
8. 意象型
9. 问句型
10. 组合型

【要求】
- 2-8 字
- 不剧透
- 有钩子
- 至少 1 个让人想看内容的

【输出】5 个候选章名 + 推荐 1 个。
```

### Prompt：检查全本章名

```
请检查全本章名（章节 {{X-Y}}）的一致性。

【章名列表】
{{}}

【检查项】
1. 长度是否一致？
2. 风格是否统一？
3. 是否有重复类型？
4. 是否有 5 个反模式？
5. 是否有"读者记忆点"章名？
6. 是否有"钩子"章名？

按 P0/P1/P2 列出问题。
```
