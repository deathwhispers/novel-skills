# 流程检查清单

这是章节写作流程的强制检查清单。每个步骤完成后，必须逐项检查，全部通过才能进入下一步。

## 第一步：写前准备检查清单

### 必须通过的检查项

- [ ] 最小上下文包已加载
  - [ ] `00-书核/作品总表.md` 已读取
  - [ ] `00-书核/立项单.md` 已读取
  - [ ] 当前章节 beat 已读取
  - [ ] 当前卷文件已读取

- [ ] 写前检查表已完成
  - [ ] 读取 `references/写前检查表.md`
  - [ ] 所有空项已补全
  - [ ] 没有三项以上说不清的问题

- [ ] 本章任务已锁定
  - [ ] 本章要完成什么任务已明确
  - [ ] 本章冲突是什么已明确
  - [ ] 前后照应关系已明确
  - [ ] 人物状态已明确
  - [ ] 章尾推进器已设计

- [ ] 可以下笔的确认
  - [ ] 所有必要信息就位
  - [ ] 没有遗漏的关键设定
  - [ ] 可以开始写作

### 失败处理

如果以上任何检查项不通过：
- 停止写作流程
- 转回 `novel-outline` 补齐信息
- 补齐后重新开始

## 第二步：草稿执行检查清单

### 必须通过的检查项

- [ ] 起稿执行链已完成
  - [ ] 本章最重要的一场戏已锁定
  - [ ] 角色这次主动做什么已决定
  - [ ] 冲突和阻力已摆上桌
  - [ ] 局面已发生变化
  - [ ] 章尾发动机已保住

- [ ] 场景级写作循环已完成
  - [ ] 每场至少完成：角色想做什么、被什么卡住、局面变成什么、留下什么债
  - [ ] 如果参考了拆文结果，额外自检已完成

- [ ] 质量底线已守住
  - [ ] 没有用整段总结代替戏
  - [ ] 没有用作者解释代替人物行动
  - [ ] 没有用抽象情绪词代替身体反应
  - [ ] 没有让角色在没有代价的情况下突然改变
  - [ ] 章尾留下了问题、压力、决定或余震

- [ ] 写后自检已完成
  - [ ] 读取 `references/写后自检卡.md`
  - [ ] 检查本章是否达到 `references/章节质量标准.md` 的最低标准
  - [ ] 决定下一步方向

### 失败处理

如果以上任何检查项不通过：
- 停止继续写作
- 转 `novel-quality` 检查问题
- 根据检查结果决定下一步

## 第三步：章节体检检查清单

### 必须通过的检查项

- [ ] 体检流程已完成
  - [ ] 使用 `novel-quality` 进行体检
  - [ ] 按六层检查顺序进行
  - [ ] 体检报告已生成到 `40-修订/体检报告/`

- [ ] 体检内容已完成
  - [ ] 本章自己是否成立已检查
  - [ ] 和上一章、上一场是否接得上已检查
  - [ ] 人物状态、关系状态、已知信息是否延续已检查
  - [ ] 伏笔、信息债、物件债、关系债有没有丢已检查
  - [ ] 章尾是否有效推动下一章已检查
  - [ ] 如果是长篇：卷级节奏、长线推进和追读压力是否还在已检查

- [ ] 体检结果已明确
  - [ ] 问题清单已输出
  - [ ] 下一步方向已明确

### 失败处理

如果体检发现问题：
- 停止进入下一步
- 转 `novel-quality` 进行修订
- 修订后重新体检

## 第四步：修订统稿检查清单（如果需要）

### 必须通过的检查项

- [ ] 主问题模板已选择
  - [ ] 因果断裂、动机悬浮 -> `references/逻辑修补模板.md`
  - [ ] 细节稀薄、情绪不落地 -> `references/细节修补模板.md`
  - [ ] 伏笔散、回收硬 -> `references/回收修补模板.md`

- [ ] 三层修订已完成
  - [ ] 结构层：beat是否成立、章节目标是否清晰、转折是否发生、结尾钩子是否有效
  - [ ] 场景层：对抗是否足够、行动与因果是否顺、信息是否在正确时机揭示、对话是否有角色区分度
  - [ ] 句子层：是否过度总结、是否讲解味太重、是否对称平铺泛化、是否存在明显AI套话

- [ ] 修订报告已输出
  - [ ] 修订后的正文已完成
  - [ ] `40-修订/修稿报告/` 下对应的修订报告已生成
  - [ ] 如涉及 canon 改动，`90-运行/决策记录.md` 已更新

### 失败处理

如果修订失败：
- 停止进入下一步
- 转回 `novel-outline` 重新规划
- 重新开始写作流程

## 第五步：去AI味检查清单

### 必须通过的检查项

- [ ] AI味检查已完成
  - [ ] 解释过多问题已检查
  - [ ] 节奏过匀问题已检查
  - [ ] 情绪靠概括词问题已检查
  - [ ] 角色说话像同一个人问题已检查

- [ ] 去味处理已完成
  - [ ] 先删解释句和总结句
  - [ ] 再拆平滑句式（长短句交替）
  - [ ] 再补动作、视角感知和人味停顿
  - [ ] 最后检查是否改坏角色声音

- [ ] 题材差异化去味已完成
  - [ ] 玄幻/仙侠：避免修炼过程过于详细
  - [ ] 都市/现实：避免场景描写面面俱到
  - [ ] 言情/感情流：避免角色情绪被直接说出来
  - [ ] 悬疑/推理：避免线索解释过于清楚

- [ ] 去味结果已输出
  - [ ] 主要AI味来源已说明
  - [ ] 修订后的正文已完成
  - [ ] 修改说明已提供

### 失败处理

如果去味失败：
- 停止进入下一步
- 重新检查AI味问题
- 重新处理

## 第六步：状态回写检查清单

### 必须通过的检查项

- [ ] 章节文件已更新
  - [ ] `30-正文/第X卷-卷名/第XXX章.md` 中对应章节已更新

- [ ] 进度文件已更新
  - [ ] `90-运行/当前进度.md` 中的当前进度已更新
  - [ ] `90-运行/当前进度.md` 中的未决问题已更新
  - [ ] `90-运行/当前进度.md` 中的下一步已更新

- [ ] 相关文件已更新
  - [ ] 如出现新 hard canon，`10-设定/硬设定.md` 或 `90-运行/决策记录.md` 已更新
  - [ ] 如本章推进了伏笔、人物状态、因果债或关系债，`20-大纲/因果/场景因果图.md` 或 `20-大纲/回收/回收总账.md` 已更新

### 失败处理

如果状态回写失败：
- 停止流程
- 手动更新相关文件
- 确保所有文件都已更新

## 流程完成确认

所有步骤完成后，确认以下事项：

- [ ] 所有检查清单项都已通过
- [ ] 所有产物都已生成
- [ ] 所有相关文件都已更新
- [ ] 章节写作流程已完成
- [ ] 可以进入下一章写作

## 质量门禁总结

这个流程确保每个章节都经过：

1. **完整的写前准备** - 避免信息不足导致的写作问题
2. **标准的草稿执行** - 确保戏成立、逻辑闭合、细节落地
3. **强制的章节体检** - 发现并解决潜在问题
4. **必要的修订统稿** - 修复体检发现的问题
5. **专业的去AI味** - 确保文字自然流畅
6. **完整的状态回写** - 保持项目连续性

通过这个强制流程，可以确保每个章节都达到高质量标准，避免常见的写作问题，保持长篇连载的连续性和质量稳定性。
